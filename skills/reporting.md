# Reporting Skill

Use when writing daily and weekly routine reports.

Inputs: memory summary, research, strategy decision, risk decision, orders, PnL notes, errors.

Outputs: Markdown report under `reports/daily/` or `reports/weekly/`.

Safety rules: no profit claims; include limitations; include rejected orders; include next human review actions.

Failure modes: missing market data, missing broker state, research unavailable.

Example: write `reports/daily/YYYY-MM-DD-close.md` with trades, risk events, PnL notes, and next actions.
