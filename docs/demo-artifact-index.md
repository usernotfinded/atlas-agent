# Atlas Agent Demo Artifact Index

> **Not financial advice.** Atlas Agent is a software tool, not a financial advisor. Trading involves significant risk of loss.

## Purpose

This index explains the local artifacts produced or referenced by the paper demo workflow. It is for reviewers who want to understand what the demo proves, what each artifact contains, and what safety boundaries remain in force.

## Safety Scope

- Paper/local only.
- No live broker credentials required.
- No provider API keys required.
- No live orders submitted.
- No provider execution unlocked.
- No broker execution unlocked.
- No financial advice.
- No autonomous trading claim.

## Demo Command

```bash
./scripts/demo_paper_workflow.sh
```

## Artifact Summary

| Artifact / Output | Typical Location | Produced By | Purpose | Safety Meaning |
| ----------------- | ---------------- | ----------- | ------- | -------------- |
| Demo workspace | `/tmp/atlas-agent-demo.XXXXXX` (or `$DEMO_WORKSPACE`) | `atlas init` | Isolated temporary workspace for the demo | Local-only; safe to delete after review |
| Config file | `<workspace>/.atlas/config.toml` | `atlas init`, then `atlas config set` | Stores `market.symbol = "ATLAS-DEMO"` and default paper mode | Live trading disabled by default |
| Discipline profile | `<workspace>/.atlas/discipline.md` | `atlas discipline setup --manual --yes` | Explicit safe discipline profile required for agentic workflows | No unsafe defaults; manual opt-in |
| Backtest result JSON | `<workspace>/.atlas/backtests/bt-<timestamp>/result.json` | `atlas backtest run` | Deterministic sample-data backtest output | Local-only simulation; no broker contact |
| Backtest report MD | `<workspace>/.atlas/backtests/bt-<timestamp>/report.md` | `atlas backtest run` | Human-readable research summary | Research-only; no performance guarantee |
| Audit directory | `<workspace>/audit/` | `atlas init` (empty) | Directory for future run manifests and event logs | Tamper-evident evidence surface |
| Pending orders dir | `<workspace>/pending_orders/` | `atlas init` (empty) | Queue for orders awaiting approval | Remains empty in demo; no orders created |
| Runtime directories | `<workspace>/memory/`, `reports/`, `events/` | `atlas init` | Standard workspace runtime structure | No sensitive data written in demo |

## Artifact Details

### 1. Demo Workspace

Path:

- Temporary path printed at the start and end of the script (e.g., `/tmp/atlas-agent-demo.XXXXXX`).
- Alternatively, a user-supplied `$DEMO_WORKSPACE` if set.

Contains:

- `.atlas/` — configuration, backtests, locks, safety evidence
- `memory/` — conversation and learning memory (empty after demo)
- `reports/` — daily, agent, learning, reflection, weekly report directories (empty after demo)
- `pending_orders/` — approval queue (empty after demo)
- `audit/` — audit events and manifests (empty after demo)
- `events/` — event log directory (empty after demo)

What it proves:

- Atlas can create a self-contained workspace from a template.
- The workspace is isolated in a temp directory and does not touch user home or production paths.

What it does not prove:

- It does not prove live trading readiness.
- It does not prove broker connectivity.
- It does not prove provider execution.

### 2. Config File

Path: `<workspace>/.atlas/config.toml`

Contains:

- `market.symbol = "ATLAS-DEMO"`
- Default paper-mode settings

What it proves:

- Configuration is explicit and local.
- Symbols must be set manually; Atlas does not choose one for you.

What it does not prove:

- It does not prove the symbol is tradable with a real broker.
- It does not prove any data feed is configured.

### 3. Discipline Profile

Path: `<workspace>/.atlas/discipline.md`

Contains:

- Default safe discipline profile generated by `--manual --yes`.

What it proves:

- Agentic workflows require an explicit discipline profile.
- The default profile is safe and requires no credentials.

What it does not prove:

- It does not prove the profile has been reviewed for a specific trading strategy.
- It does not prove the profile is suitable for live trading.

### 4. Backtest Result JSON

Path: `<workspace>/.atlas/backtests/bt-<timestamp>/result.json`

Contains:

- Deterministic backtest metrics, equity curve, and trade list from sample data.
- `generated_at`, `disclaimer`, and `report_type` fields injected for auditability.

What it proves:

- The backtest engine runs locally on CSV sample data.
- Output is deterministic and reproducible for the same input.

What it does not prove:

- It does not prove future performance.
- It does not prove the strategy works on real market data.
- It does not prove broker execution capability.

### 5. Backtest Report Markdown

Path: `<workspace>/.atlas/backtests/bt-<timestamp>/report.md`

Contains:

- Human-readable research summary with metrics table, benchmark section, and disclaimer.

What it proves:

- Atlas can render local research artifacts for human review.

What it does not prove:

- It is a research summary, not investment advice.
- Past simulated results do not indicate future outcomes.

### 6. Audit Directory

Path: `<workspace>/audit/`

Contains:

- Nothing after the paper demo dry-run. The directory exists because `atlas init` creates it.

What it proves:

- The workspace is structured to support tamper-evident audit trails.

What it does not prove:

- No manifests are created during the dry-run demo.
- `atlas audit verify --all` reports `No manifests found.` This is expected and safe.

### 7. Pending Orders Directory

Path: `<workspace>/pending_orders/`

Contains:

- Nothing after the demo. The paper dry-run does not create orders.

What it proves:

- The approval queue structure exists but remains empty unless explicitly populated.

What it does not prove:

- It does not prove order submission to a broker.
- It does not prove the approval gate is configured for live flow.

## Success Criteria

A successful demo should:

- Exit with code `0`
- Complete local validation
- Complete paper workflow
- Complete sample backtest/report step
- Verify audit/manifest step (expected: no manifests found)
- Require no live broker credentials
- Require no provider API keys
- Submit no live orders

## Common Non-Artifact Outputs

The following are printed to stdout but are not persisted artifacts:

- `Atlas Agent paper workflow demo` banner
- Per-step CLI output from `atlas validate`, `atlas run --mode paper --dry-run`, etc.
- `Demo complete. Review the temporary workspace at: ...`

## Troubleshooting

| Symptom | Likely cause | Resolution |
|---|---|---|
| `Missing prerequisite: sample data not found` | Repository not cloned or `data/sample/ohlcv.csv` missing | Clone the repo and ensure sample data is present |
| `atlas: command not found` or Python import error | Atlas not installed in editable mode | Run `python3.11 -m pip install -e .` from the repository root |
| `Status: not ready for agentic paper workflows` | No AI provider API key configured | Expected and safe; backtest and dry-run still complete |
| `No manifests found` during audit verify | Dry-run does not create run manifests | Expected and safe |
| Generated workspace path not printed | `DEMO_WORKSPACE` was set and reused | Check the environment variable or run without it |
