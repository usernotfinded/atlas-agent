## 1. Title and candidate ID

**CAND-010: Safety-State Persistence Regression Guard**

Turn the CAND-009 manual fixed-temp write grep guard into a committed static
checker (and a small companion pytest test) so that reintroduction of fixed
`<target>.tmp` writes in safety persistence modules fails verification.

The guard is strictly static: it reads source text, asserts that the four
migrated safety persistence modules continue to call the CAND-009
`atomic_write_json` / `atomic_write_text` helper, and rejects fixed temporary
filename construction patterns.

## 3. Problem statement

Fixed `<target>.tmp` safety-state persistence was an intentional CAND-009
anti-pattern. Reintroducing it would silently regress the safety property that
CAND-009 established:

1. **Collision risk returns.** Two writers or rapid retries would again share
   the same temporary path.
2. **Stale temp files become diagnostic noise.** A crash could leave a
   `<target>.tmp` file that the next writer reuses.
3. **Review fatigue.** The manual grep guard is not in a named script, so it
   does not appear in `scripts/check_*.py` inventories and is not exercised by
   the existing checker test pattern.
4. **False confidence.** The migrated modules currently pass tests, but nothing
   in `scripts/release_check.sh --quick` explicitly forbids the old pattern
   except a one-off shell command.

CAND-010 makes the guard committed, named, tested, and integrated.

---

## 4. CAND-009 safety property to preserve

> **Safety persistence modules must not reintroduce fixed `<target>.tmp` write
> paths such as `with_suffix(... + ".tmp")` or hardcoded `.json.tmp` / `.tmp`
> target names. Safety writes should continue using the CAND-009 atomic_write
> helper with unique same-directory temp files.**

Concretely:

- All safety persistence writes that target JSON state files must route through
  `atomic_write_json` (or `atomic_write_text` for future text payloads).
- Temporary filenames must be generated by `tempfile.mkstemp` inside the helper,
  not constructed by string concatenation or `Path.with_suffix` in the callers.
- Atomic `replace` semantics, JSON formatting, and best-effort `chmod(0o600)`
  behavior must remain unchanged.

## 8. Disallowed patterns

In the four guarded files, the checker must detect the following patterns.
Matching is case-sensitive and line-oriented.

### 8.1 Core fixed-temp construction patterns

| # | Pattern | Example that must fail |
|---|---|---|
| 1 | `with_suffix(... + ".tmp")` | `tmp_path = target.with_suffix(target.suffix + ".tmp")` |
| 2 | `with_suffix(... + '.tmp')` | `tmp_path = target.with_suffix(target.suffix + '.tmp')` |
| 3 | `suffix + ".tmp"` | `tmp_path = target.suffix + ".tmp"` |
| 4 | `suffix + '.tmp'` | `tmp_path = target.suffix + '.tmp'` |
| 5 | literal `".json.tmp"` | `tmp_path = target.parent / (target.name + ".json.tmp")` |
| 6 | literal `'.json.tmp'` | `tmp_path = target.parent / (target.name + '.json.tmp')` |
| 7 | `Path(str(target) + ".tmp")` | `tmp_path = Path(str(target) + ".tmp")` |
| 8 | `str(target) + ".tmp"` | `temp = str(target) + ".tmp"` |
| 9 | `target + ".tmp"` | `temp = target + ".tmp"` (where `target` is a `Path`-like name) |

### 8.2 Additional risky temp-file patterns

| # | Pattern | Rationale |
|---|---|---|
| 10 | `NamedTemporaryFile(delete=False)` outside the helper | Could reintroduce a fixed or shared temp path. |
| 11 | `mktemp` | Insecure fixed-name temp generation. |
| 12 | Hardcoded temp paths in guarded modules | e.g., `"/tmp/kill_switch_state.json.tmp"` or `"memory/state.json.tmp"`. |

### 8.3 Pattern definitions (regex-ready)

The implementation should use regexes equivalent to:

```text
r'\.with_suffix\s*\([^)]*\+\s*["\']\.tmp["\']'
r'\.with_suffix\s*\([^)]*\+\s*["\']\.json\.tmp["\']'
r'suffix\s*\+\s*["\']\.tmp["\']'
r'suffix\s*\+\s*["\']\.json\.tmp["\']'
r'["\']\.json\.tmp["\']'
r'Path\s*\(\s*str\s*\([^)]+\)\s*\+\s*["\']\.tmp["\']\s*\)'
r'str\s*\([^)]+\)\s*\+\s*["\']\.tmp["\']'
r'\+\s*["\']\.tmp["\']'
r'NamedTemporaryFile\s*\('
r'\bmktemp\b'
r'["\'][^"\']*\.json\.tmp["\']'
```

Notes:

- Pattern `r'\+\s*["\']\.tmp["\']'` is intentionally broad; it catches any
  string concatenation that produces a `.tmp` suffix. To avoid false positives
  in legitimate comments, the checker should treat it as a code-context pattern
  (i.e., the line must also contain a `Path`-like variable, `with_suffix`,
  `write_text`, `replace`, or assignment). If that proves too noisy, it may be
  demoted to a warning rather than a hard failure.
- The literal `".json.tmp"` pattern (#5/#6) is intentionally strict: any
  hardcoded fixed temp extension in a guarded module is a regression risk.
- `mktemp` is flagged only as a word boundary so that `mkstemp` in the helper
  is not matched.

## 9. Allowed patterns

The checker must allow the following:

1. **Helper imports** in guarded files:
   ```python
   from atlas_agent.safety.atomic_write import atomic_write_json, atomic_write_text
   ```

2. **Helper calls** in guarded files:
   ```python
   atomic_write_json(self.heartbeat_path, payload, chmod=0o600)
   atomic_write_text(self.state_path, content)
   ```

3. **Unique temp creation in `atomic_write.py`**:
   ```python
   fd, temp_str = tempfile.mkstemp(
       dir=target.parent,
       prefix=f"{target.name}.",
       suffix=".tmp",
   )
   ```
   The checker either excludes `atomic_write.py` from the fixed-temp scan or
   recognizes this exact `mkstemp` pattern as an allowed unique-temp
   construction.

4. **Tests that intentionally assert the checker catches regressions**:
   Tests may contain the disallowed literal strings inside helper functions
   that create temporary patched source files. The checker script itself must
   not scan `tests/test_check_safety_atomic_write.py` with the same strict
   rules, or the test must construct patterns dynamically so they are not
   present in the test file's own source text.

5. **Historical documentation** describing old behavior, unless the checker is
   intentionally extended to scan docs. The default scope is source files only.

6. **Comments in guarded files** that mention `.tmp` in a negative or
   historical context, provided they do not contain actual code patterns
   (e.g., a comment saying `# CAND-009: do not use target.with_suffix(".tmp")`
   is allowed; a comment containing the literal executable pattern
   `tmp = target.with_suffix(".tmp")` may still be flagged).

## 10. Recommended checker/test architecture

### 10.1 Approach comparison

| Approach | Description | Pros | Cons | Recommendation |
|---|---|---|---|---|
| **A — Dedicated checker script** | `scripts/check_safety_atomic_write.py` scans guarded files and exits non-zero on regressions. | Follows existing `scripts/check_*.py` convention; integrates into shell gates; emits clear file/line errors; reusable in CI. | One new file to maintain. | **Primary recommendation.** |
| **B — Pytest regression test only** | `tests/safety/test_atomic_write_regression.py` scans source text. | No new script; runs with normal pytest. | Does not run in `release_check.sh` shell gates unless pytest is invoked; less visible. | Reject as primary, but useful as companion. |
| **C — Hybrid checker + pytest wrapper** | Both a checker script and a pytest test that imports/invokes it. | Strongest coverage; script runs in gates, test verifies script behavior. | Slightly more maintenance; two files. | Accept as the implemented form: primary script + companion test. |

### 10.2 Chosen architecture

**Primary: Approach A with a lightweight Approach-C test companion.**

Create:

- `scripts/check_safety_atomic_write.py` — the static regression guard.
- `tests/test_check_safety_atomic_write.py` — verifies the checker passes on
  the current repo and fails on injected regressions.

This mirrors repository conventions:

- `scripts/check_demo_proof.py` → `tests/test_demo_proof_checker.py`
- `scripts/check_submit_execution_safety.py` → `tests/test_submit_execution_safety_check.py`
- `scripts/check_forbidden_claims.py` → no dedicated test, but used in gates

The checker is the authoritative gate; the test is the safety net that
prevents silent checker breakage.

## 11. Checker algorithm

### 11.1 Module outline

```text
scripts/check_safety_atomic_write.py
```

Responsibilities:

1. Resolve repository root.
2. Resolve the four guarded files and the helper file.
3. For each guarded file:
   - Read source text.
   - Scan each line for disallowed patterns.
   - Record violations with file path, line number, and matched pattern.
4. Optionally verify that each guarded file imports and calls an
   `atomic_write_*` helper (positive signal, not a hard gate on its own).
5. Print violations to stderr/stdout.
6. Exit `0` if clean, `2` if violations found, `1` if an internal error occurs.

### 11.2 Detailed algorithm

```python
def main() -> int:
    repo_root = resolve_repo_root()
    guarded_files = [
        repo_root / "src" / "atlas_agent" / "safety" / "heartbeat.py",
        repo_root / "src" / "atlas_agent" / "safety" / "deadman.py",
        repo_root / "src" / "atlas_agent" / "safety" / "kill_switch.py",
        repo_root / "src" / "atlas_agent" / "safety" / "state.py",
    ]
    helper_file = repo_root / "src" / "atlas_agent" / "safety" / "atomic_write.py"

    violations = []
    for path in guarded_files:
        violations.extend(scan_file(path))

    if violations:
        print("Safety atomic-write regression check FAILED")
        for v in violations:
            print(f"  {v.path}:{v.line}: {v.message}")
        return 2

    print("Safety atomic-write regression check PASSED")
    return 0
```

### 11.3 Scan function

```python
 DISALLOWED_PATTERNS: list[tuple[str, str]] = [
    (r'\.with_suffix\s*\([^)]*\+\s*["\']\.tmp["\']', "fixed with_suffix .tmp pattern"),
    (r'\.with_suffix\s*\([^)]*\+\s*["\']\.json\.tmp["\']', "fixed with_suffix .json.tmp pattern"),
    (r'suffix\s*\+\s*["\']\.tmp["\']', "suffix + .tmp pattern"),
    (r'suffix\s*\+\s*["\']\.json\.tmp["\']', "suffix + .json.tmp pattern"),
    (r'["\']\.json\.tmp["\']', "literal .json.tmp"),
    (r'Path\s*\(\s*str\s*\([^)]+\)\s*\+\s*["\']\.tmp["\']\s*\)', "Path(str(target) + .tmp) pattern"),
    (r'str\s*\([^)]+\)\s*\+\s*["\']\.tmp["\']', "str(target) + .tmp pattern"),
    (r'NamedTemporaryFile\s*\(', "NamedTemporaryFile usage outside helper"),
    (r'\bmktemp\b', "mktemp usage"),
    (r'["\'][^"\']*\.json\.tmp["\']', "hardcoded .json.tmp path"),
]

def scan_file(path: Path) -> list[Violation]:
    violations = []
    text = path.read_text(encoding="utf-8")
    for lineno, line in enumerate(text.splitlines(), start=1):
        stripped = line.strip()
        if stripped.startswith("#"):
            # Skip pure comment lines for broad .tmp patterns to reduce noise.
            continue
        for pattern, message in DISALLOWED_PATTERNS:
            if re.search(pattern, line):
                violations.append(Violation(path, lineno, message, line.strip()))
                break  # One violation per line is sufficient.
    return violations
```

### 11.4 Positive-signal check (optional)

The checker may also verify that each guarded file contains at least one
reference to `atomic_write_json` or `atomic_write_text`. This is a diagnostic,
not a hard gate; a missing import would likely be caught by tests, but the
checker can emit a warning.

```python
def has_atomic_write_reference(text: str) -> bool:
    return re.search(r'atomic_write_(json|text)\s*\(', text) is not None
```

### 11.5 Helper-file verification (optional)

The checker may verify that `atomic_write.py` contains `tempfile.mkstemp` and
does **not** contain the disallowed fixed-temp patterns. This is a diagnostic
warning, not a hard gate, because the helper is explicitly allowed to construct
`.tmp` names via `mkstemp`.

## 12. Error message format

Each violation must be printed on a single line in this format:

```text
<relative-path>:<line>: <pattern-description>
    <source-line-snapshot>
```

Example:

```text
Safety atomic-write regression check FAILED
  src/atlas_agent/safety/heartbeat.py:18: fixed with_suffix .tmp pattern
      tmp_path = self.heartbeat_path.with_suffix(self.heartbeat_path.suffix + ".tmp")
  src/atlas_agent/safety/state.py:42: suffix + .tmp pattern
      tmp_path = self.state_path.suffix + ".tmp"
```

On success:

```text
Safety atomic-write regression check PASSED
```

Exit codes:

| Code | Meaning |
|---|---|
| `0` | No disallowed patterns found. |
| `1` | Internal checker error (file not found, bad argument). |
| `2` | One or more disallowed patterns found. |

### 14.3 Test implementation notes

- Use `subprocess.run([sys.executable, str(CHECKER_SCRIPT), str(tmp_repo)], ...)`
  to test the checker as a black-box script.
- For injected regression tests, create a minimal temp directory that mirrors
  only `src/atlas_agent/safety/` and run the checker against it. This avoids
  mutating the real repo.
- For test #7, build the forbidden string at runtime with string concatenation
  so the literal disallowed pattern does not appear in the test source.

Example helper:

```python
def _inject_pattern_and_run(src: Path, pattern: str, tmp_path: Path) -> subprocess.CompletedProcess[str]:
    repo = tmp_path / "repo"
    target = repo / "src" / "atlas_agent" / "safety"
    target.mkdir(parents=True)
    text = src.read_text(encoding="utf-8")
    # Inject pattern into the record method region (or simply append after imports).
    injected = text + f"\n# injected regression\n{pattern}\n"
    (target / src.name).write_text(injected, encoding="utf-8")
    return subprocess.run(
        [sys.executable, str(CHECKER_SCRIPT), str(repo)],
        capture_output=True,
        text=True,
    )
```

## 20. Open questions, resolved conservatively

1. **Should the checker scan comments and docstrings?**
   - **Resolved:** Pure comment lines are skipped for the broad `.tmp`
     concatenation pattern to reduce noise. Literal `".json.tmp"` and
     `with_suffix(...)` patterns are still flagged even if they appear in
     comments, because they describe the exact anti-pattern we want to prevent.

2. **Should the checker also scan tests under `tests/safety/`?**
   - **Resolved:** No. Tests intentionally reference `.tmp` filenames in
     assertions (e.g., `assert not (tmp_path / "target.json.tmp").exists()`).
     The checker test file builds injected patterns dynamically to avoid
     self-flagging.

3. **Should the checker require an `atomic_write_*` call in every guarded file?**
   - **Resolved:** This is a diagnostic warning, not a hard gate. The primary
     gate is the absence of disallowed patterns. A missing helper call would be
     caught by existing tests and is not in CAND-010 scope.

4. **Should the checker verify that `atomic_write.py` itself uses `mkstemp`?**
   - **Resolved:** Optional positive check only. The helper is trusted by
     candidate boundary; the guard's job is to ensure callers do not bypass it.

5. **Should the companion cleanup be part of CAND-010 or a separate item?**
   - **Resolved:** Companion cleanup is part of CAND-010 but is explicitly
     optional and non-behavioral. It fixes ruff/mypy issues introduced or
     exposed by CAND-009. If it conflicts with other work, it may be deferred.

6. **What if a future caller legitimately needs a fixed temp name?**
   - **Resolved:** Safety persistence modules must not use fixed temp names.
     Any future exception requires an explicit design review and an allowlist
     entry in the checker with documented justification.
