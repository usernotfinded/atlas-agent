# Safety

Atlas Agent v0.5.3 prioritizes safety
 through multiple decoupled layers:

## 1. Deterministic Risk Manager
Every order proposal is intercepted and validated by the `RiskManager`. It enforces hard limits on:
- Maximum position notional
- Maximum single trade notional
- Daily loss limits
- Symbol allowlists/blocklists
- Portfolio exposure percentages

## 2. Advanced Kill Switch
The global kill switch provides hierarchical emergency modes:
- **Soft Pause**: Prevents new orders while allowing the agent to manage existing ones.
- **Cancel All**: Automatically cancels all active pending orders.
- **Flatten All**: Executes a generated plan to close all open positions safely.
- **Heartbeat Protection**: A "dead-man" heartbeat failure stops the system if the controller process crashes.

## 3. Gated Action Execution
Emergency "Safety Action Plans" are generated by the `SafetyActionPlanner` but require manual approval before the `SafetyActionExecutor` can process them. This prevents AI-driven "panic selling" without oversight.

## 4. Tamper-Evident Audit
All system actions, including risk rejections and kill-switch activations, are recorded in a cryptographically linked hash-chain. Run manifests and root hashes ensure that logs cannot be truncated or modified without detection.

## 5. Live Mode Isolation
Live trading is never enabled by default. It requires:
- `TRADING_MODE=live`
- `ENABLE_LIVE_TRADING=true`
- Manual approval for every order by default.
- Active broker synchronization.
