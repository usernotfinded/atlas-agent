## Scope

`v0.6.19` contains CAND-011, a kill-switch type-safety cleanup. It narrows the
audit payload construction in `src/atlas_agent/safety/kill_switch.py` to
eliminate the pre-existing mypy `union-attr` warning on
`last_heartbeat().isoformat()`, preserves the `last_heartbeat` audit payload key
and `str | None` value shape, and adds mandatory `heartbeat_expired`
audit-payload regression tests. No protected runtime safety boundary changed
during the release cutover.

## Included Security Hardening

- Kill-switch audit payload construction is now type-safe and reads the heartbeat
  state exactly once, removing a narrow inter-call race.
- All existing deterministic risk, approval, kill-switch, dead-man heartbeat,
  and audit hash-chain requirements remain in place.
- No protected runtime safety boundary changed during the release cutover.
