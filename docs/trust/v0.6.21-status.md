## Scope

`v0.6.21` contains three accepted candidates:

- **CAND-013:** Public/Trust Docs Drift Coverage Guard — extends
  `scripts/check_public_docs_consistency.py` to use release metadata as the
  current/next-planned authority and to catch stale trust README current-public
  labels, autonomy-roadmap candidate-state contradictions, and related public-docs
  drift.
- **CAND-014 Phase 2:** Provider Artifact Engine Deduplication — one-module pilot
  extraction that moves shared artifact mechanics for
  `src/atlas_agent/research/provider_mock_response_final_safety_seal.py` behind a
  minimal `src/atlas_agent/research/artifact_engine.py` while preserving Phase 1
  golden compatibility. Extraction beyond this pilot remains not authorized.
- **CAND-015:** Bounded Live Autonomy Readiness Gate — adds an evidence-only,
  simulated-only L2/L3 autonomy readiness gate (`atlas agent bounded-live-readiness`).

No protected runtime safety boundary changed during the release cutover.

## Included Security Hardening

- Public/trust docs drift is now continuously validated against release metadata,
  reducing the risk of stale current-public labels or contradictory candidate-state
  claims.
- Provider artifact golden tests preserve pilot-module behavior through the
  CAND-014 extraction.
- Bounded live autonomy readiness is evaluated through a deterministic local gate
  that records evidence only; it does not submit orders or claim live readiness.
- All existing deterministic risk, approval, kill-switch, dead-man heartbeat, and
  audit hash-chain requirements remain in place.

## Included Provider Evidence Tooling

- Provider and broker evidence remains local, read-only, and non-authorizing.
- No provider or broker execution was enabled by this release.
- CAND-014 extraction is strictly limited to the pilot module; broader provider
  refactoring remains unauthorized.

- The dashboard remains read-only and zero-secret.
- CAND-006 remains simulated-only.
- CAND-007 remains no-live-submit.

- CAND-009 runtime atomic-write behavior remains unchanged.
- CAND-010 remains a static regression guard only.
- CAND-011 remains a type-safety cleanup only.
- CAND-012 remains a static checker/test/docs addition only.

## Known Limitations

- The project remains paper-first and safety-gated.
- External security review is recommended before funded use.
- Live operation requires separate explicit configuration and approvals.
- This release does not establish live-trading readiness.
- CAND-015 clean-room independence caveat: implementation and implementation
  review were performed by the same agent system, so a clean-room independent
  review is recommended before treating the bounded-live-readiness gate as fully
  closed for any future execution path.
- A full `pytest` run or full `bash scripts/release_check.sh` may exceed local
  interactive timeouts. The `--quick` gate and all targeted checks pass.
