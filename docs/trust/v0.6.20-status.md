## Scope

`v0.6.20` contains CAND-012, a candidate-chain consistency guard. It adds a
deterministic local read-only checker for release-metadata / candidate-chain
JSON / candidate-chain Markdown consistency, fixture-based tests, dev/CI
integration, and checks-reference documentation. It also remediates pre-existing
stale public-release doc claims and stale test expectations. No protected
runtime safety boundary changed during the release cutover.

## Included Security Hardening

- Release metadata and candidate-chain docs are now continuously validated by a
  deterministic local checker, reducing the risk of premature release claims or
  public-docs drift in future releases.
- All existing deterministic risk, approval, kill-switch, dead-man heartbeat,
  and audit hash-chain requirements remain in place.
- No protected runtime safety boundary changed during the release cutover.

## Included Provider Evidence Tooling

- Provider and broker evidence remains local, read-only, and non-authorizing.
- No provider or broker execution was enabled by this release.

## Assurance and CI Evidence

- Version, release metadata, forbidden-claim, public-doc, candidate-chain,
  trust, development, CI-parity, and quick release gates are required.
- Release assurance remains read-only and does not publish packages.

## Auto-Updater Status

- Auto-updater verification resolves `v0.6.20` through the GitHub tag/release.
- It does not install packages, call providers, touch brokers, or enable trading.

## Safety Defaults

- Live trading is disabled by default.
- Live submit is disabled by default.
- Provider execution is disabled by default.
- Broker execution is disabled by default.
- `RiskManager`, approval gates, the kill switch, and audit hash-chain controls
  remain required.
- No real notifications are sent and no orders are generated or submitted by
  the release evidence workflows.
- No real human approval is created or recorded by the release evidence
  workflows.
- The dashboard remains read-only and zero-secret.
- CAND-006 remains simulated-only.
- CAND-007 remains no-live-submit.
- CAND-008 remains evidence-only, simulated-only, and non-executing.
- CAND-009 runtime atomic-write behavior remains unchanged.
- CAND-010 remains a static regression guard only and does not change runtime
  behavior.
- CAND-011 remains a type-safety cleanup only and does not change kill-switch
  decision logic, fail-closed behavior, heartbeat/deadman behavior, `RiskManager`,
  or audit hash-chain.
- CAND-012 is a static checker/test/docs addition only and does not change
  runtime safety decision logic.
- This release makes no live-readiness, autonomous-live-readiness,
  profitability, absolute-safety, or future-performance claim.
- This software is not financial advice.

## Known Limitations

- The project remains paper-first and safety-gated.
- External security review is recommended before funded use.
- Live operation requires separate explicit configuration and approvals.
- This release does not establish live-trading readiness.
- A full `pytest` run or full `bash scripts/release_check.sh` may exceed local
  interactive timeouts. The `--quick` gate and all targeted checks pass.
