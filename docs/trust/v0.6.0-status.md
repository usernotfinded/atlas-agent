## Included Capabilities

- Backtesting foundation with strategy interface, registry, and validation harness.
- Backtesting strategy pack with `buy_and_hold`, `moving_average_cross`, and `rsi_mean_reversion`.
- Backtest report summaries in JSON and Markdown.
- Local report generator for daily, weekly, and ad-hoc Markdown/JSON reports.
- Reflection artifact foundation with create, list, show, submit, approve, reject, archive CLI.
- Skill candidate pipeline with manual-only promotion into a local skill library.
- Learning suggestion pipeline with advisory-only execution policy.
- Dashboard data layer and read-only static HTML UI.
- Safe notification foundation with dry-run default, Slack webhook, and redaction.
- Broker roadmap with guarded adapter status and fail-closed guards.
- v0.6.0 readiness audit and capability inventory.

## Included Security Hardening

- Fail-closed broker guards (`guard_submit`, `guard_sync`) for unsupported/disabled/placeholder brokers.
- Broker support inventory with explicit status for PaperBroker, Alpaca, Binance, CCXT, and IBKR.
- Read-only dashboard UI with no secrets, no trading controls, and no external assets.
- Notification redaction and dry-run-by-default transport.
- Reflection, skill, and learning artifact safety boundaries (offline, research-only, manual review).
- All prior v0.5.9.5 security hardening remains in place.

## Auto-Updater Status

Auto-updater delivery for `v0.6.0` is verified against the GitHub release/tag.
The updater check is dry-run oriented and must not install packages, call
providers, touch brokers, enable trading, or require credentials.

## Safety Defaults

- Live trading is disabled by default.
- Live submit is disabled by default.
- Provider execution is disabled by default.
- Broker execution is disabled by default.
- Risk gates remain deterministic.
- Passing risk gates does not mean automatic live execution.
- Human approval remains required.
- Telegram/remote control is disabled by default and operator-gated.

## Known Limitations

- PyPI was not published for `v0.6.0`.
- The project remains safety-first and paper-first.
- Live trading requires explicit local operator configuration.
- External security review is still recommended before funded use.
- Telegram/remote approval is not enabled by default.
- This software is not financial advice.
- Autonomous trading is not claimed.
