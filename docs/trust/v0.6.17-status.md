## Scope

`v0.6.17` contains CAND-009, a safety-state persistence hardening candidate. It
replaces fixed `<target>.tmp` writes in `heartbeat.py`, `deadman.py`,
`kill_switch.py`, and `state.py` with a stdlib-only unique same-directory
atomic-write helper, preserving file formats, public APIs, permissions, and
fail-closed behavior. The release also documents the existing read-only `doctor`
CLI command in the command-compatibility fixture.

## Included Security Hardening

- Safety-state persistence uses unique temporary filenames, removing temp-name
  collision risk while preserving atomic `replace` semantics.
- All existing deterministic risk, approval, kill-switch, dead-man heartbeat,
  and audit hash-chain requirements remain in place.
- No protected runtime safety boundary changed during the release cutover.
