## Scope

`v0.6.22` contains one released candidate:

- **CAND-016:** Release-Maintenance Drift Hardening — extends
  `scripts/check_public_docs_consistency.py` with a metadata-driven reverse-drift
  roadmap guard that flags already-released candidates listed as candidate-entry
  bullets under the next-planned planning-line section of
  `docs/autonomy-roadmap.md`, plus deterministic tests. Retains the
  metadata-driven release-assurance demo default.

No protected runtime safety boundary changed during the release cutover.

## Included Security Hardening

- Release-maintenance drift is now continuously validated against release
  metadata, reducing the risk of mislabeled roadmap candidate sections after a
  cutover.
- The reverse-drift guard complements the existing forward-drift check to keep
  `docs/autonomy-roadmap.md` consistent with the candidate-chain JSON.
- All existing deterministic risk, approval, kill-switch, dead-man heartbeat, and
  audit hash-chain requirements remain in place.

## Known Limitations

- The project remains paper-first and safety-gated.
- External security review is recommended before funded use.
- Live operation requires separate explicit configuration and approvals.
- This release does not establish live-trading readiness.
- CAND-015 clean-room independence caveat: implementation and implementation
  review were performed by the same agent system, so a clean-room independent
  review is recommended before treating the bounded-live-readiness gate as fully
  closed for any future execution path.
- A full `pytest` run or full `bash scripts/release_check.sh` may exceed local
  interactive timeouts. The `--quick` gate and all targeted checks pass.
