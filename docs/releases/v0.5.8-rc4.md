# v0.5.8rc4 Release Notes

> **Not financial advice.** Atlas Agent is a software tool, not a financial advisor. Trading involves significant risk of loss.

## What this release candidate is

`v0.5.8rc4` is a **release candidate**, not a stable final release. It exists so reviewers and maintainers can verify the v0.5.8 feature set, docs, and safety posture before a stable tag is created.

**This is not a commitment to release.** Stable v0.5.8 will only be tagged after explicit maintainer review and approval.

## What changed since v0.5.8rc3

### Fixed

- **Superseded already-consumed `v0.5.8rc3`.** The `v0.5.8rc3` tag points to the RC3 cutover commit before the CI checkout/clean-install fix. Because moving or force-pushing release candidate tags is not allowed, `v0.5.8rc3` is treated as a historical superseded candidate. `v0.5.8rc4` now represents the active corrected RC line.
- **Active RC checker advanced to RC4.** The cutover checker now verifies only the active `v0.5.8rc4` tag against HEAD while allowing older RC tags (`v0.5.8rc1`, `v0.5.8rc2`, `v0.5.8rc3`) to exist without requiring them to match current HEAD.

### Historical context

- `v0.5.8rc1` was tagged correctly at `0d9fb23aaf9bd708b142597d993307f8ecbaa98d`.
- `v0.5.8rc2` was tagged incorrectly and points to the same commit as `v0.5.8rc1`. It remains untouched as a historical record.
- `v0.5.8rc3` points to `2831f43a3c2b5cf1f09a7ed75eac779a7dfcf254` and remains untouched as a historical record.
- `v0.5.8rc4` supersedes those candidates as the active release candidate under test.

## Safety posture

- **Not financial advice.** Atlas Agent does not imply profitability or trading correctness.
- **Not proof of trading safety.** Risk gates reduce unintended actions but do not eliminate trading risk.
- **Not proof of profitability.** No backtest, simulation, or paper result guarantees real-money performance.
- **Not production trading ready.** Live trading is disabled by default and requires explicit multi-gate opt-in.
- **Live trading** remains disabled by default.
- **Provider execution** remains locked (governed by risk manager and artifact policy, not a runtime API block).
- **Broker execution** remains blocked unless explicit opt-in gates pass.
