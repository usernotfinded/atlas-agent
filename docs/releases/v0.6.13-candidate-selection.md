## Candidate Selection Rules

A candidate is eligible for the `v0.6.13` planning line only if **all** of the
following are true:

1. **No live trading enablement.** It must not make live trading the default,
   remove fail-closed configuration requirements, or weaken live-mode gates.
2. **No broker/provider execution enablement.** It must not enable broker order
   submission, real provider/LLM execution, or direct AI-to-broker paths.
3. **No secrets or credentials.** It must not add real broker credentials,
   provider API keys, account IDs, portfolio values, or private financial data.
4. **No unsafe marketing claims.** It must not claim that returns are
   guaranteed, that trading is without risk, that live trading is safe, that
   production trading readiness exists, or that autonomous trading readiness
   exists.
5. **Protected runtime boundaries preserved.** Changes under
   `src/atlas_agent/config`, `src/atlas_agent/brokers`,
   `src/atlas_agent/execution`, `src/atlas_agent/safety`, or
   `src/atlas_agent/risk` are avoided unless explicitly justified, narrowly
   scoped, fully tested, and separately approved.
6. **Small candidate batches.** Candidates should be small, reviewable, and
   independently valuable.
7. **Checker/test/doc coverage.** Every candidate must have deterministic
   checker and/or test coverage, and docs updates where the change is visible to
   reviewers.
8. **Green CI before completion.** The candidate must pass `./scripts/dev_check.sh`,
   `./scripts/ci_check.sh`, and `./scripts/release_check.sh --quick` before it is
   considered complete.
9. **No release cutover side effects.** A candidate must not create tags,
   GitHub Releases, or PyPI publications, and must not bump the package/source
   version beyond `0.6.12`.
