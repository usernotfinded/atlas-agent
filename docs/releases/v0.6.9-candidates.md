# v0.6.9 Patch Candidate Selection

> **Not financial advice.** Atlas Agent is a software tool, not a financial advisor. Trading involves significant risk of loss.

> **Status:** planning — backtest/report quality candidates selected and implemented

`v0.6.8` is the current public GitHub release (tagged and published). `v0.6.9` is the
next planning line. This document selects the safe backtest-report quality
candidates for the `v0.6.9` patch release. It does **not** authorize version bumps,
tag creation, GitHub release creation, or PyPI publishing. Those steps require
explicit owner approval in a separate release-prep batch.

## Context

The v0.6.9 batch focuses on backtest engine hygiene, backtest report schema
contract, validation UX, and Markdown report readability. All candidates are
local-only, deterministic, and offline; no runtime trading, broker, provider,
execution, risk, approval, or kill-switch behavior changes.

## Selection criteria

A candidate is eligible for `v0.6.9` only if **all** of the following are true:

- It improves backtest determinism, report clarity, or validation trust.
- It is a small, backward-compatible fix or docs/checker improvement.
- It does not change runtime trading, broker, provider, risk, approval, or kill-switch behavior.
- It does not add new runtime features, broker adapters, or provider integrations.
- It does not require new secrets, credentials, or network calls.
- It is safe to implement directly on `main` without a feature branch.
- It does not weaken existing checks or safety boundaries.

## Safety boundaries

- Live trading remains disabled by default.
- Live submit remains disabled by default.
- Provider execution remains disabled by default.
- Broker execution remains disabled by default.
- Risk gates remain deterministic and unchanged.
- Approval queue behavior remains unchanged.
- Kill-switch behavior remains unchanged.
- Human approval remains required for live order flow.
- Credentials are not loaded unless explicitly configured.
- No provider APIs, broker APIs, or real webhooks are called from CI or local checks.
