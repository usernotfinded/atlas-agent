# Release v0.5.7.dev23 — Provider Execution Opt-In State Machine Skeleton

## Summary

Introduces the local-only Provider Execution Opt-In State Machine. This is a skeleton that defines deterministic state transitions for moving from `disabled` → `dry_run_only` → `manual_unlock_required` → `provider_call_allowed_but_not_implemented`. No real provider calls are made at any state.

## What Changed

### New Module

- `src/atlas_agent/research/provider_execution_state.py`
  - `VALID_STATES`: `disabled`, `dry_run_only`, `manual_unlock_required`, `provider_call_allowed_but_not_implemented`
  - `evaluate_provider_execution_state_transition()`: deterministic local transition rules
  - `build_provider_execution_state_dict()`: creates artifact with all 6 safety booleans hardcoded to `False`
  - `safe_validate_provider_execution_state_data()`: strict validation with fail-closed behavior
  - `validate_provider_execution_state_artifact()`: detailed check-by-check validation
  - `replay_provider_execution_state()`: rebuilds from source dry-run and compares hashes
  - `iter_provider_execution_state_artifacts()`: safe iteration with invalid sentinels

### New CLI Commands

- `atlas research provider-execution-state DRY_RUN_ID --to STATE`
- `atlas research provider-execution-state-list`
- `atlas research provider-execution-state-show ID`
- `atlas research provider-execution-state-validate ID [--strict]`
- `atlas research provider-execution-state-replay ID [--strict]`

### Session Integration

- `check-artifacts`: counts provider execution states, validates artifacts, detects impossible booleans
- `build_research_timeline`: nests `provider_execution_states` under each dry-run entry
- `build_dossier`: includes state summary and linked artifacts

### Demo Workflow

- Extended with state transitions after provider-execution-dry-run replay
- Timeline validation asserts `provider_execution_states` nested under correct dry-run

### Bug Fix

- Removed `choices=...` from `--to` argument to prevent argparse from leaking raw invalid values in error messages.

## Safety

- Provider execution state artifacts are local only.
- Transitions are local artifact transitions only.
- No provider call is made.
- No API keys are read.
- No network is used.
- No provider SDKs are imported.
- No trading signal is generated.
- No live trading authorization is created.
- Even `provider_call_allowed_but_not_implemented` does not allow real calls.
- Tampered state artifacts fail safely with generic error envelopes.
- All configless invariants preserved.
