# Release v0.5.7.dev26 — Provider Preflight Freeze Audit & Evidence Pack

## Summary

Introduces the Provider Preflight Freeze, a local-only artifact that consolidates the full 9-artifact provider-preflight chain into a single immutable, auditable envelope. The freeze answers whether the chain is frozen and internally consistent, which artifacts are present or missing, which hashes match or drifted, and which safety gates block provider execution. All 10 safety booleans and 9 no-action attestations are enforced as `False`.

## What Changed

### New Module

- `src/atlas_agent/research/provider_preflight_freeze.py`
  - `build_provider_preflight_freeze_dict()`: creates deterministic freeze artifact from a readiness report with hash, validation, command surface, boundary, and denylist manifests.
  - `create_provider_preflight_freeze()`: loads source readiness report, builds artifact, persists to `.atlas/research/<SYMBOL>/provider_preflight_freezes/<id>.json`.
  - `safe_validate_provider_preflight_freeze_data()`: strict validation with forbidden-fragment scan; returns safe invalid sentinel on failure.
  - `validate_provider_preflight_freeze_artifact()`: detailed check-by-check validation including denylist manifest safety.
  - `replay_provider_preflight_freeze()`: rebuilds from source readiness report, compares hashes, returns replay envelope.
  - `iter_provider_preflight_freeze_artifacts()`: safe iteration with invalid sentinels.
  - `summarize_provider_preflight_freeze_for_run()`: read-only summary without writing artifacts or events.
  - `_BOOLEAN_SAFETY_FLAGS`: 10 mandatory-false booleans covering provider state, execution state, and trading/action state.
  - `no_action_attestations`: 9 mandatory-false attestations covering provider calls, network, API keys, SDKs, trading signals, approvals, orders, broker, and live trading.
  - `denylist_manifest`: stores only safe metadata (profile name, fragment count, safety flags); never stores raw forbidden fragment strings.

### New CLI Commands

- `atlas research provider-preflight-freeze READINESS_REPORT_ID`
- `atlas research provider-preflight-freeze-list`
- `atlas research provider-preflight-freeze-show FREEZE_ID`
- `atlas research provider-preflight-freeze-validate FREEZE_ID`
- `atlas research provider-preflight-freeze-replay FREEZE_ID`
- `atlas research provider-preflight-freeze-summary RUN_ID`

All commands are configless, local-only, and do not call providers, read API keys, or authorize live trading.

### Session Integration

- `check-artifacts`: counts freeze artifacts, validates with safe validation, detects impossible booleans, duplicate IDs, forbidden fragments.
- `build_research_timeline`: indexes freeze artifacts by `source_provider_execution_readiness_report_id`, nests under each readiness report dict as `provider_preflight_freezes`.
- `build_dossier`: includes freeze count, summary, linked artifacts, missing links warning.

### Demo Workflow

- Extended with provider preflight freeze creation after provider-execution-chain-doctor (steps 49–55).
- Validates freeze artifact is denylist-clean (no forbidden fragments).
- Validates freeze contains all 10 `False` safety flags and 9 `False` no-action attestations.
- Timeline validation asserts freeze nested under correct readiness report.
- Replay validation asserts hash match.
- Summary validation asserts `provider_execution_allowed=false` equivalent.
- All 43 fake-atlas inline templates updated with freeze command handlers.

### Bug Fix

- `denylist_manifest` no longer stores raw forbidden fragment strings (`/Users/`, `Authorization`, `Bearer`, `APCA`, `SECRET`, `TOKEN`, `PASSWORD`, `API_KEY`, `sk-`, `broker.example.com`) inside freeze artifacts. Previously the manifest stored `forbidden_fragments_checked` as a list of raw strings, causing the artifact itself to contain forbidden fragments and fail validation/show/replay.

## Safety

- Freeze artifacts are local only.
- All 10 safety booleans are hardcoded `False`.
- All 9 no-action attestations are hardcoded `False`.
- No provider call is made.
- No API keys are read.
- No network is used.
- No provider SDKs are imported.
- No trading signal is generated.
- No approval is created.
- No pending order is created.
- No broker is touched.
- No live trading authorization is created.
- Freeze status reflects local chain integrity/completeness only; it is NOT trading confidence.
- Even `freeze_status=provider_preflight_frozen` does NOT imply provider execution is allowed.
- Tampered freeze artifacts fail safely with generic error envelopes.
- Summary command is strictly read-only and never writes artifacts or events.
- All configless invariants preserved.
