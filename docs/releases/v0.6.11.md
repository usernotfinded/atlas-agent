# Atlas Agent v0.6.11 Release Notes

> **Status:** current public release
> **Released:** 2026-06-15
> **Previous public release:** `v0.6.10`
> **Not financial advice.** Atlas Agent is a software tool, not a financial advisor. Trading involves significant risk of loss.

## Summary

v0.6.11 is a reviewer-facing docs, checker, diagnostic, dashboard/report, and packaging hygiene patch. It implements the seven selected `v0.6.11` candidates (CAND-001 through CAND-007). No runtime trading, broker, provider, execution, risk, approval, or kill-switch behavior has changed.

The annotated tag `v0.6.11` was created and pushed, and the GitHub Release `v0.6.11` was published, as part of the owner-approved public cutover. PyPI was not published.

## Changes

### CAND-001 — Post-release hardening and observability cleanup
- Refreshed stale version references across reviewer-facing docs, public-repo hygiene, and public-feedback checklists.
- Aligned `scripts/check_public_docs_consistency.py`, `scripts/check_trust_center.py`, and `scripts/check_reviewer_onboarding.py` with the `v0.6.10` public / `v0.6.11` planning state.
- Removed observability/logging inconsistencies found in the `v0.6.10` post-release audit.

### CAND-002 — Backtest/report dashboard usability follow-up
- Improved dashboard empty-state messaging so reviewers see clear guidance instead of blank output.
- Added export timestamp headers to dashboard Markdown/HTML exports.
- Improved backtest summary table column alignment for Markdown/HTML output.
- Preserved the read-only, local, zero-secret dashboard contract.

### CAND-003 — Broker/provider preflight diagnostics without enabling execution
- Added deterministic, redacted `atlas doctor` output showing configured (but not activated) broker/provider settings.
- Added credential-presence checks without revealing values.
- Added connection-readiness hints with no network calls, no broker connections, no provider API calls, and no orders placed.
- Covered by `tests/cli/test_doctor_cli.py` for human/JSON output, no-network guards, redaction, and blocked live execution.

### CAND-004 — Paper-trading workflow documentation and safe examples
- Created/refreshed `docs/paper-trading-guide.md` with step-by-step paper-trading setup, safety reminders, and annotated fail-closed example configurations.
- Added/refreshed `scripts/demo_paper_workflow.sh` as the canonical offline, paper-only demo.
- Added `examples/paper_trading_demo/config.toml` as a safe, fail-closed paper example.
- Verified by demo smoke checks and public-docs consistency tests.

### CAND-005 — Release/checker simplification after v0.6.10
- Archived historical version-specific release checkers from the `v0.5.x`/`v0.6.0`–`v0.6.8` era into `scripts/historical_release_checkers/`.
- Preserved active `scripts/check_v0610_release_prep.py` (post-release mode) and `scripts/check_v0611_planning.py`.
- Added `scripts/check_v0611_release_prep.py` and matching tests for `v0.6.11` release-prep state validation.
- Historical checker archive and evidence remain intact.

### CAND-006 — Test-suite performance and generated-artifact hygiene
- Profiled slow tests and marked subprocess-heavy integration tests for targeted runs.
- Isolated CLI test workspaces to prevent local-state flakiness.
- Batched focused CI pytest collection for faster feedback.
- Hardened `scripts/check_generated_artifacts.py` detection and guidance.
- Preserved complete pytest coverage; no tests were skipped or removed solely to improve speed.

### CAND-007 — User-facing quickstart and reviewer demo consolidation
- Consolidated overlapping quickstart/demo content across `README.md`, `docs/external-reviewer-walkthrough.md`, and `docs/reviewer-golden-path.md` into one canonical flow.
- Linked canonical paper/preflight references and the demo artifact index.
- Verified by `scripts/check_public_docs_consistency.py`, `scripts/check_reviewer_onboarding.py`, and `scripts/verify_readme_quickstart.py`.

## Safety Boundaries

- **Live trading** remains disabled by default.
- **Live submit** remains disabled by default.
- **Provider execution** remains disabled by default.
- **Broker order submission** remains blocked by `can_submit=false` for all non-paper brokers.
- **Credentials** are not loaded unless explicitly configured.
- **Trust** remains blocked.
- **Risk gates** remain unchanged.
- **Kill switch** and **approval queue** remain unchanged.
- **Diagnostics** remain read-only, no-network, and redacted.
- **Dashboard** remains read-only, local, and zero-secret.

## Non-Goals

- **PyPI was not published** for `v0.6.11`.
- **No new runtime features** — scope per SemVer 2.0.0.
- **No new broker adapters**.
- **No new provider integrations**.
- **No live trading default changes**.
- **No provider execution default changes**.
- **No automatic skill activation or learning execution**.
- **Autonomous trading is not claimed**.
- **No runtime trading behavior changed**.
- **No broker/provider/risk behavior changed**.

## Known Limitations

- Same as `v0.6.10`: Binance adapter is partial, CCXT adapter is disabled, IBKR adapter is a placeholder, strategy pack is limited to built-in strategies, reports and reflection artifacts are local-only.
- PyPI was not published for `v0.6.1`, `v0.6.2`, `v0.6.3`, `v0.6.4`, `v0.6.5`, `v0.6.6`, `v0.6.7`, `v0.6.8`, `v0.6.9`, `v0.6.10`, or `v0.6.11`.
