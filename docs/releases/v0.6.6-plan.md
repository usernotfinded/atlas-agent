# v0.6.6 Planning

> **Not financial advice.** Atlas Agent is a software tool, not a financial advisor. Trading involves significant risk of loss.

## Status

This document is a planning-only artifact created after the `v0.6.5` release.
`v0.6.5` is the current public GitHub release (tagged and published). `v0.6.6` is
not version-prepared, not tagged, and not released. This plan does not authorize
implementation, version bumps, tag creation, GitHub release creation, or PyPI
publishing.

## Context

The `v0.6.5` release cutover completed on 2026-06-07. Commit `670432b`
(`docs: prepare v0.6.5 release identity`) was tagged as `v0.6.5` and published as
a GitHub release after all required checks passed and owner approval was
provided.

This batch (Batch 73) performs post-release documentation and checker metadata
hygiene: updating public docs from version-prepared wording to
tagged-and-published wording, aligning release metadata in checkers, and
preparing this planning document for a future `v0.6.6` maintenance patch.

The `v0.6.6` release line is reserved for small, backward-compatible
maintenance improvements to docs, checkers, and public-release reference
consistency. No new runtime features, broker adapters, provider integrations, or
trading behavior changes are planned in this document.

## Rejected / Out-of-Scope Items

The following items are explicitly rejected for `v0.6.6`:

- **Live trading default changes** — Atlas remains disabled-by-default for live trading.
- **Broker execution enablement** — No broker adapter may be promoted to default live-submit enabled.
- **Provider execution unlock** — Provider adapters remain fail-closed and disabled by default.
- **Autonomous trading** — Not claimed or enabled.
- **PyPI publish** — Not approved in this planning batch; requires separate process.
- **Major dashboard rewrite** — Out of patch scope.
- **Large architecture rewrite** — Out of patch scope and too risky.
- **New broker adapters** — New features, not maintenance.
- **New strategy packs** — New features, not maintenance.
- **Automatic skill activation** — Skill candidates remain manually reviewed and offline.
- **Automatic learning execution** — Learning suggestions remain offline and manually reviewed.
- **Profitability or trading correctness claims** — Forbidden.
- **Financial advice wording** — Forbidden.
- **Version bump to 0.6.6** — Not in this batch.
- **Creation of `docs/releases/v0.6.6.md`** — Not in this batch.
- **Tag creation, GitHub release creation, or PyPI publish** — Not in this batch.

## Safety Boundaries

Regardless of which candidates are accepted later, these boundaries remain
non-negotiable for any `v0.6.6` work:

- Live trading remains disabled by default.
- Live submit remains disabled by default.
- Provider execution remains disabled by default.
- Broker execution remains disabled by default.
- Risk gates remain deterministic and unchanged.
- Approval queue behavior remains unchanged.
- Kill-switch behavior remains unchanged.
- Human approval remains required for live order flow.
- Credentials are not loaded unless explicitly configured.
- No provider APIs, broker APIs, or real webhooks are called from CI or local checks.
