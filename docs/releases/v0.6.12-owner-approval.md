# v0.6.12 Owner Approval Gate

> **Not financial advice.** Atlas Agent is a software tool, not a financial
> advisor. Trading involves significant risk of loss.

## Purpose

This document is the explicit owner-approval gate for the `v0.6.12` release. It
separates **release-prep approval** (version bump, release notes, trust status,
changelog, and checker integration) from **public cutover approval** (tag
creation, GitHub Release publication, and any distribution action).

## Release-prep approval

Release-prep approval authorizes the following changes on `main`:

- Bumping `pyproject.toml` and `src/atlas_agent/__init__.py` to `0.6.12`.
- Creating `docs/releases/v0.6.12.md` and `docs/trust/v0.6.12-status.md`.
- Updating `CHANGELOG.md` with a `[0.6.12]` entry.
- Updating public status docs (`README.md`, `SECURITY.md`, `docs/trust/README.md`,
  `docs/development/main-health.md`, `docs/public-launch-readiness.md`,
  `docs/release-checklist.md`).
- Creating `scripts/check_v0612_release_prep.py` and matching tests.
- Integrating the v0.6.12 release-prep check into local gates and CI.

Release-prep approval does **not** authorize any of the following:

- Creating the annotated tag `v0.6.12`.
- Creating or publishing the GitHub Release `v0.6.12`.
- Publishing to PyPI.
- Changing live trading, live submit, provider execution, broker execution,
  risk gates, approval gates, kill-switch behavior, or audit logic.

## Public cutover approval

Public cutover approval is required before any of the following actions:

- `git tag -a v0.6.12 -m "Atlas Agent v0.6.12"`
- `git push origin v0.6.12`
- `gh release create v0.6.12 ...`
- Any `twine upload` or PyPI publish step.

Cutover approval may only be granted when:

- [x] `v0.6.12` release-prep artifacts are present and valid.
- [x] `python3.11 scripts/check_v0612_release_prep.py --release-prep` passes.
- [x] `python3.11 -m pytest tests/test_v0612_release_prep.py -q` passes.
- [x] `./scripts/dev_check.sh` passes.
- [x] `./scripts/ci_check.sh` passes.
- [x] `./scripts/release_check.sh --quick` passes.
- [x] `python3.11 -m pip check` passes.
- [x] `git diff --check` and `git diff --cached --check` pass.
- [x] `python3.11 scripts/check_no_protected_staged.py` passes.
- [x] Protected runtime boundaries (`src/atlas_agent/config`, `brokers`,
      `execution`, `safety`, `risk`) show no unintended diff.
- [x] No secrets or credential-like strings are present in the diff.
- [x] Owner explicitly approves the cutover in this document or in a
      project-tracked approval record.
