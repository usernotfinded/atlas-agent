## Safety boundaries

All v0.6 additions preserve the existing safety posture:

- **Live trading** remains disabled by default.
- **Live submit** remains disabled by default.
- **Provider execution** remains disabled by default.
- **Broker execution** remains blocked by default unless explicit opt-in gates pass.
- **No autonomous live trading** path exists.
- **No financial advice** or profit guarantees are claimed.
- **No real credentials** are required for any default workflow.
- **No provider API calls** in tests.
- **No broker API calls** in tests.
- **Dashboard remains read-only** and does not expose secrets or trading controls.
- **Notifications remain disabled/dry-run by default** and never alter execution state.
- **Reflections, skills, and learning suggestions** are offline, research-only, and require manual review.
- **Fail-closed broker guards** block unsupported, disabled, and placeholder brokers.

## Known deferred items

The following remain deferred and are **not** blockers for v0.6.0:

- **PyPI publish** — not planned for v0.6.0.
- **Autonomous trading without human oversight** — explicitly out of scope.
- **New broker live submit expansion** — Binance remains partial, CCXT disabled, IBKR placeholder.
- **Provider execution unlock by default** — remains locked; governed by risk manager and artifact policy.
- **Multi-user cloud deployment** — not a v0.6.0 goal.
- **Advanced frontend framework dashboard** — dashboard remains static HTML with inline CSS.
- **Real LLM-driven reflections, skill mining, and nudges** — current implementations are static/templates.
- **Network benchmark data fetch** — SPY benchmark requires explicit local CSV data.

## Non-goals
