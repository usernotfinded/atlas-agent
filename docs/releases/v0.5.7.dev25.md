# Release v0.5.7.dev25 — Provider Execution Readiness Report & Chain Doctor

## Summary

Introduces the Provider Execution Readiness Report, a local-only artifact that scores the completeness and integrity of the research/provider-preflight chain (0–100). A read-only Chain Doctor diagnoses the full chain without creating artifacts or calling providers. All 10 mandatory safety booleans are surfaced in `safety_gate_summary` and must be `False` for a valid readiness score.

## What Changed

### New Module

- `src/atlas_agent/research/provider_execution_readiness_report.py`
  - `_BOOLEAN_SAFETY_FLAGS`: 10 mandatory-false booleans covering provider state, execution state, and trading/action state.
  - `_build_safety_gate_summary()`: sources all 10 flags directly from the source audit packet top-level fields.
  - `_compute_readiness_score()`: deterministic 0–100 scoring. Any true safety flag forces score 0 and `chain_invalid`.
  - `_map_score_to_readiness_status()`: maps score/health to `chain_review_ready`, `manual_review_required`, `chain_incomplete`, or `chain_invalid`.
  - `build_provider_execution_readiness_report_dict()`: creates artifact with `artifact_chain`, `chain_diagnostics`, `hash_diagnostics`, `safety_gate_summary`, `no_action_attestations`.
  - `create_provider_execution_readiness_report()`: loads source audit packet, builds artifact, persists to `.atlas/research/<SYMBOL>/provider_execution_readiness_reports/<id>.json`.
  - `safe_validate_provider_execution_readiness_report_data()`: strict validation with forbidden-fragment scan; returns safe invalid sentinel on failure.
  - `validate_provider_execution_readiness_report_artifact()`: detailed check-by-check validation.
  - `replay_provider_execution_readiness_report()`: rebuilds from source audit packet, compares hashes, returns replay envelope.
  - `run_provider_execution_chain_doctor()`: read-only diagnostic across the full chain for a run ID.
  - `iter_provider_execution_readiness_report_artifacts()`: safe iteration with invalid sentinels.

### New CLI Commands

- `atlas research provider-execution-readiness AUDIT_PACKET_ID`
- `atlas research provider-execution-readiness-list`
- `atlas research provider-execution-readiness-show ID`
- `atlas research provider-execution-readiness-validate ID [--strict]`
- `atlas research provider-execution-readiness-replay ID [--strict]`
- `atlas research provider-execution-chain-doctor RUN_ID`

All commands are configless, local-only, and do not call providers, read API keys, or authorize live trading.

### Session Integration

- `check-artifacts`: counts readiness reports, validates with safe validation, detects impossible booleans, duplicate IDs, forbidden fragments.
- `build_research_timeline`: indexes readiness reports by `source_provider_execution_audit_packet_id`, nests under each audit packet dict as `provider_execution_readiness_reports`.
- `build_dossier`: includes readiness report count, summary, linked artifacts, missing links warning.

### Demo Workflow

- Extended with readiness report creation after provider-execution-audit-replay (steps 43–50).
- Validates readiness report artifact contains all 10 `false` safety flags and denylist-clean output.
- Timeline validation asserts readiness reports nested under correct audit packet.
- Chain doctor validation asserts `chain_health` is safe.
- All 43 fake-atlas inline templates updated with readiness report command handlers.

### Bug Fix

- `_build_safety_gate_summary()` now includes all 10 mandatory boolean flags (`provider_enabled`, `network_enabled`, `credentials_loaded`, `provider_call_allowed`, `actual_provider_call_made`, `future_provider_execution_possible`, `trading_signal_generated`, `approval_created`, `pending_order_created`, `broker_touched`). Previously only 6 flags were included, causing valid chains to score 0 and report `chain_invalid`.

## Safety

- Readiness report artifacts are local only.
- All 10 safety booleans are hardcoded `False`.
- No provider call is made.
- No API keys are read.
- No network is used.
- No provider SDKs are imported.
- No trading signal is generated.
- No approval is created.
- No pending order is created.
- No broker is touched.
- No live trading authorization is created.
- Readiness score reflects local chain integrity/completeness only; it is NOT trading confidence.
- Even `readiness_score=100` does NOT imply provider execution is allowed.
- Tampered readiness report artifacts fail safely with generic error envelopes.
- Chain doctor is strictly read-only and never writes artifacts or events.
- All configless invariants preserved.
