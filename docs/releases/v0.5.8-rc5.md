# v0.5.8rc5 Release Notes

> **Not financial advice.** Atlas Agent is a software tool, not a financial advisor. Trading involves significant risk of loss.

## What this release candidate is

`v0.5.8rc5` is a **release candidate**, not a stable final release. It supersedes `v0.5.8rc4` because of reviewer-found issues and docs polish. It exists so reviewers and maintainers can verify the v0.5.8 feature set, docs, and safety posture before a stable tag is created.

**This is not a commitment to release.** Stable v0.5.8 will only be tagged after explicit maintainer review and approval.

## What changed since v0.5.8rc4

### Fixed

- **Reviewer golden-path smoke diagnostics bug.** The optional `release_check.sh --quick` step in `scripts/smoke_reviewer_golden_path.py` now correctly captures stdout and stderr from the subprocess. Previously the capture was discarded (`rc, out, err = subprocess.run(...).returncode, "", ""`). Captured output is now preserved in the JSON/step envelope with redaction applied.
- **README release notes link.** Updated from `docs/releases/v0.5.7.md` to `docs/releases/v0.5.8-rc5.md` in the current status banner.
- **README current RC wording.** Updated "Current Development Status" to state that `v0.5.8-rc5` is the current release candidate under reviewer validation and `v0.5.7` remains the latest stable public release.
- **External reviewer walkthrough numbering.** Fixed duplicated/misnumbered list items in "What to inspect in the repo" by switching to Markdown auto-numbering.

### Documentation

- **Documented local safe diagnostic commands.** Added `atlas memory doctor --json` and `atlas events doctor` to the README quickstart/commands section. These are local diagnostics with no broker submission, no live trading, and no credentials required.

### Safety

- No live trading, provider execution, broker execution, credential loading, tag publishing, package publishing, or GitHub release creation was performed.
- No protected runtime boundaries were changed.
- No network calls were added.
- No profitability claims were added.

## Safety posture

- **Not financial advice.** Atlas Agent does not imply profitability or trading correctness.
- **Not proof of trading safety.** Risk gates reduce unintended actions but do not eliminate trading risk.
- **Not proof of profitability.** No backtest, simulation, or paper result guarantees real-money performance.
- **Not production trading ready.** Live trading is disabled by default and requires explicit multi-gate opt-in.
- **Live trading** remains disabled by default.
- **Provider execution** remains locked (governed by risk manager and artifact policy, not a runtime API block).
- **Broker execution** remains blocked unless explicit opt-in gates pass.
