# Atlas Agent v0.6.1 Release Notes

> **Release Date:** 2026-06-06
> **Not financial advice.** Atlas Agent is a software tool, not a financial advisor. Trading involves significant risk of loss.

## Summary

v0.6.1 is a maintenance patch release that corrects documentation, hardens post-release verification, and improves release-process reliability. No new runtime features, broker adapters, or provider integrations are included.

This patch contains only backward-compatible bug fixes and doc corrections per Semantic Versioning 2.0.0.

## Maintenance Changes

### Post-Release Readiness Checker Dual-Mode / CI Coverage
- Added `--post-release` mode to `scripts/check_v060_readiness.py` so the v0.6.0 release can be validated after tag and GitHub release publication.
- Hardened `_check_github_release()` to treat `gh` authentication and network failures as warnings rather than errors, preventing CI flakiness while preserving real missing-release detection.
- Added the post-release readiness step to the CI `quick-gate` job so post-release validation runs on every push.

### v0.6 Post-Release Readiness Test Isolation
- Fixed `tests/test_v060_readiness.py` post-release tests to be deterministic and offline-safe by mocking `_check_v060_tag` and `_check_github_release` instead of relying on real local git tags or GitHub release visibility.

### Long-Running Check Timing and Diagnostics
- Added per-step elapsed timing and total elapsed summary to all local gate scripts (`dev_check.sh`, `ci_check.sh`, `research_check.sh`, `release_check.sh`).
- Added `scripts/check_runtime_diagnostics.py`, a read-only helper documenting expected check runtimes, focused subsets, and timeout triage guidance.

### v0.6.1 Patch Candidate Selection and Implementation
- Added `docs/releases/v0.6.1-candidates.md` and `docs/releases/v0.6.1-candidates.json` to formally select safe maintenance candidates.
- Implemented CAND-001 (plan doc refresh), CAND-002 (capability inventory labeling), CAND-003 (post-release readiness CI coverage), and CAND-005 (checks-reference cross-links).

### Capability Inventory Labeling Cleanup
- Corrected `docs/v0.6-capability-inventory.md` section header from "v0.6.1 — Backtesting Strategy Pack" to "v0.6.0 — Backtesting Strategy Pack" to accurately reflect that the strategy pack shipped in v0.6.0.

### Checks-Reference Cross-Link Polish
- Added cross-links in `docs/development/checks-reference.md` for `scripts/check_v061_candidates.py`, post-release readiness modes, and long-running check guidance.

## Safety Boundaries

- **Live trading** remains disabled by default.
- **Provider execution** remains disabled by default.
- **Broker order submission** remains blocked by `can_submit=false` for all non-paper brokers.
- **Credentials** are not loaded unless explicitly configured.
- **Trust** remains blocked.
- **Risk gates** remain unchanged.
- **Kill switch** and **approval queue** remain unchanged.

## Non-Goals

- **PyPI publish: not performed**.
- **No new runtime features** — patch scope per SemVer 2.0.0.
- **No new broker adapters**.
- **No new provider integrations**.
- **No live trading default changes**.
- **No provider execution default changes**.
- **No automatic skill activation or learning execution**.
- **Autonomous trading is not claimed**.

## Known Limitations

- Same as v0.6.0: Binance adapter is partial, CCXT adapter is disabled, IBKR adapter is a placeholder, strategy pack is limited to 3 built-in strategies, reports and reflection artifacts are local-only.
- GitHub release body for v0.6.0 may differ from `docs/releases/v0.6.0.md` by exactly one trailing newline. This is benign formatting with no user impact.
