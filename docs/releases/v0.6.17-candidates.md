# v0.6.17 Release Candidates

Status: released as the current public GitHub release on 2026-07-01. `v0.6.16`
is the historical previous public release. `v0.6.17` is a GitHub-only release;
PyPI was not published.

## Candidate acceptance

**CAND-009** is accepted into the `v0.6.17` candidate chain with acceptance
verdict `PASS` on 2026-06-30. `v0.6.17` was released on 2026-07-01 after a
separate release-readiness review. CAND-009 remains a safety-state persistence
hardening candidate; it does not enable live trading, live submit, order
placement, broker/provider execution, credential loading, or approval queue
mutation.

## Implemented

- **CAND-009:** Safety State Atomic-Write Hardening.
  - Adds `src/atlas_agent/safety/atomic_write.py`, a stdlib-only atomic-write
    helper that uses unique same-directory temporary files.
  - Migrates `src/atlas_agent/safety/heartbeat.py`,
    `src/atlas_agent/safety/deadman.py`, `src/atlas_agent/safety/kill_switch.py`,
    and `src/atlas_agent/safety/state.py` from fixed `<target>.tmp` writes to
    helper calls, preserving JSON formats, public APIs, and best-effort
    `chmod(0o600)` behavior.
  - Adds regression and concurrency coverage under `tests/safety/`.
  - Resolves the `doctor` CLI compatibility contract warning by adding
    `"doctor"` to `tests/fixtures/cli_command_contract.json`.
  - No live trading, live submit, broker order submission, real provider
    execution, credential loading, or order placement is enabled. The release
    includes the source/package version bump from `0.6.16` to `0.6.17`, an
    annotated `v0.6.17` tag, a GitHub Release, and no PyPI publication.

## Proposed

No additional candidates beyond CAND-009 are proposed for `v0.6.17`.
