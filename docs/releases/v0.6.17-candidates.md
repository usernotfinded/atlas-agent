Status: planning-only / not released. `v0.6.16` is the current public GitHub
release. The source/package version remains `0.6.16`, no annotated tag exists at

## Candidate acceptance

**CAND-009** is accepted into the `v0.6.17` candidate chain with acceptance
verdict `PASS` on 2026-06-30. Acceptance is documentation/governance acceptance
only. It does not authorize `v0.6.17` release cutover and does not enable live
trading, live submit, order placement, broker/provider execution, credential
loading, or approval queue mutation. CAND-009 remains a safety-state persistence
hardening candidate, not a live-trading capability. `v0.6.17` is still not
released; a release-readiness review is still required before any tag, GitHub
Release, PyPI publication, or source/package version bump.

## Implemented

- **CAND-009:** Safety State Atomic-Write Hardening.
  - Adds `src/atlas_agent/safety/atomic_write.py`, a stdlib-only atomic-write
    helper that uses unique same-directory temporary files.
  - Migrates `src/atlas_agent/safety/heartbeat.py`,
    `src/atlas_agent/safety/deadman.py`, `src/atlas_agent/safety/kill_switch.py`,
    and `src/atlas_agent/safety/state.py` from fixed `<target>.tmp` writes to
    helper calls, preserving JSON formats, public APIs, and best-effort
    `chmod(0o600)` behavior.
  - Adds regression and concurrency coverage under `tests/safety/`.
  - Resolves the `doctor` CLI compatibility contract warning by adding
    `"doctor"` to `tests/fixtures/cli_command_contract.json`.
  - No live trading, live submit, broker order submission, real provider
    execution, credential loading, or order placement is enabled. No version
    bump, tag, release, or PyPI publication.
