# Atlas Agent v0.6.4 Release Notes

> **Release Date:** 2026-06-07
> **Not financial advice.** Atlas Agent is a software tool, not a financial advisor. Trading involves significant risk of loss.

## Summary

v0.6.4 is a maintenance patch release containing docs, checker, test, and CI reliability improvements. No new runtime features, broker adapters, or provider integrations are included. No trading, broker, provider, execution, risk, approval, or kill-switch behavior has changed.

This patch contains only backward-compatible maintenance improvements per Semantic Versioning 2.0.0.

## Maintenance Changes

### CAND-001 — Main Health Release Metadata Drift Detection
- Hardened `scripts/main_health.py` release metadata checks by centralizing constants into a `ReleaseMetadata` dataclass with a `validate()` method that detects drift against local git tags and source version.
- Tests verify drift detection.

### CAND-002 — Release Assurance Artifact Hygiene
- Clarified generated-artifact checker messages and docs guidance for `artifacts/release_assurance/*` folders produced by `scripts/release_assurance.py`.
- Release-assurance packs are local-only evidence with deterministic cleanup instructions.

### CAND-003 — Package Distribution Checker Follow-up
- Hardened `--dry-run` path and missing-tool error handling in `scripts/check_package_distribution.py`.
- `--dry-run` produces a deterministic plan report; missing tooling yields actionable hints.

### CAND-004 — Release/Trust Docs Consistency
- Final consistency pass across release notes, trust center, and checker docs to remove remaining stale `v0.6.2` references and align examples to `v0.6.3`.

### CAND-006 — CI Deterministic Reliability
- Hardened deterministic ordering in `src/atlas_agent/research/provider_execution_readiness_report.py` by replacing `list(set(...))` with `sorted(set(...))` for artifact lists.
- Added regression tests to enforce deterministic patterns and verify pytest-xdist collection stability.

### CAND-007 — Generated Artifact Cleanup Ergonomics
- Enhanced `scripts/check_generated_artifacts.py` to emit copy-paste-ready `mv` backup commands for untracked local artifacts.
- No destructive cleanup is performed automatically.

### CAND-009 — PyPI Non-Publish Messaging Consistency
- Standardized PyPI non-publish messaging across README, release notes, trust center, and release readiness docs to consistently use "PyPI was not published".

### CAND-010 — v0.6.4 Release Prep Checker
- Added `scripts/check_v064_release_prep.py` and `tests/test_v064_release_prep.py` to validate v0.6.4 release prep state in both planning mode and release-prep mode.
- Supports text and JSON output with deterministic checks.

## Safety Boundaries

- **Live trading** remains disabled by default.
- **Provider execution** remains disabled by default.
- **Broker order submission** remains blocked by `can_submit=false` for all non-paper brokers.
- **Credentials** are not loaded unless explicitly configured.
- **Trust** remains blocked.
- **Risk gates** remain unchanged.
- **Kill switch** and **approval queue** remain unchanged.

## Non-Goals

- **PyPI was not published**.
- **No new runtime features** — patch scope per SemVer 2.0.0.
- **No new broker adapters**.
- **No new provider integrations**.
- **No live trading default changes**.
- **No provider execution default changes**.
- **No automatic skill activation or learning execution**.
- **Autonomous trading is not claimed**.
- **Release cutover required owner approval and passing gates**.
- **No runtime trading behavior changed**.
- **No broker/provider/risk behavior changed**.
- **Tag not yet created**.
- **GitHub release not yet created**.

## Known Limitations

- Same as v0.6.3: Binance adapter is partial, CCXT adapter is disabled, IBKR adapter is a placeholder, strategy pack is limited to 3 built-in strategies, reports and reflection artifacts are local-only.
- PyPI was not published for v0.6.1, v0.6.2, v0.6.3, or v0.6.4.
