# Atlas Agent v0.5.7.dev3

## Summary

This is a development tag focused on release verification, package smoke checks, internal architecture hardening, and redaction cleanup.

- This is a **development tag**, not a production live-trading guarantee.
- No financial outcome is promised.
- Live trading remains disabled by default and explicitly gated.

## Highlights

- **Clean-clone tag smoke verification** — `scripts/smoke_release_tag.sh` verifies the tag from a fresh clone after push.
- **Wheel/sdist package smoke verification** — `scripts/smoke_package_build.sh` builds and installs the wheel in a clean environment, then runs a paper-only workspace smoke.
- **Offline package smoke mode** — `--offline` / `--skip-build-deps-install` supports no-network environments where `python -m build` is already available.
- **Release candidate audit document** — `docs/release-candidate-audit-v0.5.7.dev2.md` records release gate results, smoke status, safety contracts, and known limitations.
- **Architecture/performance hardening** — CLI command extraction, shared JSONL helpers, shared redaction engine, optional SQLite memory index, broker sync parallelization, CSV cache invalidation, tool contract/runtime split, and schema caching.
- **Redaction constant cleanup** — `SECRET_MARKERS` is now defined only in `src/atlas_agent/redaction.py`; the audit compatibility wrapper imports it from the shared module.

## Safety and Compatibility Notes

- Live submit remains disabled by default and explicitly gated.
- Reconcile remains read-only and does not call `place_order` or `resolve_execution_broker("live")`.
- Broker submit behavior is not expanded.
- Markdown memory remains the source of truth.
- SQLite memory index is optional and does not replace Markdown files.
- AuditWriter remains separate for hash-chain and manifest safety.

## Upgrade Notes

Existing users should rerun the release check:

```bash

```

Optional post-tag verification (after the tag is pushed):

```bash
./scripts/smoke_release_tag.sh v0.5.7.dev3
```

Optional package smoke (if build dependencies are available):

```bash
./scripts/smoke_package_build.sh
# or offline mode:
./scripts/smoke_package_build.sh --offline
```

## Known Limitations

- This is a development tag, not a claim of production live-trading readiness.
- Real broker E2E/sandbox verification is not completed.
- Default package smoke may require PyPI/network for build dependency installation.
- Offline package smoke requires a Python where `python -m build` works.
- Market-order live submit still requires safe quote handling before it should proceed.
