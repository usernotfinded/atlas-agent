# Atlas Agent v0.6.0 Release Notes

> **Release Date:** 2026-06-05
> **Not financial advice.** Atlas Agent is a software tool, not a financial advisor. Trading involves significant risk of loss.

## Summary

v0.6.0 expands Atlas Agent from a release-hardened trading framework into a **research, learning, reporting, and strategy platform**. This release delivers the full v0.6.x roadmap: backtesting foundation, strategy pack, local report generator, reflection artifacts, skill candidates, learning suggestions, dashboard read-only views, broker roadmap with fail-closed guards, and a safe notification foundation.

Live trading remains disabled by default. No autonomous trading is enabled. No provider execution unlocks by default.

## Major Capabilities

### Backtesting Foundation & Strategy Pack
- Typed `BacktestStrategy` protocol with metadata, registry, validation, and benchmark abstraction
- Built-in strategies: `buy_and_hold`, `moving_average_cross`, `rsi_mean_reversion`
- Strategy parameter specs with coercion and fail-closed validation
- CLI parameter overrides for `atlas backtest run` and `atlas backtest validate`
- Local-only SPY benchmark abstraction requiring explicit local CSV data
- `atlas backtest list-strategies`, `describe`, `validate`, `run --report json|markdown`

### Local Report Generator
- Daily, weekly, and ad-hoc Markdown/JSON reports using only local data
- `atlas report generate --type daily|weekly|ad-hoc --format markdown|json`
- Sections: portfolio, backtest, research, risk, audit, system health
- Empty-data and missing-benchmark fallback renderers with explicit warnings

### Reflection, Skills & Learning Loop
- Local-first reflection artifacts with provenance, approval/rejection workflows
- Local-first skill candidates with manual-only promotion to a local skill library
- Local-first learning suggestions with reflection/skill provenance and advisory-only policy
- Full CLI lifecycles for create/list/show/submit/approve/reject/archive

### Dashboard
- Enhanced read-only local dashboard with structured sections
- System health, safety status, reports, backtests, reflections, skills, learning, audit, warnings
- Static rendering with no external assets, no provider calls, no broker calls

### Notifications
- Safe notification foundation with disabled, dry-run, and Slack webhook modes
- Secret redaction, structured delivery results, local audit storage
- `atlas notifications test/send` with `--transport` and `--severity`, defaulting to dry-run

### Broker Roadmap & Guards
- Broker support inventory (`BrokerSupportEntry`) for PaperBroker, Alpaca, Binance, CCXT, IBKR
- Fail-closed broker guard helpers (`guard_submit`, `guard_sync`)
- `atlas broker status` read-only CLI command
- No live submit enabled by default for any broker

## Safety Boundaries

- **Live trading** is disabled by default and requires explicit multi-factor opt-in.
- Provider execution remains disabled by default — no real LLM/provider calls are made by default.
- **Broker order submission** is blocked by `can_submit=false` for all non-paper brokers.
- **Credentials** are not loaded unless explicitly configured.
- **Trust** remains blocked — mock responses in safety workflows are explicitly not trusted.
- **Reflection, skill, and learning artifacts** are offline, research-only, and require operator review.
- **Reports** are local, offline, and research-only.
- **Dashboard** is strictly read-only and does not expose trading controls.
- **Notifications** do not alter trading, provider, broker, skill, or learning execution state.

## Non-Goals

- PyPI was not published.
- **Autonomous trading without operator supervision** is not enabled.
- **New broker live submit expansion** is not enabled — only PaperBroker can submit by default.
- **Provider execution unlock by default** — remains locked.
- **Multi-user cloud deployment** — not supported.
- **Advanced frontend dashboard** — the dashboard is static, local, and read-only.

## Known Limitations

- **Binance** adapter is partial (read-only market data).
- **CCXT** adapter is disabled.
- **IBKR** adapter is a placeholder.
- **Strategy pack** is limited to 3 built-in strategies.
- **Reports** are local-only; no cloud or email delivery.
- **Reflection/skill/learning** artifacts are local-only and require manual review.
