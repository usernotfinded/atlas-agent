# Release v0.5.7.dev47 — Pre-Release Hardening Sweep

## Summary

Disciplined pre-release hardening sweep focused on documentation consistency, README claim verification, and public-command audit. No execution behavior changes. No broker changes. No modifications to risk gates, config, or safety boundaries.

## What changed

- **Version consistency**: bumped to `v0.5.7.dev47` across `pyproject.toml`, `src/atlas_agent/__init__.py`, `README.md`, and `docs/release-checklist.md`.
- **README fixes**:
  - Fixed duplicate `### Development checks` heading.
  - Fixed stale release-notes link (was pointing to `v0.5.7.dev40.md`).
  - Updated current-status version references from `dev46` to `dev47`.
- **Verification script hardening** (`scripts/verify_readme_quickstart.py`):
  - Fixed `str.find()` false-negative bug: now checks ALL occurrences of forbidden claims via `re.finditer`, so a negative-context mention earlier in the file no longer masks a positive claim later.
  - Hardened secret detection in bash blocks: no longer skips lines containing `#`; instead checks that credential-like tokens do not appear as unquoted values.
  - Added explicit check for profitability/trading-correctness limitation wording.
- **Public docs consistency script** (`scripts/check_public_docs_consistency.py`):
  - New deterministic local script scanning README and selected public docs.
  - Checks forbidden positive claims, forbidden fragments, secret-like patterns, unsafe commands, stale version references, and missing core safety wording.
  - Fails closed with non-zero exit on any unsafe content.
- **Public docs consistency tests** (`tests/test_public_docs_consistency.py`):
  - 16 tests covering pass-on-current-docs, rejection of unsafe claims, acceptance of negative safety wording, rejection of absolute paths, forbidden fragments, secret-like examples, and unsafe commands.
- **Release checklist hardening** (`docs/release-checklist.md`):
  - Added `verify_readme_quickstart.py` to required validation commands.
  - Added `check_public_docs_consistency.py` to required validation commands.
  - Added explicit staging reminder: do not use `git add .`, do not stage `.atlas/`, `.env`, `.env.atlas`, `memory/`, `build/`, `dist/`, `*.egg-info/`, generated reports, credentials, local logs, scratch files.
  - Updated smoke tag script version references to `dev47`.
- **Cumulative release map** (`docs/releases/v0.5.7-dev40-to-dev47-summary.md`):
  - Concise factual summary of safety/documentation progression from dev40 to dev47.

## What did not change

- No execution behavior changes.
- No broker adapter changes.
- No modifications to risk gates.
- No config system changes.
- No safety boundary changes.
- Live trading remains disabled by default.
- Provider execution remains locked.
- Trust remains blocked.
- No broker/order path enabled.
- No credentials loaded by default.
- No network calls in docs verification scripts.
