# v0.6.16 Release Candidates

Status: planning-only. `v0.6.16` is the next planning line; `v0.6.15` remains
the current public GitHub release. No source/package bump, tag, GitHub Release,
or PyPI publication is authorized by this file.

## Implemented

- **CAND-001:** Paper Autonomous Decision Loop and Shadow-Live Readiness Contract.
  - Adds `atlas agent autonomous-paper`, a deterministic, paper-only autonomous
    decision loop that runs on local sample/CSV data, routes proposed orders
    through `RiskManager` in paper mode, and writes local audit artifacts.
  - Adds `docs/autonomous-paper-loop.md`,
    `scripts/check_autonomous_paper_loop_contract.py`, and
    `tests/test_autonomous_paper_loop_contract.py`.
  - Adds `docs/shadow-live-readiness-contract.md`,
    `scripts/check_shadow_live_contract.py`, and
    `tests/test_shadow_live_contract.py`.
  - Updates `docs/bounded-live-autonomy-governance.md` and wires the new
    checkers/tests into `scripts/dev_check.sh` and `scripts/release_check.sh`.
  - No live trading, live submit, broker order submission, or real provider
    execution is enabled. No version bump, tag, release, or PyPI publication.

- **CAND-002:** Autonomous Paper Decision Quality Scorecard and Promotion Gate.
  - Adds `atlas agent autonomous-scorecard`, a deterministic offline scorecard
    that evaluates autonomous-paper `decisions.jsonl` and `manifest.json`
    artifacts and produces a promotion state.
  - Adds `src/atlas_agent/agent/autonomous_paper_scorecard.py`,
    `scripts/check_autonomous_paper_scorecard_contract.py`,
    `tests/test_autonomous_paper_scorecard.py`,
    `tests/test_autonomous_paper_scorecard_contract.py`,
    `docs/autonomous-paper-scorecard.md`, and
    `scripts/demo_autonomous_paper_scorecard.sh`.
  - Wires the new checker/tests into `scripts/dev_check.sh` and
    `scripts/release_check.sh`.
  - No live trading, live submit, broker order submission, or real provider
    execution is enabled. No version bump, tag, release, or PyPI publication.

- **CAND-003:** Execution-Neutral Autonomous Trading Kernel and Stateful Paper Runner.
  - Adds a reusable execution-neutral trading kernel that separates signal
    generation from deterministic fill simulation.
  - Adds a stateful paper runner that persists portfolio state to `--state-dir`,
    resumes from the last processed bar with `--resume`, prevents duplicate bar
    processing, and produces honest trading metrics.
  - Introduces paper-only CLI options `--state-dir`, `--resume`,
    `--initial-cash`, `--commission-bps`, `--slippage-bps`, and `--fill-timing`
    with conservative defaults.
  - Updates `docs/autonomous-paper-loop.md` with the CAND-003 design, semantics,
    metrics, and safety boundaries.
  - No live trading, live submit, broker order submission, or real provider
    execution is enabled. No version bump, tag, release, or PyPI publication.

- **CAND-004:** Autonomous Paper Trading Quality Gate.
  - Adds `atlas agent autonomous-paper-quality`, a deterministic, offline
    trading-quality gate that evaluates stateful paper artifacts produced by the
    CAND-003 runner.
  - Reads local `metrics.json`, `decisions.jsonl`, `fills.jsonl`, and runner
    `state.json`; optionally ingests the CAND-002 scorecard and a threshold
    policy.
  - Adds `src/atlas_agent/agent/autonomous_paper_quality.py`,
    `scripts/check_autonomous_paper_quality_contract.py`,
    `tests/test_autonomous_paper_quality.py`,
    `tests/test_autonomous_paper_quality_contract.py`, and
    `docs/autonomous-paper-quality-gate.md`.
  - Produces `trading-quality-gate.json` and `trading-quality-report.md`.
  - No live trading, live submit, broker order submission, or real provider
    execution is enabled. No version bump, tag, release, or PyPI publication.

- **CAND-005:** Shadow-Live Read-Only Fixture-First Comparison.
  - Adds `atlas agent shadow-live`, a deterministic, read-only, fixture-first
    comparison between a stateful paper run and a recorded local broker-like
    snapshot.
  - Consumes the CAND-004 `trading-quality-gate.json` and an optional richer set
    of paper artifacts (`state`, `metrics`, `decisions`, `fills`).
  - Adds `src/atlas_agent/agent/autonomous_paper_shadow_live.py`,
    `scripts/check_shadow_live_readonly_contract.py`,
    `tests/test_shadow_live_readonly.py`,
    `tests/test_shadow_live_readonly_contract.py`, and
    `docs/shadow-live-readonly-comparison.md`.
  - Produces `shadow-live-comparison.json` and `shadow-live-report.md`.
  - No broker API calls, credential loading, live submit, broker-state mutation,
    or live-readiness claim. No version bump, tag, release, or PyPI publication.

- **CAND-006:** Gated Submit Conformance Rehearsal.
  - Adds `atlas agent submit-conformance`, a deterministic, simulated-only,
    fixture-driven conformance rehearsal that consumes CAND-004 quality evidence,
    CAND-005 shadow-live comparison evidence, a hypothetical order-intent
    fixture, and simulated kill-switch, risk-envelope, and approval fixtures.
  - Adds `src/atlas_agent/agent/gated_submit_conformance.py`,
    `src/atlas_agent/agent/gated_submit_conformance_cli.py`,
    `src/atlas_agent/cli_bootstrap.py` pre-routing,
    `scripts/check_gated_submit_conformance_contract.py`,
    `tests/test_gated_submit_conformance.py`,
    `tests/test_gated_submit_conformance_cli.py`,
    `tests/test_gated_submit_conformance_contract.py`,
    `tests/test_gated_submit_conformance_import_trace.py`, and
    `docs/gated-submit-conformance.md`.
  - Produces `gated-submit-conformance.json` and
    `gated-submit-conformance-report.md`.
  - Does not submit orders, call brokers or providers, load credentials,
    instantiate runtime trading objects, mutate state, or indicate live
    readiness. No version bump, tag, release, or PyPI publication.

- **CAND-007:** Runtime Readiness Envelope Evaluation.
  - Adds `atlas agent readiness-envelope`, a deterministic, simulated-only,
    fixture-driven runtime readiness envelope evaluator that consumes CAND-004
    quality evidence, CAND-005 shadow-live comparison evidence, CAND-006 gated
    submit conformance evidence, and five static local policy fixtures.
  - Adds `src/atlas_agent/agent/runtime_readiness_envelope.py`,
    `src/atlas_agent/agent/runtime_readiness_envelope_cli.py`,
    `scripts/check_runtime_readiness_envelope_contract.py`,
    `tests/test_runtime_readiness_envelope.py`,
    `tests/test_runtime_readiness_envelope_cli.py`,
    `tests/test_runtime_readiness_envelope_contract.py`,
    `tests/test_runtime_readiness_envelope_import_trace.py`, and
    `docs/runtime-readiness-envelope.md`.
  - Produces `runtime-readiness-envelope.json` and
    `runtime-readiness-envelope-report.md`.
  - Does not submit orders, call brokers or providers, load credentials,
    instantiate runtime trading objects, mutate state, or indicate live
    readiness. The status `readiness_envelope_recorded` is evidence-recording
    status only. No version bump, tag, release, or PyPI publication.

- **CAND-008:** Operator Approval Gate.
  - Adds `atlas agent operator-approval-gate`, a deterministic, simulated-only,
    evidence-only operator approval gate that consumes CAND-004 quality evidence,
    CAND-005 shadow-live comparison evidence, CAND-006 gated submit conformance
    evidence, CAND-007 runtime readiness envelope evidence, and five CAND-008
    static local fixtures.
  - Adds `src/atlas_agent/agent/operator_approval_gate.py`,
    `src/atlas_agent/agent/operator_approval_gate_cli.py`,
    `scripts/check_operator_approval_gate_contract.py`,
    `tests/test_operator_approval_gate.py`,
    `tests/test_operator_approval_gate_cli.py`,
    `tests/test_operator_approval_gate_contract.py`,
    `tests/test_operator_approval_gate_import_trace.py`, and
    `docs/operator-approval-gate.md`.
  - Produces `operator-approval-gate.json` and
    `operator-approval-gate-report.md`.
  - Does not submit orders, call brokers or providers, load credentials,
    instantiate runtime trading objects, mutate state or approval queues, or
    indicate live readiness. The status `operator_gate_recorded` is
    evidence-recording status only and is not permission to submit orders. No
    version bump, tag, release, or PyPI publication.
