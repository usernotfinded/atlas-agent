# Release v0.5.7.dev30 — Provider Outbound Payload Preview Artifact

## Summary

This release adds local provider outbound payload preview artifacts and their integration into the research chain. No real provider execution is added. No network calls. No API keys. No provider SDKs. No broker changes. No credentials are loaded.

Batch 8.3 introduces a deterministic, auditable payload preview artifact that describes what a future outbound provider request would look like after minimization and redaction, without actually sending it. It records safe bounds only: provider/model IDs, request shape metadata, minimization/redaction summaries, field categories, and a payload hash. No raw prompt body, no raw request body, no raw response body, no credentials, no absolute paths, and no trading instructions are stored. Provider execution remains disabled.

## What Changed

### New Artifact

- `src/atlas_agent/research/provider_outbound_payload_preview.py`
  - `build_provider_outbound_payload_preview_dict()`: deterministic payload preview artifact construction from a provider credential boundary.
  - `create_provider_outbound_payload_preview()`: safe creation with validation, lineage, hash, and denylist scanning.
  - `safe_validate_provider_outbound_payload_preview_data()`: strict validation with invalid sentinel returns.
  - `validate_provider_outbound_payload_preview_artifact()`: detailed per-check validation.
  - `replay_provider_outbound_payload_preview()`: deterministic hash replay from source boundary.
  - `iter_provider_outbound_payload_preview_artifacts()`: safe listing with invalid sentinels for tampered items.
  - `summarize_provider_outbound_payload_preview_state()`: read-only summary without writing artifacts.
  - `provider_outbound_payload_preview_sha256()`: deterministic canonical hash excluding volatile fields.
  - Payload shape metadata: `request_family`, `message_count_estimate`, `max_output_tokens_policy`, `temperature_policy`, `tool_calls_allowed`, `streaming_allowed`, `system_prompt_included`, `user_prompt_included`, `raw_body_included`.
  - Payload minimization summary: `minimization_required`, `raw_prompt_omitted`, `artifact_links_used_instead_of_raw_text`, `hashes_used_instead_of_raw_content`, `only_bounded_summaries_allowed`, `broker_data_omitted`, `absolute_paths_omitted`.
  - Payload redaction summary: `redaction_required`, `redaction_applied`, `secrets_redacted`, `absolute_paths_redacted`, `broker_credentials_redacted`, `raw_exception_text_redacted`, `redaction_profile`, `raw_denylist_fragments_stored`.
  - Blocked fields: safe category labels only (`credentials`, `authorization_headers`, `absolute_paths`, `broker_credentials`, `raw_prompt_body`, `raw_provider_request_body`, `raw_provider_response_body`, `live_trading_instructions`, `order_submission_fields`).
  - 25 boolean safety flags including `provider_enabled`, `network_enabled`, `credentials_loaded`, `credential_value_present`, `credential_lookup_attempted`, `env_read_attempted`, `dotenv_loaded`, `provider_call_allowed`, `actual_provider_call_made`, `outbound_request_sent`, `future_provider_execution_possible`, `trading_signal_generated`, `approval_created`, `pending_order_created`, `broker_touched`, `payload_body_stored`, `raw_prompt_stored`, `raw_provider_request_stored`, `raw_provider_response_stored`, `absolute_paths_included`, `secrets_included`, `broker_credentials_included`, `trading_instruction_included`.
  - Impossible boolean detection: `provider_outbound_payload_preview_impossible_boolean` for tampered flags.
  - denylist-clean manifest: stores only safe metadata; never stores raw forbidden fragments.

### New CLI Commands

- `atlas research provider-payload-preview PROVIDER_CREDENTIAL_BOUNDARY_ID`
  - Create a payload preview artifact from a credential boundary.
- `atlas research provider-payload-preview-list`
  - List payload preview artifacts with safe sentinels.
- `atlas research provider-payload-preview-show PREVIEW_ID`
  - Show one payload preview artifact.
- `atlas research provider-payload-preview-validate PREVIEW_ID`
  - Validate a payload preview artifact with multiple checks.
- `atlas research provider-payload-preview-replay PREVIEW_ID`
  - Replay and compare deterministic hashes.
- `atlas research provider-payload-preview-summary RUN_ID`
  - Read-only summary without writing artifacts.

All commands remain configless, local-only, and read-only where applicable.
