# v0.5.8rc2 Release Notes

> **Not financial advice.** Atlas Agent is a software tool, not a financial advisor. Trading involves significant risk of loss.

## What this release candidate is

`v0.5.8rc2` is a **release candidate**, not a stable final release. It exists so reviewers and maintainers can verify the v0.5.8 feature set, docs, and safety posture before a stable tag is created.

**This is not a commitment to release.** Stable v0.5.8 will only be tagged after explicit maintainer review and approval.

## What changed since v0.5.8rc1

### Fixed

- **RC cutover verification gap.** The `v0.5.8rc1` post-tag checker fix exposed that the RC cutover checker was too strict: it failed when an RC tag already existed locally, even if it correctly resolved to HEAD. The checker now supports both:
  - **Pre-tag state** — the active RC tag does not yet exist.
  - **Post-tag matching state** — the active RC tag exists and resolves to the current HEAD.
  - It fails only if the active RC tag exists but points to a different commit.

### Historical context

- `v0.5.8rc1` was tagged correctly at `0d9fb23aaf9bd708b142597d993307f8ecbaa98d`.
- The RC1 tag remains untouched. It is a historical record and does not need to match current HEAD.
- RC2 supersedes RC1 as the active release candidate under test.

### What was NOT done by this cutover

- No tag was pushed.
- No package was published to PyPI.
- No GitHub release was created.
- No live trading was enabled.
- No provider execution was unlocked.
- No broker execution was unlocked.
- No credentials were loaded.
- No network calls were added.
- No protected boundaries were changed.

## Safety posture

- **Not financial advice.** Atlas Agent does not imply profitability or trading correctness.
- **Not proof of trading safety.** Risk gates reduce unintended actions but do not eliminate trading risk.
- **Not proof of profitability.** No backtest, simulation, or paper result guarantees real-money performance.
- **Not production trading ready.** Live trading is disabled by default and requires explicit multi-gate opt-in.
- **Live trading** remains disabled by default.
- **Provider execution** remains locked (governed by risk manager and artifact policy, not a runtime API block).
- **Broker execution** remains blocked unless explicit opt-in gates pass.
