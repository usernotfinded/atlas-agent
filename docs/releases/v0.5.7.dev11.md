# Atlas Agent v0.5.7.dev11

## Summary

- This is a development tag.
- Focus: research provider discovery (`atlas research providers`) and hardened provider metadata contract.
- It is not production live-trading readiness.
- No financial outcome is promised.

## Highlights

- Read-only research provider discovery command (`atlas research providers`).
- `ResearchProviderInfo` metadata model for deterministic and disabled LLM placeholder providers.
- CLI tests proving no API key reads, no network imports, no broker path usage, and safe output.
- Demo workflow now validates provider discovery output (`deterministic.local=true`, disabled `llm` placeholder).
- Failure tests for unsafe provider discovery output.
- No real LLM, API, or network behavior is enabled.

## Research Provider Boundary

- `deterministic` is the enabled and default provider.
- Explicit deterministic metadata:
  - enabled: true
  - default: true
  - local: true
  - network: false
  - requires_api_key: false
- `llm` is a disabled placeholder provider (`enabled=false`, `network=false`, `requires_api_key=false`).
- Unsupported providers fail closed.
- External/LLM providers are not enabled in this tag.
- The disabled LLM stub does not call APIs.
- The `atlas research providers` command does not read API keys, call providers, or perform network requests.
- No broker credentials are required.
- Provider selection does not authorize live trading.
- Provider output remains paper-only and analysis-only.
