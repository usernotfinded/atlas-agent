# Release v0.5.7.dev33 — Provider Response Schema Contract & Manual Review Gate

## Summary

This release adds local provider response schema contract artifacts and their integration into the research chain. No real provider response handling is added. No network calls. No API keys. No provider SDKs. No broker changes. No credentials are loaded.

Batch 8.6 introduces a deterministic, auditable provider response schema contract artifact that defines the allowed future provider response schema, rejected fields, validation rules, trust boundary, and manual review gate required before any future provider response can be interpreted. It is created from a provider request/response pairing and records safe schema metadata only: expected response shape, allowed response fields, rejected response fields, schema validation policy, unsafe content policy, manual review gate policy, trust boundary policy, trading separation policy, broker separation policy, response storage policy, response hash policy, review result policy, and future response artifact requirements. No raw provider response body, no raw request body, no raw prompt body, no credentials, no absolute paths, and no trading instructions are stored. Provider execution remains disabled. The manual review gate remains closed.

## What Changed

### New Artifact

- `src/atlas_agent/research/provider_response_schema_contract.py`
  - `build_provider_response_schema_contract_dict()`: deterministic schema contract artifact construction from a pairing.
  - `create_provider_response_schema_contract()`: safe creation with validation, lineage, hash, and denylist scanning.
  - `safe_validate_provider_response_schema_contract_data()`: strict validation with invalid sentinel returns.
  - `validate_provider_response_schema_contract_artifact()`: detailed per-check validation.
  - `replay_provider_response_schema_contract()`: deterministic hash replay from source pairing.
  - `iter_provider_response_schema_contract_artifacts()`: safe listing with invalid sentinels for tampered items.
  - `summarize_provider_response_schema_contract_state()`: read-only summary without writing artifacts.
  - `doctor_provider_response_schema_contract()`: read-only chain diagnostic.
  - `provider_response_schema_contract_sha256()`: deterministic canonical hash excluding volatile fields.
  - Policy substructures:
    - `expected_response_shape`: `response_family="bounded_json_object"`, `raw_text_response_allowed=False`, `structured_fields_required=True`, `manual_review_required=True`.
    - `allowed_response_fields`: safe category names only (e.g., `response_summary`, `reasoning_summary`, `safety_warnings`).
    - `rejected_response_fields`: safe category names only (e.g., `trading_signal`, `order_instruction`, `broker_instruction`).
    - `schema_validation_policy`: `unknown_fields_behavior="manual_review_required"`, `malformed_json_behavior="fail_closed"`, `unsafe_field_behavior="fail_closed"`.
    - `unsafe_content_policy`: `secret_like_content_behavior="fail_closed"`, `broker_action_behavior="fail_closed"`, `trading_action_behavior="manual_review_required"`.
    - `manual_review_gate_policy`: `manual_review_required_before_any_use=True`, `manual_review_gate_open=False`, `automatic_review_allowed=False`.
    - `trust_boundary_policy`: `provider_response_untrusted_by_default=True`, `schema_validation_does_not_make_response_trusted=True`.
    - `trading_separation_policy`: `response_is_not_trading_signal=True`, `response_cannot_create_pending_order=True`.
    - `broker_separation_policy`: `broker_live_bridge_allowed=False`, `order_routing_allowed=False`.
    - `response_storage_policy`: `raw_response_body_stored=False`, `bounded_summary_allowed=True`.
    - `response_hash_policy`: `response_hash_required_for_future_response=True`, `hash_does_not_imply_trust=True`.
    - `review_result_policy`: `review_result_required_before_future_use=True`, `review_result_can_mark_response_trusted=False`.
    - `future_response_artifact_requirements`: `future_response_artifact_required=True`, `future_response_cannot_authorize_trading=True`.
  - 40 boolean safety flags including `schema_contract_enabled`, `manual_review_gate_open`, `automatic_review_allowed`, `future_response_artifact_present`, `future_response_schema_validated`, `provider_response_received`, `provider_response_trusted`, `provider_response_can_create_orders`, `response_schema_allows_trading_signal`, `response_schema_allows_broker_call`, `raw_response_body_stored`, `raw_prompt_body_stored`, and all standard provider execution safety flags.
  - Impossible boolean detection: `provider_response_schema_contract_impossible_boolean` for tampered flags.
  - denylist-clean manifest: stores only safe metadata; never stores raw forbidden fragments.

### New CLI Commands

- `atlas research provider-response-schema-contract PROVIDER_REQUEST_RESPONSE_PAIRING_ID`
  - Create a response schema contract artifact from a pairing.
- `atlas research provider-response-schema-contract-list`
  - List schema contract artifacts with safe sentinels.
- `atlas research provider-response-schema-contract-show CONTRACT_ID`
  - Show one schema contract artifact.
- `atlas research provider-response-schema-contract-validate CONTRACT_ID`
  - Validate a schema contract artifact with multiple checks.
- `atlas research provider-response-schema-contract-replay CONTRACT_ID`
  - Replay and compare deterministic hashes.
- `atlas research provider-response-schema-contract-summary RUN_ID`
  - Read-only summary without writing artifacts.
- `atlas research provider-response-schema-contract-doctor RUN_ID`
  - Read-only chain diagnostic across schema contracts.

All commands remain configless, local-only, and read-only where applicable.

### Integration

- `check-artifacts` now counts and validates provider response schema contract artifacts with pairing, intake policy, and payload preview lineage/hash checks.
- `timeline` now links schema contract artifacts under their source pairing.
- `dossier` now includes schema contract artifacts in the research chain summary with `manual_review_gate_open=False` and `future_response_artifact_present=False`.
- Demo workflow extended with schema contract creation, list, show, validate, replay, summary, doctor, timeline lineage checks, and check-artifacts count validation.
