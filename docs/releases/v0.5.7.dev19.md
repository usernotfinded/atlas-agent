# Atlas Agent v0.5.7.dev19

## Summary

- This is a development tag.
- Focus: CI Offload & Thermal-Friendly Workflow (Batch 7.3).
- It is not production live-trading readiness.
- No financial outcome is promised.

## Highlights

- `.github/workflows/ci.yml` runs the full safety/test gate remotely on every push/PR to `main`.
- `.github/workflows/research-ci.yml` runs the research/sandbox gate on path-filtered changes.
- `scripts/ci_check.sh` provides local CI parity for developers who want to run the exact same command sequence as CI.
- Tiered local gates remain available:
  - `./scripts/release_check.sh --quick` for fast development loops.
  - `./scripts/release_check.sh --research` for research/sandbox work.
  - `./scripts/release_check.sh --full` for full local release gate.

## Recommended Workflow

Fast active-development loop:

```bash
./scripts/release_check.sh --quick
```

When touching research/sandbox:

```bash
./scripts/release_check.sh --research
```

Before push/tag locally:

```bash
./scripts/release_check.sh --full
```

For CI parity locally:

```bash
./scripts/ci_check.sh
```

## Safety and Compatibility Notes

- CI does not require secrets, broker credentials, or `.env.atlas`.
- CI does not call real LLM/API/network providers.
- CI complements local gates; it does not replace human review.
- Quick and research modes are developer convenience checks only.
- Full mode remains required before local release/tag.
- No changes to broker behavior, live trading gates, order routing, approval manager, risk manager, or config secret loading.
