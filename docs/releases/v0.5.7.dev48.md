# Release v0.5.7.dev48 — Release Candidate Readiness Report

## Summary

Added a local, read-only release candidate readiness report that summarizes whether Atlas Agent is ready to be presented as a sandbox/paper/preflight release candidate. The report is not a live-trading readiness claim and does not unlock provider execution, broker access, or live trading.

## What changed

- **Release candidate readiness report** (`src/atlas_agent/research/release_candidate_readiness.py`):
  - New research artifact type: `release_candidate_readiness_report`.
  - Inspects repository files and verification scripts deterministically and locally.
  - Checks version consistency, README quickstart verification, public docs consistency, provider safety dossier documentation, release checklist presence, forbidden claims scan, protected boundary cleanliness, and hard-false safety invariants.
  - Produces `readiness_status` and `readiness_score` — never claims live trading readiness or trading profitability.
  - Enforces 16 hard-false invariants (provider execution locked, trust blocked, broker/order path disabled, no credentials loaded, no network enabled, etc.).
  - Validates against unsafe positive claims, forbidden fragments, and absolute paths.
- **CLI commands**:
  - `atlas research release-candidate-readiness --symbol SYMBOL --json`
  - `atlas research release-candidate-readiness-list --json`
  - `atlas research release-candidate-readiness-show <id> --json`
  - `atlas research release-candidate-readiness-validate <id> --json`
  - `atlas research release-candidate-readiness-summary <id> --json`
  - `atlas research release-candidate-readiness-doctor <id> --json`
- **Research session integration**: wired into `check_research_artifacts()` and `build_research_timeline()`.
- **Documentation**:
  - `docs/release-candidate-readiness.md` — explains the report, what it checks, what it does not mean, and limitations.
  - `docs/release-checklist.md` — added readiness report commands.
  - `README.md` — updated version reference.
- **Tests**:
  - `tests/research/test_release_candidate_readiness.py` — 25+ tests covering create, validate, show, list, summary, doctor, missing docs, unsafe claims, forbidden fragments, absolute paths, hard-false invariants, and tamper resistance.
- **Version bump**: `0.5.7.dev47` → `0.5.7.dev48`.

## What did not change

- No execution behavior changes.
- No broker adapter changes.
- No modifications to risk gates.
- No config system changes.
- No safety boundary changes.
- Live trading remains disabled by default.
- Provider execution remains locked.
- Trust remains blocked.
- No broker/order path enabled.
- No credentials loaded by default.
- No network calls in the readiness report command.
