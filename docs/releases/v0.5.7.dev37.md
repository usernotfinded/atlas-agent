# Release v0.5.7.dev37

## Summary

Batch 9.0 adds the **Mock Provider Adapter / Simulated Response** system — a fully offline, configless artifact layer that simulates provider responses using a mock-only adapter with no network, no SDK, no credentials, and no real execution.

## New Module

- `src/atlas_agent/research/provider_mock_response_simulation.py`
- `src/atlas_agent/research/provider_adapter_interface.py`

## New CLI Commands

- `atlas research provider-mock-response-simulate ADAPTER_CONTRACT_ID`
- `atlas research provider-mock-response-list`
- `atlas research provider-mock-response-show SIMULATION_ID`
- `atlas research provider-mock-response-validate SIMULATION_ID`
- `atlas research provider-mock-response-replay SIMULATION_ID`
- `atlas research provider-mock-response-summary RUN_ID`
- `atlas research provider-mock-response-doctor RUN_ID`

## Safety Invariants

- Only `mock_adapter_used=True`, `mock_response_simulated=True`, `mock_only=True`.
- All execution/network/credential/broker booleans are `False`.
- `MockProviderAdapter.send()` always raises `ProviderAdapterDisabledError` (fail-closed).
- `MockProviderAdapter.simulate_response()` returns a deterministic offline mock.
- No real provider adapter is implemented.
- No provider SDK is imported.
- No HTTP client is imported.
- Network remains disabled.
- Credentials are not loaded.
- Provider calls remain disabled.
- No provider request is sent.
- No provider response is received or trusted.
- No approvals, pending orders, or broker calls are created.
- No trading signal is generated.
- No broker is touched.

## Tests
