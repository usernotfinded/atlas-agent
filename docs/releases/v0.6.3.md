# Atlas Agent v0.6.3 Release Notes

> **Release Date:** 2026-06-06
> **Not financial advice.** Atlas Agent is a software tool, not a financial advisor. Trading involves significant risk of loss.

## Summary

v0.6.3 is a hotfix release for package distribution checker changes ONLY. No new runtime features, broker adapters, or provider integrations are included. No trading, broker, provider, execution, risk, approval, or kill-switch behavior has changed.

This patch contains only backward-compatible bug fixes and process improvements per Semantic Versioning 2.0.0.

## Maintenance Changes

### Package Distribution Dependency Verification Fix
- Hardened runtime dependency verification in package distribution checks.
- Ensures `Requires-Dist` entries in wheel METADATA and sdist PKG-INFO match declared runtime dependencies.

### Pydantic Metadata Validation
- Added explicit validation that `pydantic` is declared as a `Requires-Dist` entry in wheel/sdist metadata.

### Separation of --no-deps Install Checks
- Split the wheel install verification into a dependency-free template resource check and a separate template initialization check.
- Dependency-free install no longer runs `atlas init`, preventing false failures when runtime dependencies are absent.

### Guarded Atlas Initialization
- `atlas init` from a wheel-installed package is only attempted when runtime dependencies are confirmed present.
- When dependencies are missing, the check reports the wheel metadata as correct and skips init gracefully.

### Twine / Dev-Extra Verification
- Confirmed `twine>=4.0` remains declared in `pyproject.toml` dev extras.
- CI and release gates continue to install `.[dev]` so twine check is available.

### Release-Gate Installation Path Improvements
- Release-gate and CI workflows install the package with dev extras, ensuring package distribution tooling is available during checks.
- Dry-run plan and real build both reflect the expected verification steps.

## Safety Boundaries

- **Live trading** remains disabled by default.
- **Provider execution** remains disabled by default.
- **Broker order submission** remains blocked by `can_submit=false` for all non-paper brokers.
- **Credentials** are not loaded unless explicitly configured.
- **Trust** remains blocked.
- **Risk gates** remain unchanged.
- **Kill switch** and **approval queue** remain unchanged.

## Non-Goals

- **PyPI was not published**.
- **No new runtime features** — patch scope per SemVer 2.0.0.
- **No new broker adapters**.
- **No new provider integrations**.
- **No live trading default changes**.
- **No provider execution default changes**.
- **No automatic skill activation or learning execution**.
- **Autonomous trading is not claimed**.
- **Release cutover required owner approval and passing gates**.
- **No runtime trading behavior changed**.
- **No broker/provider/risk behavior changed**.

## Known Limitations

- Same as v0.6.2: Binance adapter is partial, CCXT adapter is disabled, IBKR adapter is a placeholder, strategy pack is limited to 3 built-in strategies, reports and reflection artifacts are local-only.
- PyPI was not published for v0.6.1, v0.6.2, or v0.6.3.
