# Atlas Agent v0.6.20 Release Notes

> **Status:** current public GitHub release
> **Released:** 2026-07-02
> **Previous public release:** `v0.6.19`
> **Distribution:** GitHub-only; PyPI was not published
> **Not financial advice.** Atlas Agent is a software tool, not a financial
> advisor. Trading involves significant risk of loss.

## Summary

`v0.6.20` is a release-engineering and candidate-chain consistency release. It
includes **CAND-012: Candidate-Chain Consistency Guard**, which adds a
deterministic local read-only checker for release-metadata / candidate-chain
consistency, fixture-based tests, dev/CI integration, and checks-reference
documentation.

The release also remediates pre-existing stale public-release doc claims and
stale test expectations that referenced earlier releases.

This is a **GitHub-only release**. PyPI was not published. No live trading, live
submit, broker order submission, provider execution, credential loading, order
placement, pending-order creation, or approval queue mutation is introduced or
enabled. `atlas run --mode live` remains fail-closed.

## Included Candidate

- **CAND-012:** Candidate-Chain Consistency Guard
  - Adds `scripts/check_candidate_chain.py`, a deterministic local read-only
    checker that validates release-metadata / candidate-chain JSON /
    candidate-chain Markdown consistency.
  - Adds `tests/test_candidate_chain.py` with fixture-based coverage using
    `tmp_path` and subprocess invocation.
  - Integrates the checker into `scripts/dev_check.sh` and `scripts/ci_check.sh`
    after release metadata and version consistency checks.
  - Documents the checker in `docs/development/checks-reference.md`.
  - Validates candidate IDs, statuses, acceptance verdicts, release lines,
    tag/GitHub Release/PyPI claims, and candidate-chain-specific forbidden
    claims.
  - Uses exit codes `0` (pass), `1` (operational error), and `2` (validation
    failure) and prints `Candidate-chain consistency PASSED` on success.
  - Uses only the Python standard library, performs no network access, loads no
    credentials, imports no `atlas_agent` runtime code, and does not mutate
    repository files.
  - Remediates pre-existing stale public-release doc drift and stale test
    expectations so that the CI parity gate passes cleanly.
  - Does not change `RiskManager`, kill-switch, deadman, heartbeat, or audit
    hash-chain behavior, and does not enable live trading, live submit,
    broker/provider execution, credential loading, network access, or approval
    queue mutation.

## Known Non-Blocking Warnings

1. A full `pytest` run or full `bash scripts/release_check.sh` may exceed local
   interactive timeouts. The `--quick` gate and all targeted checks pass.
2. Historical candidate-chain docs for pre-`v0.6.19` releases still emit
   expected warnings from `check_candidate_chain.py` because they reflect
   historical planning state rather than current metadata.
