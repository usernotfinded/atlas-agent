# v0.5.8rc3 Release Notes

> **Not financial advice.** Atlas Agent is a software tool, not a financial advisor. Trading involves significant risk of loss.

## What this release candidate is

`v0.5.8rc3` is a **release candidate**, not a stable final release. It exists so reviewers and maintainers can verify the v0.5.8 feature set, docs, and safety posture before a stable tag is created.

**This is not a commitment to release.** Stable v0.5.8 will only be tagged after explicit maintainer review and approval.

## Current version

- Package version: `0.5.8rc3`
- Previous release candidates:
  - `v0.5.8rc1` (tagged correctly at `0d9fb23`)
  - `v0.5.8rc2` (accidentally tagged at the wrong commit; superseded and must not be moved or force-pushed)
- Latest stable release: `v0.5.7`

## What changed since v0.5.8rc2

### Fixed

- **Superseded incorrectly tagged `v0.5.8rc2`.** The `v0.5.8rc2` tag was accidentally created pointing to the same commit as `v0.5.8rc1` instead of the RC2 cutover commit. Because moving or force-pushing release candidate tags is not allowed, `v0.5.8rc2` is treated as a historical superseded candidate. `v0.5.8rc3` now represents the active corrected RC line.
- **Active RC checker generalized.** The cutover checker now verifies only the active RC tag against HEAD while allowing all older RC tags (`v0.5.8rc1`, `v0.5.8rc2`) to exist without requiring them to match current HEAD.

### Historical context

- `v0.5.8rc1` was tagged correctly at `0d9fb23aaf9bd708b142597d993307f8ecbaa98d`.
- `v0.5.8rc2` was tagged incorrectly and points to the same commit as `v0.5.8rc1`. It remains untouched as a historical record.
- `v0.5.8rc3` supersedes both as the active release candidate under test.

### What was NOT done by this cutover

- No tag was pushed.
- No package was published to PyPI.
- No GitHub release was created.
- No live trading was enabled.
- No provider execution was unlocked.
- No broker execution was unlocked.
- No credentials were loaded.
- No network calls were added.
- No protected boundaries were changed.

## Safety posture

- **Not financial advice.** Atlas Agent does not imply profitability or trading correctness.
- **Not proof of trading safety.** Risk gates reduce unintended actions but do not eliminate trading risk.
- **Not proof of profitability.** No backtest, simulation, or paper result guarantees real-money performance.
- **Not production trading ready.** Live trading is disabled by default and requires explicit multi-gate opt-in.
- **Live trading** remains disabled by default.
- **Provider execution** remains locked (governed by risk manager and artifact policy, not a runtime API block).
- **Broker execution** remains blocked unless explicit opt-in gates pass.
