# Release v0.5.7.dev43

## Batch 9.6 — Provider Safety Dossier Markdown Export

### Added
- `export_provider_safety_dossier_markdown()` in `src/atlas_agent/research/provider_safety_dossier.py`
  - Reads an existing provider safety dossier artifact
  - Validates the dossier (schema, hash, booleans, forbidden claims)
  - Rejects invalid, tampered, or incomplete-chain dossiers
  - Rejects dossiers with unsafe positive claims
  - Generates a safe local Markdown report with 8 required sections:
    1. Summary
    2. Chain
    3. Safety Invariants
    4. Trust Status
    5. Execution Status
    6. Broker/Order Status
    7. Validation Result
    8. Limitations
  - Scans output for forbidden fragments before writing
  - Fails closed on any validation failure
- CLI command: `atlas research provider-safety-dossier-export <dossier_id> --output <path> --format markdown`
  - Configless, non-authorizing
  - Validates output path is inside workspace
  - Returns JSON envelope with `--json`
- Demo workflow step: provider-safety-dossier create + export after final safety seal doctor
- Fake-atlas fixture support for `provider-safety-dossier` and `provider-safety-dossier-export`
