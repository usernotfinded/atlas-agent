# Atlas Agent v0.6.21 Release Notes

> **Status:** historical public GitHub release
> **Released:** 2026-07-07
> **Previous public release:** `v0.6.20`
> **Next public release:** `v0.6.22`
> **Distribution:** GitHub-only; PyPI was not published
> **Not financial advice.** Atlas Agent is a software tool, not a financial
> advisor. Trading involves significant risk of loss.

## Summary

`v0.6.21` was a release-engineering and public-docs drift coverage release. It
includes **CAND-013: Public/Trust Docs Drift Coverage Guard**, which updates
`scripts/check_public_docs_consistency.py` to use release metadata as the
current/next-planned authority and to catch stale trust README `(current public)`
labels, autonomy-roadmap candidate-state contradictions, and related public-docs
drift. It also includes **CAND-014 Phase 2: Provider Artifact Engine
Deduplication** and **CAND-015: Bounded Live Autonomy Readiness Gate** as
planning-only candidate work reflected in release documentation.

This release remediates stale public-release test expectations that referenced
`v0.6.20` as the current public release.

This is a **GitHub-only release**. PyPI was not published. No live trading, live
submit, broker order submission, provider execution, credential loading, order
placement, pending-order creation, or approval queue mutation is introduced or
enabled. `atlas run --mode live` remains fail-closed.

## Included Candidates

- **CAND-013:** Public/Trust Docs Drift Coverage Guard
  - Updates `scripts/check_public_docs_consistency.py` to use release metadata
    as the current/next-planned authority.
  - Adds detection for stale trust README `(current public)` labels on
    non-current releases.
  - Adds detection for autonomy-roadmap candidate-state contradictions against
    next-planned candidate-chain JSON.
  - Extends `tests/test_public_docs_consistency.py` coverage.
  - Documents the checker in `docs/development/checks-reference.md`.
  - Uses only the Python standard library, performs no network access, loads no
    credentials, and does not mutate repository files.
  - Does not change `RiskManager`, kill-switch, deadman, heartbeat, or audit
    hash-chain behavior, and does not enable live trading, live submit,
    broker/provider execution, credential loading, network access, or approval
    queue mutation.

- **CAND-014 Phase 2:** Provider Artifact Engine Deduplication
  - Pilot extraction of shared artifact mechanics behind
    `src/atlas_agent/research/artifact_engine.py`.
  - Preserves Phase 1 golden compatibility and all existing behavior.
  - No spec module creation and no changes to tests, fixtures, CLI,
    docs/releases, changelog, version, safety, broker, or other provider files
    beyond the pilot module.
  - Extraction beyond this pilot remains not authorized.

- **CAND-015:** Bounded Live Autonomy Readiness Gate
  - Adds `src/atlas_agent/agent/bounded_live_autonomy_readiness.py` engine and
    `src/atlas_agent/agent/bounded_live_autonomy_readiness_cli.py` configless
    CLI.
  - Registers `atlas agent bounded-live-readiness`.
  - Adds `scripts/check_bounded_live_autonomy_readiness_contract.py` static
    contract checker, tests, and fixtures.
  - Evaluates a 15-gate fail-closed sequence and records artifacts only when all
    gates pass.
  - Does not submit orders, call brokers/providers, load credentials, mutate
    runtime state, or claim live readiness.
  - `bounded_live_readiness_recorded` is evidence-recording status only and not
    permission to trade or authorization to submit orders.

## Safety Boundaries

- This release does not enable live trading or live submit.
- This release does not enable provider execution.
- No autonomous trading is claimed or enabled by this release.
- This release does not place, cancel, or flatten orders.
- This release does not create pending orders or mutate approval queues.
- This release does not call brokers, providers, or network endpoints.
- This release does not load credentials, secrets, or API keys.
- `RiskManager`, approval gates, the kill switch, the dead-man heartbeat, and
  the audit hash-chain remain unchanged and are not weakened.
- `atlas run --mode live` remains fail-closed (exits 2).

  decision changes.
- CAND-014 Phase 2 changes only research artifact engine internals; no runtime
  safety changes.
- CAND-015 is evidence-only and simulated-only; it does not submit orders or
  call brokers/providers.
- No live-readiness, autonomous-live-readiness, trading safety, broker
  endorsement, or permission to submit orders is claimed.
- No profitability, absolute-safety, or future-performance claim is made.

## Known Non-Blocking Warnings

1. A full `pytest` run or full `bash scripts/release_check.sh` may exceed local
   interactive timeouts. The `--quick` gate and all targeted checks pass.
2. Historical candidate-chain docs for pre-`v0.6.21` releases still emit
   expected warnings from `check_candidate_chain.py` because they reflect
   historical planning state rather than current metadata.
