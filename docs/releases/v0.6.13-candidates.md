`v0.6.12` is the current public GitHub release (tagged and published). `v0.6.13`
is the next planned release. This document records the conservative, hygiene-only
candidates completed for the `v0.6.13` planning cycle so far.

## Context

The `v0.6.13` planning line continues the hygiene-only trajectory established by
`v0.6.10`, `v0.6.11`, and `v0.6.12`: reviewer-facing documentation, deterministic
checkers, release-assurance hardening, trust-center transparency, and governance
alignment. All `v0.6.13` candidates are local-only, deterministic, and offline;
none change runtime trading, broker execution, provider execution, risk gates,
approval behavior, or kill-switch behavior.

## Selection criteria

A candidate is eligible for `v0.6.13` only if **all** of the following are true:

- It improves reviewer trust, release transparency, checker coverage, docs accuracy,
  or governance alignment.
- It is a small, backward-compatible docs/checker/test/workflow improvement.
- It does not change runtime trading, broker, provider, risk, approval, or kill-switch
  behavior.
- It does not add new runtime features, broker adapters, or provider integrations.
- It does not require new secrets, credentials, or network calls.
- It is safe to implement directly on `main` without a feature branch.
- It does not weaken existing checks or safety boundaries.

## Rejected / Out-of-Scope

The following remain out of scope for `v0.6.13` unless explicitly approved in a
future candidate:

- Bumping the package/source version to `0.6.13`.
- Creating an annotated `v0.6.13` tag.
- Creating or publishing a GitHub Release for `v0.6.13`.
- Publishing `v0.6.13` to PyPI.
- Enabling live trading, broker order submission, or real provider execution by default.
- Modifying protected runtime boundaries without explicit justification and tests.
- Autonomous live trading, unattended real-money execution, or direct AI-to-broker order submission.

## Safety boundaries

Unless explicitly approved in a future candidate, these boundaries remain unchanged
for `v0.6.13`:

- **Live trading** disabled by default.
- **Live submit** disabled by default.
- **Provider execution** disabled by default.
- **Broker order submission** blocked for non-paper brokers.
- **No credentials, secrets, or API keys** committed.
- **No claims** that live trading is safe, that risk is absent, that profit is
guaranteed, or that production/autonomous trading readiness exists.
