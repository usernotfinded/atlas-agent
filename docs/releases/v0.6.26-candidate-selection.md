## Released candidates

- **CAND-021 — Metadata-driven current-state test fixtures** — released on
  2026-07-13 as part of `v0.6.26`. Adds a shared `release_identity` pytest
  fixture from `docs/releases/release-metadata.json` and refactors current-state
  tests to use fixture values instead of hardcoded release identity. Static
  test/docs-only; no runtime, safety, broker, provider, credential, network, or
  trading changes.

### Why CAND-021 was eligible

- It is a test/docs-only addition; it does not implement, authorize, or enable
  any trading or execution path.
- It closes a concrete release-maintenance gap observed after the `v0.6.25`
  cutover: current-state tests can embed release-identity literals that become
  stale, and refactoring them to read from `release-metadata.json` reduces
  cutover drift.
- It uses only the Python standard library, performs no network mutation, loads
  no credentials, and does not mutate repository files.
- It preserves the CAND-014 single-module extraction boundary.
- It ships deterministic, offline tests.

## Selection rules

Any future `v0.6.27` candidate must satisfy the repository's candidate selection
rules before acceptance:

1. No live trading enablement — fail-closed defaults unchanged.
2. No broker/provider execution enablement — no direct AI-to-broker path.
3. No secrets or credentials committed.
4. No unsafe marketing claims — no promises of specific returns, eliminated
   risk, live readiness, or autonomous trading readiness.
5. Protected runtime boundaries preserved — `RiskManager`, approval queues, kill
   switch, heartbeat, deadman, audit hash-chain, live-submit opt-in, and
   `can_submit` are not bypassed.
6. Small, reviewable candidate batches that are independently valuable.
7. Deterministic test and documentation coverage.
8. Green verification before acceptance.
9. The CAND-014 provider-artifact extraction boundary is not broadened beyond
   `src/atlas_agent/research/provider_mock_response_final_safety_seal.py`.
