# Atlas Agent v0.6.10 Release Notes

> **Released:** 2026-06-13
> **Not financial advice.** Atlas Agent is a software tool, not a financial advisor. Trading involves significant risk of loss.

## Summary

v0.6.10 is a reviewer-facing docs, checker, report, and packaging hygiene patch. It implements the seven selected `v0.6.10` candidates (CAND-001 through CAND-007). No runtime trading, broker, provider, execution, risk, approval, or kill-switch behavior has changed.

This release contains only backward-compatible improvements per Semantic Versioning 2.0.0.

## Changes

### CAND-001 — Reviewer validation command guide refresh
- Refreshed reviewer walkthroughs, golden path, checklist, FAQ, and safety docs.
- Repaired broken internal links and stale safety-doc version references.
- Linked the reviewer golden path from the README and external walkthrough.

### CAND-002 — Dashboard/report UX polish after v0.6.9
- Fixed dashboard status-badge mapping for schema statuses.
- Added the HTML safety banner to Markdown dashboard exports.
- Improved empty/redacted diagnostics rendering.
- Sorted backtest runs deterministically by `run_id`.

### CAND-003 — Backtest report schema checker ergonomics
- Improved `scripts/check_backtest_report_schema.py` with `--json` output.
- Added per-status counts (`valid/invalid/legacy/unreadable`).
- Added `--fail-on-legacy` flag and a final summary.
- Added direct unit tests for the schema checker.

### CAND-004 — Template source-of-truth simplification
- Made `src/atlas_agent/templates/routine-trader/` the single canonical packaged copy.
- Removed the byte-identical root-level duplicate.
- Updated checkers, tests, and docs to reference the packaged copy.
- Preserved fail-closed defaults in `.env.example`, `risk.example.yaml`, and `brokers.example.yaml`.

### CAND-005 — Code inventory follow-up import/API classification
- Refreshed `docs/development/code-inventory-followups.md`.
- Re-verified deferred imports and improved API-classification clarity in `tests/test_code_inventory_imports.py`.

### CAND-006 — Release-assurance metadata cleanup
- Removed stale hardcoded v0.5.x-era copy-paste from `scripts/release_assurance.py`.
- Archived old tracked `artifacts/release_assurance/v0.5.9*` packs under `archive/`.
- Tightened `.gitignore` for generated local evidence.

### CAND-007 — Public docs link/reference hardening after README cleanup
- Fixed broken CHANGELOG links to v0.5.8 rc notes.
- Refreshed stale capability-inventory headers.
- Converted backticked doc paths to Markdown links.
- Extended the public-docs consistency scan list to cover reviewer-facing docs added since v0.6.8.

## Safety Boundaries

- **Live trading** remains disabled by default.
- **Live submit** remains disabled by default.
- **Provider execution** remains disabled by default.
- **Broker order submission** remains blocked by `can_submit=false` for all non-paper brokers.
- **Credentials** are not loaded unless explicitly configured.
- **Trust** remains blocked.
- **Risk gates** remain unchanged.
- **Kill switch** and **approval queue** remain unchanged.
