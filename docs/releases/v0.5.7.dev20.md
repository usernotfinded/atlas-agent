# Atlas Agent v0.5.7.dev20

## Summary

- This is a development tag.
- Focus: Disabled Provider Call-Plan Harness (Batch 7.4).
- It is not production live-trading readiness.
- No financial outcome is promised.

## Highlights

- `src/atlas_agent/research/provider_call_plan.py` creates local, auditable provider call-plan artifacts.
- Provider call-plan artifacts record what would be sent to a provider in the future, without calling any provider.
- All provider targets are disabled metadata only: `custom-openai-compatible`, `custom-chat-completions-compatible`, `custom-responses-compatible`, `manual-external-provider`.
- New configless research subcommands:
  - `atlas research provider-targets`
  - `atlas research provider-plan SANDBOX_ID --provider ID --model ID`
  - `atlas research provider-plan-list`
  - `atlas research provider-plan-show ID`
  - `atlas research provider-plan-validate ID`
  - `atlas research provider-plan-replay ID`
- Provider call plans are strictly plan-only: `provider_enabled=false`, `network_enabled=false`, `credentials_loaded=false`, `provider_call_allowed=false`.
- Timeline and dossier now link `run_id -> prompt -> sandbox_request -> provider_call_plan`.
- `check-artifacts` counts and validates provider call-plan artifacts.

## Safety and Compatibility Notes

- No real provider is called.
- No API keys are read or loaded.
- No network requests are made.
- No trading signals are generated.
- No live trading is authorized.
- Provider call plans are preparation-only artifacts for future provider integration.
- All new commands remain configless: no `AtlasConfig.from_env`, no `.env.atlas` secrets, no network calls.
