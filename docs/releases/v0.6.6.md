# Atlas Agent v0.6.6 Release Notes

> **Release Date:** 2026-06-09
> **Not financial advice.** Atlas Agent is a software tool, not a financial advisor. Trading involves significant risk of loss.

## Summary

v0.6.6 is a maintenance patch release containing docs consistency, checker infrastructure, and changelog hygiene improvements. No new runtime features, broker adapters, or provider integrations are included. No trading, broker, provider, execution, risk, approval, or kill-switch behavior has changed.

This patch contains only backward-compatible maintenance improvements per Semantic Versioning 2.0.0.

## Maintenance Changes

### CAND-001 — v0.6.6 Release Prep Checker Skeleton
- Created `scripts/check_v066_release_prep.py` and `tests/test_check_v066_release_prep.py` following the established v0.6.5 pattern.
- Supports planning mode (validates absence of v0.6.6 artifacts while source version remains `0.6.5`) and release-prep mode (validates presence of v0.6.6 artifacts after version bump).
- Supports text and JSON output with deterministic checks.

### CAND-002 — Checks-Reference Alignment for v0.6.5 Released State
- Updated `docs/development/checks-reference.md` to describe v0.6.5 release prep checks as historical/verified and note v0.6.6 as the next planning line.
- Prevents maintainer confusion from stale "current prep state" wording.

### CAND-003 — Public Docs Consistency Checker Expansion
- Expanded `scripts/check_public_docs_consistency.py` and `tests/test_public_docs_consistency.py` to validate that the README status line matches the current public release and warn on orphaned release notes not referenced in CHANGELOG.
- Low-cost automation that prevents recurring post-release docs drift.

### Batch 78 — Checker-Coverage Cleanup
- Aligned `docs/releases/v0.6.6-candidates.md` section headings with the v0.6.6 release-prep checker so the unsafe-candidates scan is genuinely active.
- Added missing CHANGELOG reference to `docs/releases/v0.5.8.1.md` so the orphan release-note warning clears.

## Safety Boundaries

- **Live trading** remains disabled by default.
- **Provider execution** remains disabled by default.
- **Broker order submission** remains blocked by `can_submit=false` for all non-paper brokers.
- **Credentials** are not loaded unless explicitly configured.
- **Trust** remains blocked.
- **Risk gates** remain unchanged.
- **Kill switch** and **approval queue** remain unchanged.

## Non-Goals

- **PyPI was not published**.
- **No new runtime features** — patch scope per SemVer 2.0.0.
- **No new broker adapters**.
- **No new provider integrations**.
- **No live trading default changes**.
- **No provider execution default changes**.
- **No automatic skill activation or learning execution**.
- **Autonomous trading is not claimed**.
- **Release cutover required owner approval and passing gates**.
- **No runtime trading behavior changed**.
- **No broker/provider/risk behavior changed**.
- **Tag not yet created**.
- **GitHub release not yet created**.

## Known Limitations

- Same as v0.6.5: Binance adapter is partial, CCXT adapter is disabled, IBKR adapter is a placeholder, strategy pack is limited to 3 built-in strategies, reports and reflection artifacts are local-only.
- PyPI was not published for v0.6.1, v0.6.2, v0.6.3, v0.6.4, v0.6.5, or v0.6.6.
