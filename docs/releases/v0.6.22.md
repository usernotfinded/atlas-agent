# Atlas Agent v0.6.22 Release Notes

> **Status:** current public GitHub release
> **Released:** 2026-07-13
> **Previous public release:** `v0.6.21`
> **Next planned release:** `v0.6.23`
> **Distribution:** GitHub-only; PyPI was not published
> **Not financial advice.** Atlas Agent is a software tool, not a financial
> advisor. Trading involves significant risk of loss.

## Summary

`v0.6.22` is a release-maintenance drift hardening release. It includes
**CAND-016: Release-Maintenance Drift Hardening**, a docs/checker/test-only
candidate that makes release-identity state metadata-driven. It adds a
reverse-drift roadmap guard to `scripts/check_public_docs_consistency.py` that
flags an already-released candidate listed as a candidate-entry bullet under the
next-planned planning-line section of `docs/autonomy-roadmap.md`, plus
deterministic tests. It also retains the metadata-driven release-assurance demo
default introduced during the `v0.6.21` post-release stabilization pass.

This release directly addresses the release-maintenance drift observed after the
`v0.6.21` cutover: stale next-planned tag guards, a stale demo default, and
roadmap candidate mislabeling caused by hardcoded version literals.

This is a **GitHub-only release**. PyPI was not published. No live trading, live
submit, broker order submission, provider execution, credential loading, order
placement, pending-order creation, or approval queue mutation is introduced or
enabled. `atlas run --mode live` remains fail-closed.

## Included Candidates

- **CAND-016: Release-Maintenance Drift Hardening**
  - Adds `_released_candidate_ids()` and
    `_check_autonomy_roadmap_released_candidates_not_next_planned()` to
    `scripts/check_public_docs_consistency.py`.
  - Flags already-released candidates listed as candidate-entry bullets under the
    next-planned planning-line section of `docs/autonomy-roadmap.md`.
  - Ignores prose cross-references to released candidates to avoid false
    positives.
  - Extends `tests/test_public_docs_consistency.py` with reverse-drift coverage.
  - Retains the metadata-driven release-assurance demo default in
    `scripts/demo_release_assurance_snapshot_bundle.sh`.
  - Uses only the Python standard library, performs no network access, loads no
    credentials, and does not mutate repository files.
  - Does not change `RiskManager`, kill-switch, deadman, heartbeat, or audit
    hash-chain behavior, and does not enable live trading, live submit,
    broker/provider execution, credential loading, network access, or approval
    queue mutation.
  - Does not broaden the CAND-014 provider-artifact extraction boundary, which
    remains strictly at
    `src/atlas_agent/research/provider_mock_response_final_safety_seal.py`.

## Safety Boundaries

- This release does not enable live trading or live submit.

- No autonomous trading is claimed or enabled by this release.
- This release does not place, cancel, or flatten orders.
- This release does not create pending orders or mutate approval queues.

- This release does not load credentials, secrets, or API keys.

  the audit hash-chain remain unchanged and are not weakened.
