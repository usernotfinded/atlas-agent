# Release v0.5.7.dev21 — Provider Execution Dry-Run Gate

## Summary
Adds a local, configless dry-run/preflight layer for provider call-plan artifacts. A provider execution dry-run artifact explains why a provider call remains blocked — it does NOT execute any provider call, network request, or credential read.

## New Module
- `src/atlas_agent/research/provider_execution_dry_run.py`
  - Mirrors the Batch 7.4 provider call-plan architecture.
  - Centralized hash exclusion: `PROVIDER_EXECUTION_DRY_RUN_HASH_EXCLUDED_FIELDS = {"artifact_hash", "created_at"}`.
  - 6 boolean safety flags: `provider_enabled`, `network_enabled`, `credentials_loaded`, `provider_call_allowed`, `would_call_provider`, `actual_provider_call_made`.
  - Impossible boolean combination detection.
  - Strict read-path validation with safe sentinels for tampered artifacts.

## New CLI Commands
- `atlas research provider-execution-dry-run PLAN_ID` — create dry-run artifact.
- `atlas research provider-execution-list` — list artifacts.
- `atlas research provider-execution-show ID` — show one artifact (fail-closed).
- `atlas research provider-execution-validate ID` — validate artifact (15+ checks).
- `atlas research provider-execution-replay ID` — replay and compare hashes.

## Integration
- Timeline: dry-runs nested under `provider_call_plans`.
- Dossier: dry-run counts, linked artifacts, summaries, missing_links.
- `check-artifacts`: validates dry-run hashes, lineage, impossible booleans, forbidden fragments.
