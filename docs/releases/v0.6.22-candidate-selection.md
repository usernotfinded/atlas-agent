## Released candidates

- **CAND-016: Release-Maintenance Drift Hardening** — released on 2026-07-13 as
  part of `v0.6.22`. A docs/checker/test-only candidate that makes the
  release-maintenance path metadata-driven: it adds a reverse-drift roadmap guard
  (an already-released candidate must not be listed under the next-planned
  planning line) plus deterministic tests, and retains the metadata-driven
  release-assurance demo default. It changes no runtime, safety, CLI, broker,
  provider, credential, version, or release-metadata behavior and does not
  broaden the CAND-014 boundary. Design and plan:
  [docs/cand-016-release-maintenance-drift-hardening.md](../cand-016-release-maintenance-drift-hardening.md).

### Why CAND-016 was eligible

- It is a docs/checker/test-only addition; it does not implement, authorize, or
  enable any trading or execution path.
- It closes a concrete release-maintenance drift class observed after the
  `v0.6.21` cutover, using release metadata as the authority rather than
  hardcoded release literals.
- It uses only the Python standard library, performs no network access, loads no
  credentials, and does not mutate repository state at runtime.
- It preserves the deliberate per-release version-pin tripwire assertions and the
  CAND-014 single-module extraction boundary.
- It ships deterministic, offline tests and preserves the checker's `0`/`1` exit
  codes.

## Selection rules

Any future `v0.6.23` candidate must satisfy the repository's candidate selection
rules before acceptance:

1. No live trading enablement — fail-closed defaults unchanged.
2. No broker/provider execution enablement — no direct AI-to-broker path.
3. No secrets or credentials committed.
4. No unsafe marketing claims — no promises of specific returns, eliminated
   risk, live readiness, or autonomous trading readiness.
5. Protected runtime boundaries preserved — `RiskManager`, approval queues, kill
   switch, heartbeat, deadman, audit hash-chain, live-submit opt-in, and
   `can_submit` are not bypassed.
6. Small, reviewable candidate batches that are independently valuable.
7. Deterministic test and documentation coverage.
8. Green verification before acceptance.
9. The CAND-014 provider-artifact extraction boundary is not broadened beyond
   `src/atlas_agent/research/provider_mock_response_final_safety_seal.py`.
