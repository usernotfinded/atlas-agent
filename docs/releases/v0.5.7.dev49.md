# Release v0.5.7.dev49 - Release Candidate Cutover Dry Run

## Summary

Added a local, deterministic release candidate cutover dry-run layer for checking whether Atlas Agent can move from the dev series toward a future sandbox/paper/preflight RC tag such as `v0.5.7-rc1`.

This is not the RC release. It does not tag, push, publish, unlock provider execution, enable broker execution, or change trading behavior.

## What changed

- **RC cutover dry-run report** (`src/atlas_agent/research/release_candidate_cutover.py`):
  - Adds `release_candidate_cutover_dry_run` artifacts.
  - Validates target RC version shape without running tag or push commands.
  - Checks dev-to-RC transition coherence for `0.5.7.dev49` to `v0.5.7-rc1`.
  - Checks local docs, verification scripts, forbidden claims, protected boundaries, and release checklist entries.
  - Preserves hard-false safety invariants.
  - Recomputes derived fields during validation so recalculated-hash tampering is rejected.
- **CLI commands**:
  - `atlas research release-candidate-cutover-dry-run --target-version v0.5.7-rc1 --json`
  - `atlas research release-candidate-cutover-dry-run-list --json`
  - `atlas research release-candidate-cutover-dry-run-validate <id> --json`
  - `atlas research release-candidate-cutover-dry-run-summary <id> --json`
  - `atlas research release-candidate-cutover-dry-run-doctor <id> --json`
- **Documentation**:
  - Added `docs/release-candidate-cutover.md`.
  - Added a short README link to the cutover dry-run docs.
  - Updated the release checklist with the dry-run command and review reminder.
- **Tests**:
  - Added focused tests for target validation, output safety, hard-false invariants, blocked statuses, and tamper resistance.
  - Added docs tests for the new cutover documentation.
- **Version bump**: `0.5.7.dev48` to `0.5.7.dev49`.

## What did not change

- No execution behavior changes.
- No broker adapter changes.
- No modifications to risk gates.
- No config system changes.
- No safety boundary changes.
- Live trading remains disabled by default.
- Provider execution remains locked.
- Trust remains blocked.
- No broker/order path enabled.
- No credentials loaded by default.
- No network calls in the cutover dry-run command.
