# Atlas Agent v0.6.5 Release Notes

> **Release Date:** 2026-06-07
> **Not financial advice.** Atlas Agent is a software tool, not a financial advisor. Trading involves significant risk of loss.

## Summary

v0.6.5 is a maintenance patch release containing docs, checker, test, and CI reliability improvements. No new runtime features, broker adapters, or provider integrations are included. No trading, broker, provider, execution, risk, approval, or kill-switch behavior has changed.

This patch contains only backward-compatible maintenance improvements per Semantic Versioning 2.0.0.

## Maintenance Changes

### CAND-001 — Public Docs Aligned to v0.6.4 Current Release
- Updated trust center, public launch readiness, release readiness, and capability inventory docs to consistently describe `v0.6.4` as the current public GitHub release.
- Removed stale "not yet tagged" wording from public docs.

### CAND-002 — Public Tag / Version Consistency Metadata Aligned
- Updated all checker constants (`check_version_consistency.py`, `check_trust_center.py`, `check_public_launch_readiness.py`, `check_public_docs_consistency.py`, `check_stable_release_decision.py`, `check_package_distribution.py`, `main_health.py`) from stale `v0.6.3` to current `v0.6.4`.
- Preserved package-version checks for `0.6.4` separate from tag-format checks for `v0.6.4`.

### CAND-003 — Regression Test for `security_md_current` Package-Version Validation
- Added dedicated regression test proving `release_assurance.py --version v0.6.4` validates `SECURITY.md` using package version `0.6.4`, not tag string `v0.6.4`.
- Test verifies that `security_md_current` fails when `SECURITY.md` contains only the tag version and not the package version.

### CAND-004 — Release Checklist Version References Fixed
- Updated `docs/release-checklist.md` tag example from stale `v0.6.3` to current `v0.6.4`.

### CAND-005 — Deterministic JSON Output Hardening
- Added `sort_keys=True` to `json.dumps` calls in `scripts/check_v065_candidates.py` and `scripts/check_v065_release_prep.py` to ensure deterministic JSON output across platforms.

### CAND-006 — v0.6.5 Release Prep Checker Skeleton
- Created `scripts/check_v065_release_prep.py` and `tests/test_v065_release_prep.py` following the established v0.6.4 pattern.
- Supports planning mode (validates absence of v0.6.5 artifacts while source version remains `0.6.4`) and release-prep mode (validates presence of v0.6.5 artifacts after version bump).
- Supports text and JSON output with deterministic checks.

## Safety Boundaries

- **Live trading** remains disabled by default.
- **Provider execution** remains disabled by default.
- **Broker order submission** remains blocked by `can_submit=false` for all non-paper brokers.
- **Credentials** are not loaded unless explicitly configured.
- **Trust** remains blocked.
- **Risk gates** remain unchanged.
- **Kill switch** and **approval queue** remain unchanged.

## Non-Goals

- **PyPI was not published**.
- **No new runtime features** — patch scope per SemVer 2.0.0.
- **No new broker adapters**.
- **No new provider integrations**.
- **No live trading default changes**.
- **No provider execution default changes**.
- **No automatic skill activation or learning execution**.
- **Autonomous trading is not claimed**.
- **Release cutover required owner approval and passing gates**.
- **No runtime trading behavior changed**.
- **No broker/provider/risk behavior changed**.
- **Tag not yet created**.
- **GitHub release not yet created**.

## Known Limitations

- Same as v0.6.4: Binance adapter is partial, CCXT adapter is disabled, IBKR adapter is a placeholder, strategy pack is limited to 3 built-in strategies, reports and reflection artifacts are local-only.
- PyPI was not published for v0.6.1, v0.6.2, v0.6.3, v0.6.4, or v0.6.5.
