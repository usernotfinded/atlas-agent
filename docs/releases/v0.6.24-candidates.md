## Released

- **CAND-018: Hardcoded Release-Literal Sweep** — released on 2026-07-13 as
  part of `v0.6.24`.
  - Adds `scripts/check_hardcoded_release_literals.py` and
    `tests/test_check_hardcoded_release_literals.py`.
  - Scans active scripts for string literals matching the current source version,
    current public tag, or next planned tag from `release-metadata.json`.
  - Flags literals used in comparisons, assignments, and collections so
    release-maintenance drift is caught before cutover.
  - Excludes intentionally pinned historical release-specific checkers
    (`check_v06*`, `check_v05*`).
  - Fixes the same class of hardcoded version-literal drift in
    `scripts/check_autonomous_paper_workflow_demo.py` and
    `scripts/check_paper_provider_isolation.py` that had already been fixed in
    `scripts/check_bounded_autonomy_governance.py` during the v0.6.23
    post-release pass.
  - Uses only the Python standard library, performs no network access, loads no
    credentials, and does not mutate repository files.
  - Does not enable live trading, live submit, broker/provider execution,
    credential loading, network access, order placement, pending-order creation,
    or approval queue mutation.
  - Does not change `RiskManager`, kill switch, deadman, heartbeat, or audit
    hash-chain behavior.
  - Does not broaden the CAND-014 provider-artifact extraction boundary.
  - The supporting script and tests have landed on `main` as unreleased
    supporting work and are now released in `v0.6.24`.
