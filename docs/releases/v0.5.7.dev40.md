# Release v0.5.7.dev40

## Summary

Added Batch 9.3 — Mock Response Trust Decision Blocker.

## What's New

### Provider Mock Response Trust Decision Blocker

- New module: `src/atlas_agent/research/provider_mock_response_trust_decision_blocker.py`
- Creates, validates, replays, summarizes, and doctors mock response trust decision blocker artifacts
- Derived exclusively from `provider_mock_response_review_sandbox` artifacts
- Hardcodes `provider_id="mock"` — no other provider accepted
- Explicitly blocks all trust decisions, trust upgrades, and trading authorization
- All execution/network/credential/broker booleans are hardcoded `False`
- 12 policy sub-dicts document the absence of trust at every boundary
- Deterministic hash with source hash validation against all 20+ upstream artifacts
- Semantic positive-claim scanning across all policy fields

### New CLI Commands

- `atlas research provider-mock-response-trust-decision-blocker REVIEW_SANDBOX_ID`
- `atlas research provider-mock-response-trust-decision-blocker-list`
- `atlas research provider-mock-response-trust-decision-blocker-show BLOCKER_ID`
- `atlas research provider-mock-response-trust-decision-blocker-validate BLOCKER_ID`
- `atlas research provider-mock-response-trust-decision-blocker-replay BLOCKER_ID`

### Integration

- `check-artifacts` counts and validates trust decision blockers
- `timeline` links trust blockers under review sandboxes
- `dossier` includes trust blocker counts, summaries, and linked artifacts
- `demo_research_workflow.sh` includes full trust blocker flow with assertions

### Tests

- 45+ new tests covering creation, validation, replay, tamper detection, summary, doctor, timeline/check/dossier integration, CLI configless safety, provider-ID regression, and boolean safety flags

## Safety Boundaries

- No trust decision granted
- No trust upgrade performed
- No real provider response trust
- No mock response trust
- No provider SDK imported
- No HTTP client imported
- No network calls
- No credentials loaded
- No env lookup
- No dotenv loading
- No broker/live/execution changes
- No approvals or pending orders created
