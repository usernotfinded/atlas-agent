# Release v0.5.7.dev39 — Mock Response Review Sandbox

## Summary

Added Batch 9.2: a local, configless mock response review sandbox artifact layer.

This creates a sandboxed, non-authorizing review layer for `provider_mock_response_import_candidate` artifacts created by Batch 9.1. This is NOT a real provider response review, NOT a trust upgrade, NOT an approval, and NOT trading signal generation.

## Changes

### New module
- `src/atlas_agent/research/provider_mock_response_review_sandbox.py`
  - Creates, loads, lists, shows, validates, replays, summarizes, and doctors mock response review sandbox artifacts
  - Hard invariant: `provider_id="mock"` only
  - All execution/network/credential/broker booleans hardcoded to `False`
  - Deterministic hash excluding volatile fields
  - Source hash validation against upstream artifacts
  - Semantic positive-claim validation (hash recomputation does NOT bypass)
  - Forbidden fragment scan

### New CLI commands
- `atlas research provider-mock-response-review-sandbox IMPORT_CANDIDATE_ID`
- `atlas research provider-mock-response-review-sandbox-list`
- `atlas research provider-mock-response-review-sandbox-show SANDBOX_ID`
- `atlas research provider-mock-response-review-sandbox-validate SANDBOX_ID`
- `atlas research provider-mock-response-review-sandbox-replay SANDBOX_ID`
- `atlas research provider-mock-response-review-sandbox-summary RUN_ID`
- `atlas research provider-mock-response-review-sandbox-doctor RUN_ID`

All are configless research commands.

### Session integration
- `check_research_artifacts`: counts review sandboxes, detects tampering, impossible booleans, forbidden fragments
- `build_research_timeline`: links review sandboxes under import candidates
- `build_dossier`: includes review sandbox count, linked artifacts, summaries

### Tests
- `tests/research/test_research_provider_mock_response_review_sandbox.py` — 45 tests

### Demo script
- `scripts/demo_research_workflow.sh` — 8 new review sandbox assertions

## Safety boundaries

- No real provider response review
- No raw response body review
- No external response file/stdin review
- No API/network
- No HTTP client
- No provider SDK
- No credentials loaded
- No env lookup
- No dotenv loading
- No real request sent
- No real response received
- Mock review sandbox remains untrusted
- Manual review not completed
- No provider call allowed
- No broker/live/execution changes
- No approvals/pending orders
