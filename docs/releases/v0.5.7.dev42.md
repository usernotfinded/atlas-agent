# Release v0.5.7.dev42

## Summary

Added Batch 9.5 — Provider Safety Dossier.

## What's New

### Provider Safety Dossier

- New module: `src/atlas_agent/research/provider_safety_dossier.py`
- Read-only, offline safety report summarizing the completed mock provider safety chain
- Traverses the full 6-node chain of custody:
  - `mock_response_simulation`
  - `mock_response_import_candidate`
  - `mock_response_review_sandbox`
  - `mock_response_trust_decision_blocker`
  - `mock_response_final_safety_seal`
  - `provider_safety_dossier`
- Reports `chain_complete` / `chain_incomplete` health based on full chain traversal
- Reports `safety_verdict` as `sandbox_chain_complete` or `chain_incomplete`
- Hardcodes `provider_id="mock"` — no other provider accepted
- Stores `source_seal_hash` for lineage verification against the final safety seal
- 16 hard-false boolean invariants enforced at creation and validation:
  `provider_call_allowed`, `actual_provider_call_made`, `provider_response_trusted`,
  `mock_response_trusted`, `trading_signal_generated`, `approval_created`,
  `pending_order_created`, `broker_touched`, `network_enabled`, `credentials_loaded`,
  `trust_upgrade_performed`, `trust_decision_granted`, `provider_execution_unlocked`,
  `real_provider_response_imported`, `live_trading_path_enabled`, `broker_order_path_enabled`
- Deterministic hash excluding `{"artifact_hash", "created_at"}`
- Semantic positive-claim scanning rejects unsafe trust/authorization language
- `sandbox_only = True` enforced at validation

### Tests

- 6 tests covering creation with full chain traversal, validation, tamper detection

  and provider-ID restriction

## Safety Boundaries

- No provider execution unlocked
- No trust decision granted
- No trust upgrade performed
- No provider response trusted
- No mock response trusted
- No network enabled
- No credentials loaded
- No broker touched
- No trading signals, approvals, or pending orders created
- No real provider response imported
- No live trading path enabled
- No broker order path enabled
- Dossier is strictly a local, offline, read-only safety report
