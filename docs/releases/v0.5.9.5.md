# Atlas Agent v0.5.9.5

Public trust status: [v0.5.9.5 Trust and Release Status](../trust/v0.5.9.5-status.md).

## Summary

Atlas Agent `v0.5.9.5` promotes the current `main` state after `v0.5.9.4` into a public GitHub maintenance release so update detection can see the release-assurance local workspace fix.

This release bumps the package/source version from `0.5.9.4` to `0.5.9.5` and updates the public release identity from `v0.5.9.4` to `v0.5.9.5`.

## Included Changes

- Release assurance updater dry-run checks now run in an isolated temporary Atlas workspace.
- Source/package version is `0.5.9.5`.
- Public GitHub release identity is `v0.5.9.5`.
- The `v0.5.9.4` release remains historical and was not retagged.

## Release Assurance Fix

`v0.5.9.5` exists because `main` received a post-release fix after `v0.5.9.4`: release assurance dry-run updater checks now stay local by initializing and checking an isolated temporary workspace instead of mutating repository-local update state.

The fix is release-engineering only. It does not alter trading execution, broker adapters, provider execution, risk gates, approval gates, live-mode defaults, or dashboard behavior.

## CI / Workflow Maintenance

The maintenance context from `v0.5.9.4` remains included:

- GitHub Actions Node 24 modernization remains included.
- `actions/checkout`, `actions/setup-python`, and `actions/upload-artifact` remain on `@v6` where used.
- GitHub Actions version guard remains included.
- Main health and generated artifact hygiene checks remain included.
- Trust center, onboarding, and release-readiness checks remain included.

## Safety Defaults

- No live trading default changes.
- No live submit default changes.
- No provider execution default changes.
- No broker execution default changes.
- No runtime trading behavior changes.
- Risk gates, approval gates, kill-switch behavior, audit hash-chain behavior, and manifest behavior are unchanged.

  --version v0.5.9.5 \
