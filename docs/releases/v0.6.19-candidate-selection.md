## Proposed candidates

- **CAND-011:** Kill-Switch `last_heartbeat()` Type-Safety Cleanup.

### Why CAND-011 is eligible

- It is a type-safety cleanup only; it does not implement, authorize, or enable
  any trading or execution path.
- It removes the pre-existing mypy `union-attr` warning in
  `src/atlas_agent/safety/kill_switch.py` by narrowing the nullable return value
  of `HeartbeatManager.last_heartbeat()` before calling `.isoformat()`.
- It preserves the `last_heartbeat` audit payload key and the `str | None` value
  shape.
- It adds mandatory `heartbeat_expired` audit-payload regression tests covering
  the ISO-string branch and the `None` branch.
- It introduces no live trading, live submit, broker/provider execution,
  credential loading, order placement, pending-order creation, network access,
  or approval queue mutation.
- It changes no kill-switch decision logic, fail-closed behavior, return values,
  heartbeat/deadman behavior, `RiskManager`, or audit hash-chain.
- It includes deterministic tests and passes the targeted verification commands.
- Release cutover, if any, will be performed only after a separate
  release-readiness review and will preserve all safety boundaries: no live
  trading enablement, no live submit, no order placement, no broker/provider
  execution, no credential loading, no network access, and no PyPI publication.
