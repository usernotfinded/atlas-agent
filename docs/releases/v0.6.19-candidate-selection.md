## Proposed candidates

- **CAND-011:** Kill-Switch `last_heartbeat()` Type-Safety Cleanup.

### Why CAND-011 was eligible

- It is a type-safety cleanup only; it does not implement, authorize, or enable
  any trading or execution path.
- It removes the pre-existing mypy `union-attr` warning in
  `src/atlas_agent/safety/kill_switch.py` by narrowing the nullable return value
  of `HeartbeatManager.last_heartbeat()` before calling `.isoformat()`.
- It preserves the `last_heartbeat` audit payload key and the `str | None` value
  shape.
- It adds mandatory `heartbeat_expired` audit-payload regression tests covering
  the ISO-string branch and the `None` branch.
- It introduces no live trading, live submit, broker/provider execution,
  credential loading, order placement, pending-order creation, network access,
  or approval queue mutation.
- It changes no kill-switch decision logic, fail-closed behavior, return values,
  heartbeat/deadman behavior, `RiskManager`, or audit hash-chain.
- It includes deterministic tests and passes the targeted verification commands.

## Candidate acceptance

**CAND-011** is accepted into the `v0.6.19` candidate chain with acceptance
verdict `PASS` on 2026-07-01 and is released in `v0.6.19`. This acceptance does
not change runtime behavior beyond the type-safety cleanup already landed on
`main`, nor does it enable live trading or live submit, authorize order
placement, load credentials, call brokers or providers, access the network, or
mutate runtime state or approval queues. It is not an approval to trade, not a
live-readiness claim, and not permission to submit orders.

### Selection rules

CAND-011 satisfies the repository's candidate selection rules:

1. No live trading enablement — fail-closed defaults unchanged.
2. No broker/provider execution enablement — no direct AI-to-broker path.
3. No secrets or credentials committed.
4. No unsafe marketing claims — no promises of specific returns, eliminated
   risk, live readiness, or autonomous trading readiness.
5. Protected runtime boundaries preserved — changes are confined to a single
   local-variable narrowing in `kill_switch.py` and companion regression tests.
6. Small candidate batch — reviewable and independently valuable.
7. Test/doc coverage — deterministic behavior-preservation tests and updated
   candidate-chain documentation.
8. Green verification before acceptance — passes targeted verification commands.

3. A full `pytest` run or full `bash scripts/release_check.sh` may exceed local
