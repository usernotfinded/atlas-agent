# Atlas Agent v0.6.8 Release Notes

> **Release Date:** 2026-06-10 (tagged and released)
> **Not financial advice.** Atlas Agent is a software tool, not a financial advisor. Trading involves significant risk of loss.

## Summary

v0.6.8 is a public-demo-proof maintenance patch release. No runtime features, broker adapters, or provider integrations are included. No trading, broker, provider, execution, risk, approval, or kill-switch behavior has changed.

This patch contains only backward-compatible maintenance improvements per Semantic Versioning 2.0.0.

## Maintenance Changes

### CAND-001 — Demo Artifact Index
- Added `docs/demo-artifact-index.md`, a complete indexed view of every paper-demo artifact, its path, content summary, and safety invariant.
- Cross-referenced from `README.md`, `docs/demo-paper-workflow.md`, and `docs/external-reviewer-walkthrough.md`.

### CAND-002 — Demo Proof Checker
- Added `scripts/check_demo_proof.py` and `tests/test_demo_proof_checker.py`, a deterministic demo proof checker that validates demo documentation, artifact index consistency, safety invariants, script/doc alignment, canonical reviewer path, symbol consistency, and over-promise claims without executing the demo.
- Local-only; no credentials, network, or execution.

### CAND-003 — Reviewer Demo Path Consolidation
- Consolidated the reviewer demo path across `README.md`, `docs/external-reviewer-walkthrough.md`, `docs/demo-paper-workflow.md`, `docs/demo-artifact-index.md`, and `docs/trust/README.md`.
- Fixed README quickstart to use `ATLAS-DEMO` consistently with the demo script.
- Corrected stale `0.6.8` source-version-prepared wording in README.
- Removed over-promise v0.6.8 release-notes-prepared claim from trust center.
- Replaced ambiguous `production-ready` wording in brokers doc with safer phrasing.

### CAND-004 — Demo Command Smoke Validation
- Added `scripts/check_demo_command_smoke.py` and `tests/test_demo_command_smoke.py`, a lightweight static smoke checker that validates the demo script exists, is executable, is referenced in docs, contains paper-only safety wording, and excludes forbidden high-risk patterns.
- Integrated into `scripts/smoke_check.sh` and `scripts/local_quick_check.sh`.

## Safety Boundaries

- **Live trading** remains disabled by default.
- **Provider execution** remains disabled by default.
- **Broker order submission** remains blocked by `can_submit=false` for all non-paper brokers.
- **Credentials** are not loaded unless explicitly configured.
- **Trust** remains blocked.
- **Risk gates** remain unchanged.
- **Kill switch** and **approval queue** remain unchanged.

## Non-Goals

- **PyPI was not published**.
- **No new runtime features** — patch scope per SemVer 2.0.0.
- **No new broker adapters**.
- **No new provider integrations**.
- **No live trading default changes**.
- **No provider execution default changes**.
- **No automatic skill activation or learning execution**.
- **Autonomous trading is not claimed**.
- **Release cutover required owner approval and passing gates**.
- **No runtime trading behavior changed**.
- **No broker/provider/risk behavior changed**.
- **Tag**: `v0.6.8` created.
- **GitHub release**: created and published.

## Known Limitations

- Same as v0.6.7: Binance adapter is partial, CCXT adapter is disabled, IBKR adapter is a placeholder, strategy pack is limited to 3 built-in strategies, reports and reflection artifacts are local-only.
- PyPI was not published for v0.6.1, v0.6.2, v0.6.3, v0.6.4, v0.6.5, v0.6.6, or v0.6.7.
