# Atlas Agent v0.6.25 Release Notes

> **Status:** current public GitHub release
> **Released:** 2026-07-13
> **Previous public release:** `v0.6.24`
> **Next planned release:** `v0.6.26`
> **Distribution:** GitHub-only; PyPI was not published
> **Not financial advice.** Atlas Agent is a software tool, not a financial
> advisor. Trading involves significant risk of loss.

## Summary

`v0.6.25` is a release-maintenance tooling release. It includes two accepted
candidates:

- **CAND-019: Docs/config hardcoded release-literal sweep** extends
  `scripts/check_hardcoded_release_literals.py` to scan `configs/`, `examples/`,
  and `docs/examples/` for monitored release-identity literals, and adds
  regression tests.
- **CAND-020: Post-v0.6.24 test drift sweep** updates stale post-`v0.6.24` test
  fixtures and assertions to the current release identity, adds `0.6.24` to the
  `SECURITY.md` supported-versions table, and records this candidate.

Both candidates are checker/test/docs-only. No live trading, live submit, broker
order submission, provider execution, credential loading, order placement,
pending-order creation, or approval queue mutation is introduced or enabled.
`atlas run --mode live` remains fail-closed.

This is a **GitHub-only release**. PyPI was not published.

## Included Candidates

- **CAND-019: Docs/config hardcoded release-literal sweep**
  - Extends `scripts/check_hardcoded_release_literals.py` to also scan
    `configs/`, `examples/`, and `docs/examples/`.
  - Reads source version, current public tag, and next planned tag from
    `docs/releases/release-metadata.json`.
  - Scans all active scripts under `scripts/` plus the new docs/config/example
    directories for string literals equal to any of those release-identity values
    (with or without a leading `v`).
  - Flags literals used in comparisons, assignments, annotated assignments,
    calls, and collections.
  - Excludes intentionally pinned historical release-specific checkers
    (`check_v06*`, `check_v05*`) from the scan.
  - Adds/updates regression tests in
    `tests/test_check_hardcoded_release_literals.py`.
  - Uses only the Python standard library, performs no network access, loads no
    credentials, and does not mutate repository files.
  - Does not change `RiskManager`, kill switch, deadman, heartbeat, or audit
    hash-chain behavior.
  - Does not enable live trading, live submit, broker/provider execution,
    credential loading, network access, order placement, pending-order creation,
    or approval queue mutation.
  - Does not broaden the CAND-014 provider-artifact extraction boundary, which
    remains strictly at
    `src/atlas_agent/research/provider_mock_response_final_safety_seal.py`.

- **CAND-020: Post-v0.6.24 test drift sweep**
  - Updates stale post-`v0.6.24` test fixtures and assertions to the current
    release identity (`0.6.25` / `v0.6.25` / `v0.6.26`).
  - Adds `0.6.24` to the `SECURITY.md` supported-versions table.
  - Records this candidate in the v0.6.25 candidate chain.
  - Test-only drift fixes; no runtime, safety, broker, provider, credential,
    network, or trading changes.
  - Does not change `RiskManager`, kill switch, deadman, heartbeat, or audit
    hash-chain behavior.
  - Does not enable live trading, live submit, broker/provider execution,
    credential loading, network access, order placement, pending-order creation,
    or approval queue mutation.
  - Does not broaden the CAND-014 provider-artifact extraction boundary.

## Safety Boundaries

- This release does not enable live trading or live submit.
- This release does not enable provider execution.
- No autonomous trading is claimed or enabled by this release.
- This release does not place, cancel, or flatten orders.
- This release does not create pending orders or mutate approval queues.
- This release does not call brokers, providers, or network endpoints.
- This release does not load credentials, secrets, or API keys.
- `RiskManager`, approval gates, the kill switch, the dead-man heartbeat, and
  the audit hash-chain remain unchanged and are not weakened.
- `atlas run --mode live` remains fail-closed (exits 2).
- CAND-019 and CAND-020 are static checker/test/docs additions only; no runtime
  safety decision changes.
- No live-readiness, autonomous-live-readiness, trading safety, broker
  endorsement, or permission to submit orders is claimed.
- No profitability, absolute-safety, or future-performance claim is made.

   expected warnings from `check_candidate_chain.py` because they reflect
   historical planning state rather than current metadata.
