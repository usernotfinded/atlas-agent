# Release v0.5.7.dev38

## Summary

Batch 9.1 adds the **Mock Response Import Candidate** system — a local, configless schema-validation sandbox that derives import-candidate artifacts exclusively from existing `provider_mock_response_simulation` artifacts. It verifies mock-source lineage, checks structural compatibility, and enforces all mock-only safety boundaries without importing real provider responses.

## New Module

- `src/atlas_agent/research/provider_mock_response_import_candidate.py`

- `atlas research provider-mock-response-import-candidate-doctor RUN_ID`

## Safety Invariants

- `provider_id` is hardcoded to `"mock"` and validated strictly (hash recomputation does NOT bypass).
- `source_provider_id` preserves the upstream provider for lineage tracing.
- All execution/network/credential/broker booleans are `False`.
- `mock_schema_candidate_valid=True` means **only** local structural compatibility — it never implies `future_response_schema_validated=True`, `provider_response_trusted=True`, or any real-response validation.
- Semantic positive-claim validation recursively scans all policy fields for unsafe phrases.
- Source hash validation covers mock simulation + all upstream artifacts (adapter contract, unlock state, review result, schema contract, pairing, intake policy, payload preview).
- No file/stdin/raw/url input options on creation — accepts ONLY an existing simulation ID.
- No real provider response is imported.
- No network, credentials, SDKs, trading signals, approvals, pending orders, or broker calls.
