# Release v0.5.7.dev45 — Public Safety Proof Pack

## Scope

Batch 9.8 — Public Safety Proof Pack.

This batch is documentation-heavy and test/guard-heavy. It creates a public-facing documentation and verification pack that makes the provider safety pipeline understandable, reproducible, and credible for GitHub users, without adding any execution capability.

## What changed

- **README integration**: Added a concise `Provider Safety Dossier` section explaining the mock/offline safety chain, safe command examples, and required safety wording.
- **Public docs**: Added `docs/provider-safety-dossier.md` with full explanation of what the feature is and is not, the safety chain, invariants, commands, safe output policy, review checklist, and limitations.
- **Workflow example**: Added `docs/examples/provider-safety-dossier-workflow.md` with a safe copy-paste workflow for discovering and exporting dossiers.
- **Documentation guard tests**: Added `tests/test_provider_safety_dossier_docs.py` to verify docs contain required safe phrases, lack forbidden overclaims, contain no absolute paths, and contain no forbidden fragments.
- **Version bump**: `0.5.7.dev44` → `0.5.7.dev45`.

## What did not change

- No new trading behavior.
- No broker integrations.
- Risk gates remain unchanged.
- No execution logic changes.
- No config loading changes.
- No live trading enablement.
- No credential loading.
- No network calls.
- No OS open/file-launch commands.
- No HTML/PDF export.
- No dashboard features.
- No new artifact type.
- Protected boundaries (`config/`, `brokers/`, `execution/`, `safety/`, `risk/`) remain untouched.
