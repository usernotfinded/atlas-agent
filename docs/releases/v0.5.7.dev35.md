# Release v0.5.7.dev35

## Summary

Batch 8.8 adds the **Provider Execution Unlock State Machine** — a local, configless artifact that records the explicit state of provider execution unlocking, keeps provider execution locked, requires manual unlock, and never grants it.

## New Module

- `src/atlas_agent/research/provider_execution_unlock_state.py`

## New CLI Commands

- `atlas research provider-execution-unlock-state REVIEW_RESULT_ID`
- `atlas research provider-execution-unlock-state-list`
- `atlas research provider-execution-unlock-state-show UNLOCK_STATE_ID`
- `atlas research provider-execution-unlock-state-validate UNLOCK_STATE_ID`
- `atlas research provider-execution-unlock-state-replay UNLOCK_STATE_ID`
- `atlas research provider-execution-unlock-state-summary RUN_ID`
- `atlas research provider-execution-unlock-state-doctor RUN_ID`

## Safety Invariants

- Provider execution remains locked.
- Manual unlock is required but not granted.
- Provider calls remain disabled.
- Provider adapter is not implemented.
- Credential loader is not implemented.
- Network remains disabled.
- No provider request is sent.
- No provider response is received.
- No provider response is trusted.
- No approvals, pending orders, or broker calls are created.
- No trading signal is generated.
- No broker is touched.
