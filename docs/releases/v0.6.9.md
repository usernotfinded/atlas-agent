# Atlas Agent v0.6.9 Release Notes

> **Release Date:** 2026-06-11
> **Not financial advice.** Atlas Agent is a software tool, not a financial advisor. Trading involves significant risk of loss.

## Summary

v0.6.9 is a backtest-report quality and validation release. It improves deterministic local backtesting, formalizes the backtest report schema contract, and makes validation diagnostics more useful for reviewers. No runtime trading, broker, provider, execution, risk, approval, or kill-switch behavior has changed.

This release contains only backward-compatible improvements per Semantic Versioning 2.0.0.

## Changes

### Backtest Engine Improvements
- Added date filtering to local backtests (`--start-date`, `--end-date`).
- Added `atlas backtest runs` CLI for listing historical backtest results.
- Added realized-PnL tracking per sell fill.

### Backtest Report Schema Contract
- Added `src/atlas_agent/backtest/report_schema.py` with the `backtest.report.v1` schema contract.
- Added deterministic schema validation for JSON backtest reports.
- Added accumulated schema error collection (`collect_backtest_report_schema_errors`).

### Backtest Validation UX
- Added `--validate` flag to `atlas backtest runs` with detailed JSON/text output.
- Added structured schema validation results (`schema_status`, `schema_valid`, `schema_error`, `schema_errors`, `schema_version`).
- Surfaced unreadable `result.json` files in validation output instead of silently skipping them.

### Dashboard / Report Quality
- Surfaced `latest_schema_version` and `latest_validation_status` in report quality output.
- Added Diagnostics and Fills Summary sections to Markdown backtest reports.
- Added realized-PnL trade metrics to Markdown reports.
- Added percentage trade metrics (`Best Trade %`, `Worst Trade %`, `Average Trade %`) to Markdown reports.

### Strategy Entry-Point Dogfooding
- Declared built-in backtest strategies as `pyproject.toml` entry points and added tests that verify they are discoverable.

### Release Metadata / Checker Hardening
- Migrated remaining release metadata checkers to read from `docs/releases/release-metadata.json` dynamically.

## Safety Boundaries

- **Live trading** remains disabled by default.
- **Provider execution** remains disabled by default.
- **Broker order submission** remains blocked by `can_submit=false` for all non-paper brokers.
- **Credentials** are not loaded unless explicitly configured.
- **Trust** remains blocked.
- **Risk gates** remain unchanged.
- **Kill switch** and **approval queue** remain unchanged.
