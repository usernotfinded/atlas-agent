# Atlas Agent v0.5.7.dev18

## Summary

- This is a development tag.
- Focus: Efficient Local Test Gates (Batch 7.2).
- It is not production live-trading readiness.
- No financial outcome is promised.

## Highlights

- `scripts/dev_check.sh` provides a fast local development gate without full pytest, demos, or pip check.
- `scripts/research_check.sh` provides a medium-cost research/sandbox gate covering research tests and the research demo workflow.
- `scripts/release_check.sh` supports `--quick`, `--research`, and `--full` modes.
  - `--quick` runs `dev_check.sh`.
  - `--research` runs `research_check.sh`.
  - `--full` (default) runs the complete release gate unchanged.
- Optional thermal-friendly environment variables for pytest:
  - `ATLAS_CHECK_FAIL_FAST=1` appends `-x` to pytest.
  - `ATLAS_CHECK_LAST_FAILED=1` appends `--lf` to pytest.
  - `ATLAS_CHECK_PYTEST_ARGS` appends arbitrary extra pytest arguments safely.

## Recommended Local Loop

Fast active-development loop:

```bash
./scripts/release_check.sh --quick
```

When touching research/sandbox code:

```bash
./scripts/release_check.sh --research
```

Before push/tag (required):

```bash
./scripts/release_check.sh --full
```

## Safety and Compatibility Notes

- Quick and research modes are developer convenience checks only.
- Full mode remains required before push/tag.
- No changes to broker behavior, live trading gates, order routing, approval manager, risk manager, or config secret loading.
- No network calls, no API keys read, no orders submitted.
- Live submit remains disabled by default and gated.
