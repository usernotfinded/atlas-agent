> advisor. Trading involves significant risk of loss.

## Summary

`v0.6.24` is a release-maintenance tooling release. It includes
**CAND-018: Hardcoded Release-Literal Sweep**, a checker/test-only candidate
that adds `scripts/check_hardcoded_release_literals.py` and tests. The script
scans active checker scripts for string literals matching the current source
version, current public tag, or next planned tag from
`docs/releases/release-metadata.json`, and fails when it finds them. This
prevents the post-release drift where a script embeds a release identity literal
that becomes stale at the next cutover.

The release also fixes the same class of hardcoded version-literal drift in
`scripts/check_autonomous_paper_workflow_demo.py` and
`scripts/check_paper_provider_isolation.py` that had already been fixed in
`scripts/check_bounded_autonomy_governance.py` during the v0.6.23 post-release
pass.

This is a **GitHub-only release**. PyPI was not published. No live trading, live
submit, broker order submission, provider execution, credential loading, order
placement, pending-order creation, or approval queue mutation is introduced or
enabled. `atlas run --mode live` remains fail-closed.

## Included Candidates

- **CAND-018: Hardcoded Release-Literal Sweep**
  - Adds `scripts/check_hardcoded_release_literals.py` and
    `tests/test_check_hardcoded_release_literals.py`.
  - Reads source version, current public tag, and next planned tag from
    `docs/releases/release-metadata.json`.
  - Scans all active scripts under `scripts/` for string literals equal to any
    of those release-identity values (with or without a leading `v`).
  - Flags literals used in comparisons, assignments, annotated assignments,
    calls, and collections.
  - Excludes intentionally pinned historical release-specific checkers
    (`check_v06*`, `check_v05*`) from the scan.
  - Fixes the same hardcoded version-literal drift in
    `scripts/check_autonomous_paper_workflow_demo.py` and
    `scripts/check_paper_provider_isolation.py`.
  - Uses only the Python standard library, performs no network access, loads no
    credentials, and does not mutate repository files.
  - Does not change `RiskManager`, kill switch, deadman, heartbeat, or audit
    hash-chain behavior.
  - Does not enable live trading, live submit, broker/provider execution,
    credential loading, network access, order placement, pending-order creation,
    or approval queue mutation.
  - Does not broaden the CAND-014 provider-artifact extraction boundary, which
    remains strictly at
    `src/atlas_agent/research/provider_mock_response_final_safety_seal.py`.

## Known Non-Blocking Warnings

1. A full `pytest` run or full `bash scripts/release_check.sh` may exceed local
   interactive timeouts. The `--quick` gate and all targeted checks pass.

   expected warnings from `check_candidate_chain.py` because they reflect
   historical planning state rather than current metadata.
