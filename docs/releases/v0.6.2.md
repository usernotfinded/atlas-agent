# Atlas Agent v0.6.2 Release Notes

> **Release Date:** 2026-06-06
> **Not financial advice.** Atlas Agent is a software tool, not a financial advisor. Trading involves significant risk of loss.

## Summary

v0.6.2 is a maintenance hotfix release that incorporates the post-release CI fixes discovered after the v0.6.1 tag/release. No new runtime features, broker adapters, or provider integrations are included.

This patch contains only backward-compatible bug fixes and process improvements per Semantic Versioning 2.0.0.

## Maintenance Changes

### Post-Release v0.6.1 CI Failure Fixes
- Fixed `tests/test_discipline_profile.py` pytest-xdist collection mismatch by changing `list(_FORBIDDEN_PATTERNS)` to `sorted(_FORBIDDEN_PATTERNS)` for deterministic parameter ordering across workers.
- Fixed `tests/test_provider_policy_docs.py` to allow `notifications/transports.py` in the urllib provider-policy allowlist, with detailed safety justification documenting it is a notification webhook transport (not provider execution), disabled/dry-run by default, and testable via injected `http_post`.
- Added `twine>=4.0` to `pyproject.toml` dev dependencies for package distribution verification tooling.

### Public Documentation Corrections
- Updated `README.md`, `SECURITY.md`, `docs/public-launch-readiness.md`, and `docs/security/release-readiness.md` to reflect that `v0.6.1` is the current stable GitHub release (no longer version-prepared).
- Aligned `docs/releases/v0.6.1.md` Non-Goals section with exact phrases expected by `scripts/release_assurance.py` `non_claims_preserved` check.

### Package Distribution Verification Improvements
- Dev dependency list now includes `twine>=4.0` so wheel/sdist validation (`twine check`) is available in CI and local dev environments.
- Hardened package distribution checks with deterministic wheel/sdist artifact discovery and stronger staged generated-artifact detection for `.egg-info` paths.
- Made release assurance JSON and checksum output deterministic for stable local and CI evidence comparisons.
- Removed the release assurance shell invocation from local updater dry-run verification while preserving the same non-mutating checker behavior.
- No runtime dependency changes.

## Safety Boundaries

- **Live trading** remains disabled by default.
- **Provider execution** remains disabled by default.
- **Broker order submission** remains blocked by `can_submit=false` for all non-paper brokers.
- **Credentials** are not loaded unless explicitly configured.
- **Trust** remains blocked.
- **Risk gates** remain unchanged.
- **Kill switch** and **approval queue** remain unchanged.

## Non-Goals

- **PyPI publish: not performed**.
- **No new runtime features** — patch scope per SemVer 2.0.0.
- **No new broker adapters**.
- **No new provider integrations**.
- **No live trading default changes**.
- **No provider execution default changes**.
- **No automatic skill activation or learning execution**.
- **Autonomous trading is not claimed**.
- **Release cutover required owner approval and passing gates**.

## Known Limitations

- Same as v0.6.1: Binance adapter is partial, CCXT adapter is disabled, IBKR adapter is a placeholder, strategy pack is limited to 3 built-in strategies, reports and reflection artifacts are local-only.
- PyPI publish was not performed for v0.6.1 or v0.6.2.
