# Release v0.5.7.dev44 — Batch 9.7: Provider Safety Dossier Discovery UX

## Summary

Improved read-only discovery UX for provider safety dossiers. Users can now find the latest valid dossier and filter dossier lists by safe status without changing the safety model.

## New CLI Commands

- `atlas research provider-safety-dossier-latest [--json]` — Returns the latest valid provider safety dossier metadata. Ignores invalid/tampered dossiers. Returns safe metadata only (no paths, no raw invalid fields).
- `atlas research provider-safety-dossier-list --status <status> [--json]` — Filter dossier list by safe status.

## Enhanced CLI Commands

- `atlas research provider-safety-dossier-list` now supports:
  - `--status sandbox_chain_complete` — Valid dossiers with complete chain
  - `--status chain_incomplete` — Valid dossiers with incomplete chain
  - `--status chain_invalid` — Invalid dossiers (validation failed)
  - `--status unsafe_tamper_detected` — Tampered dossiers (hash mismatch, impossible boolean, forbidden trust claim, seal hash mismatch)

## Leak Safety

- CLI JSON and text output never contain absolute paths (`/Users/`, `/private/var/`).
- List and latest envelopes never contain raw invalid fields.
- Invalid dossiers are summarized with static safe sentinels only.
- Status filtering operates on validated data, never on raw tampered status fields.
