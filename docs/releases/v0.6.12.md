# Atlas Agent v0.6.12 Release Notes

> **Status:** current public release
> **Released:** 2026-06-17
> **Previous public release:** `v0.6.11`
> **Not financial advice.** Atlas Agent is a software tool, not a financial
> advisor. Trading involves significant risk of loss.

## Summary

v0.6.12 is a release-preparation and owner-approval-gate patch. It consolidates
the v0.6.12 candidate work (CAND-001 through CAND-016) into a version-bumped,
documentation-complete release and completes the public GitHub release cutover
(CAND-019). No runtime trading, broker, provider, execution, risk, approval, or
kill-switch behavior has changed.

This release batch:

- Bumps the package/source version from `0.6.11` to `0.6.12`.
- Creates `docs/releases/v0.6.12.md` and `docs/trust/v0.6.12-status.md`.
- Creates the owner-approval gate document at
  `docs/releases/v0.6.12-owner-approval.md`.
- Updates `CHANGELOG.md`, `README.md`, `SECURITY.md`, `docs/trust/README.md`,
  `docs/development/main-health.md`, `docs/public-launch-readiness.md`, and
  `docs/release-checklist.md` to reflect `0.6.12` source / `v0.6.12` public /
  `v0.6.13` next-planning state.
- Adds `scripts/check_v0612_release_prep.py` and matching tests for v0.6.12
  release-prep state validation.
- Adds `scripts/check_v0612_release_cutover.py` and matching tests for v0.6.12
  public-release cutover validation.
- Integrates the v0.6.12 release-prep check into `scripts/dev_check.sh`,
  `scripts/ci_check.sh`, `scripts/release_check.sh`, and the GitHub Actions
  `quick-gate` workflow.

The annotated tag `v0.6.12` was created and pushed, and the GitHub Release
`v0.6.12` was published, as part of the owner-approved public cutover. PyPI was
not published.

## Safety Boundaries

- **Live trading** remains disabled by default.
- **Live submit** remains disabled by default.
- **Provider execution** remains disabled by default.
- **Broker order submission** remains blocked by `can_submit=false` for all non-paper brokers.
- **Credentials** are not loaded unless explicitly configured.
- **Trust** remains blocked.
- **Risk gates** remain unchanged.
- **Kill switch** and **approval queue** remain unchanged.
- **Diagnostics** remain read-only, no-network, and redacted.
- **Dashboard** remains read-only, local, and zero-secret.

## Non-Goals

- **PyPI was not published** for `v0.6.12`.
- **No new runtime features** — scope per SemVer 2.0.0.
- **No new broker adapters**.
- **No new provider integrations**.
- **No live trading default changes**.
- **No provider execution default changes**.
- **No automatic skill activation or learning execution**.
- **Autonomous trading is not claimed**.
- **No runtime trading behavior changed**.
- **No broker/provider/risk behavior changed**.

## Known Limitations

- Same as `v0.6.11`: Binance adapter is partial, CCXT adapter is disabled, IBKR adapter is a placeholder, strategy pack is limited to built-in strategies, reports and reflection artifacts are local-only.
- PyPI was not published for `v0.6.1`, `v0.6.2`, `v0.6.3`, `v0.6.4`, `v0.6.5`, `v0.6.6`, `v0.6.7`, `v0.6.8`, `v0.6.9`, `v0.6.10`, `v0.6.11`, or `v0.6.12`.
