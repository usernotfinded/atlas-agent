# Atlas Agent v0.6.18 Release Notes

> **Status:** current public GitHub release
> **Released:** 2026-07-01
> **Previous public release:** `v0.6.17`
> **Distribution:** GitHub-only; PyPI was not published
> **Not financial advice.** Atlas Agent is a software tool, not a financial
> advisor. Trading involves significant risk of loss.

## Summary

`v0.6.18` is a safety-regression-guard release. It includes **CAND-010:
Safety-State Persistence Regression Guard**, a static checker that prevents
reintroduction of the fixed `<target>.tmp` write patterns that CAND-009 removed
from the safety persistence modules. The release also fixes a release-assurance
diagnostic formatting bug that could cause protected-boundary remediation text
to raise a `KeyError`.

This is a **GitHub-only release**. PyPI was not published. No live trading, live
submit, broker order submission, provider execution, credential loading, order
placement, pending-order creation, or approval queue mutation is introduced or
enabled. `atlas run --mode live` remains fail-closed.

## Included Candidate

- **CAND-010:** Safety-State Persistence Regression Guard
  - Adds `scripts/check_safety_atomic_write.py`, a stdlib-only static checker
    that scans `src/atlas_agent/safety/heartbeat.py`,
    `src/atlas_agent/safety/deadman.py`,
    `src/atlas_agent/safety/kill_switch.py`, and
    `src/atlas_agent/safety/state.py` for reintroductions of fixed
    `<target>.tmp` write patterns.
  - Allows `src/atlas_agent/safety/atomic_write.py` as the unique temporary
    helper that may construct `.tmp` names via `tempfile.mkstemp`.
  - Adds `tests/test_check_safety_atomic_write.py` coverage for pass, fail,
    output format, exit codes, per-module regressions, and helper allowance.
  - Integrates the checker into `scripts/dev_check.sh` and `scripts/ci_check.sh`
    so that `scripts/release_check.sh --quick` covers the guard.
  - Preserves CAND-009 atomic-write runtime behavior; no live execution,
    broker/provider, credential, or order semantics are changed.

## Safety Boundaries

- This release does not enable live trading or live submit.
- This release does not enable provider execution.
- No autonomous trading is claimed or enabled by this release.
- This release does not place, cancel, or flatten orders.
- This release does not create pending orders or mutate approval queues.
- This release does not call brokers, providers, or network endpoints.
- This release does not load credentials, secrets, or API keys.
- `RiskManager`, approval gates, the kill switch, the dead-man heartbeat, and
  the audit hash-chain remain unchanged and are not weakened.
- `atlas run --mode live` remains fail-closed (exits 2).
- CAND-006 remains simulated-only.
- CAND-007 remains no-live-submit.
- CAND-008 remains evidence-only, simulated-only, and non-executing.
- CAND-009 runtime atomic-write behavior remains unchanged.
- CAND-010 is a static regression enforcement tool only.
- No live-readiness, autonomous-live-readiness, trading safety, broker
  endorsement, or permission to submit orders is claimed.
- No profitability, absolute-safety, or future-performance claim is made.
