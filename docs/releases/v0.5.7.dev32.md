# Release v0.5.7.dev32 — Provider Request/Response Pairing Contract Artifact

## Summary

This release adds local provider request/response pairing contract artifacts and their integration into the research chain. No real provider response handling is added. No network calls. No API keys. No provider SDKs. No broker changes. No credentials are loaded.

Batch 8.5 introduces a deterministic, auditable pairing contract artifact that binds a provider outbound payload preview (request side) to a future provider response (response side). It is created from a provider response intake policy and records safe pairing metadata only: request side, response side, correlation policy, provider trace policy, request hash policy, response hash policy, pairing validation policy, pairing replay policy, mismatch policy, trust boundary policy, manual review policy, and future response requirements. No raw provider response body, no raw request body, no credentials, no absolute paths, and no trading instructions are stored. Provider execution remains disabled.

## What Changed

### New Artifact

- `src/atlas_agent/research/provider_request_response_pairing.py`
  - `build_provider_request_response_pairing_dict()`: deterministic pairing contract artifact construction from an intake policy.
  - `create_provider_request_response_pairing()`: safe creation with validation, lineage, hash, and denylist scanning.
  - `safe_validate_provider_request_response_pairing_data()`: strict validation with invalid sentinel returns.
  - `validate_provider_request_response_pairing_artifact()`: detailed per-check validation.
  - `replay_provider_request_response_pairing()`: deterministic hash replay from source intake policy.
  - `iter_provider_request_response_pairing_artifacts()`: safe listing with invalid sentinels for tampered items.
  - `summarize_provider_request_response_pairing_state()`: read-only summary without writing artifacts.
  - `doctor_provider_request_response_pairing_state()`: read-only chain diagnostic.
  - `provider_request_response_pairing_sha256()`: deterministic canonical hash excluding volatile fields.
  - Policy substructures:
    - `request_side`: `raw_request_body_stored=False`, `request_body_hash_required=True`, `request_preview_linked=True`.
    - `response_side`: `raw_response_body_stored=False`, `response_body_hash_required=True`, `response_trusted_by_default=False`, `response_review_required=True`.
    - `correlation_policy`: `explicit_correlation_required=True`, `external_correlation_id_allowed=False`, `provider_trace_id_allowed=False`.
    - `provider_trace_policy`: `provider_call_trace_required=True`, `dry_run_trace_required=True`, `state_transition_trace_required=True`.
    - `request_hash_policy`: `request_hash_required=True`, `hash_algorithm="sha256"`, `canonical_json_required=True`.
    - `response_hash_policy`: `response_hash_required=True`, `hash_algorithm="sha256"`, `canonical_json_required=True`.
    - `pairing_validation_policy`: `pairing_validation_required=True`, `source_intake_policy_hash_required=True`, `source_payload_preview_hash_required=True`, `mismatch_behavior="fail_closed"`.
    - `pairing_replay_policy`: `replay_required=True`, `hash_match_required=True`, `replay_behavior="read_only"`.
    - `mismatch_policy`: `hash_mismatch_behavior="fail_closed"`, `lineage_mismatch_behavior="fail_closed"`, `source_missing_behavior="fail_closed"`.
    - `trust_boundary_policy`: `provider_response_trusted=False`, `human_review_required=True`, `auto_accept_allowed=False`, `auto_execute_allowed=False`.
    - `manual_review_policy`: `human_review_required_before_any_trading_interpretation=True`, `human_review_required_before_any_future_broker_bridge=True`, `review_can_still_not_create_orders=True`.
    - `future_response_requirements`: `future_response_artifact_required=True`, `future_response_hash_required=True`, `future_response_trust_boundary_required=True`, `future_response_manual_review_required=True`.
  - 30 boolean safety flags including `provider_enabled`, `network_enabled`, `credentials_loaded`, `credential_value_present`, `credential_lookup_attempted`, `env_read_attempted`, `dotenv_loaded`, `provider_call_allowed`, `actual_provider_call_made`, `outbound_request_sent`, `provider_response_received`, `provider_response_trusted`, `provider_response_imported`, `provider_response_reviewed`, `provider_response_can_create_orders`, `provider_response_can_approve_orders`, `provider_response_can_call_broker`, `future_provider_execution_possible`, `trading_signal_generated`, `approval_created`, `pending_order_created`, `broker_touched`, `request_response_pair_completed`, `future_response_artifact_present`, `future_response_hash_present`, `provider_trace_id_present`, `external_correlation_id_present`, `raw_request_body_stored`, `raw_response_body_stored`.
  - Impossible boolean detection: `provider_request_response_pairing_impossible_boolean` for tampered flags.
  - denylist-clean manifest: stores only safe metadata; never stores raw forbidden fragments.

### New CLI Commands

- `atlas research provider-request-response-pairing PROVIDER_RESPONSE_INTAKE_POLICY_ID`
  - Create a request/response pairing contract artifact from an intake policy.
- `atlas research provider-request-response-pairing-list`
  - List pairing contract artifacts with safe sentinels.
- `atlas research provider-request-response-pairing-show PAIRING_ID`
  - Show one pairing contract artifact.
- `atlas research provider-request-response-pairing-validate PAIRING_ID`
  - Validate a pairing contract artifact with multiple checks.
- `atlas research provider-request-response-pairing-replay PAIRING_ID`
  - Replay and compare deterministic hashes.
- `atlas research provider-request-response-pairing-summary RUN_ID`
  - Read-only summary without writing artifacts.
- `atlas research provider-request-response-pairing-doctor`
  - Read-only chain diagnostic across all pairings.

All commands remain configless, local-only, and read-only where applicable.

### Integration
