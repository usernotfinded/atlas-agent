# Atlas Agent v0.6.7 Release Notes

> **Release Date:** 2026-06-09 (prepared, not yet tagged or released)
> **Not financial advice.** Atlas Agent is a software tool, not a financial advisor. Trading involves significant risk of loss.

## Summary

v0.6.7 is a public-onboarding maintenance patch release containing README quickstream improvements, demo output documentation, and onboarding docs/checker stale version reference cleanup. No new runtime features, broker adapters, or provider integrations are included. No trading, broker, provider, execution, risk, approval, or kill-switch behavior has changed.

This patch contains only backward-compatible maintenance improvements per Semantic Versioning 2.0.0.

## Maintenance Changes

### CAND-001 — README Quickstart Front-Loading
- Added "Try Atlas in 5 minutes" section immediately after the README status banner.
- Compressed the "Review and Feedback" link list to a curated set.
- The quickstart path remains paper-only, credential-free, and safe by default.

### CAND-002 — Demo Output Expectations and Discoverability
- Added expected output snippets, success criteria, artifact expectations, common failures, and safety notes to `docs/demo-paper-workflow.md`.
- Added paper workflow demo subsection with expected behavior and "how to know it worked" criteria to `docs/external-reviewer-walkthrough.md`.
- Reframed the README demo note constructively while preserving the exact test-required phrase.

### CAND-003 — Onboarding Docs and Checker Stale Version References
- Updated stale `v0.6.0`/`v0.6.1`/`v0.5.8` references across onboarding docs, reviewer checklist, release readiness doc, and checker scripts.
- Converted `scripts/check_onboarding_docs.py` `REQUIRED_FACTS` to use module-level current-release constants (`CURRENT_PACKAGE_VERSION`, `CURRENT_PUBLIC_RELEASE`, `NEXT_PLANNED_RELEASE`).
- Updated corresponding tests to match the constant-based approach.

## Safety Boundaries

- **Live trading** remains disabled by default.
- **Provider execution** remains disabled by default.
- **Broker order submission** remains blocked by `can_submit=false` for all non-paper brokers.
- **Credentials** are not loaded unless explicitly configured.
- **Trust** remains blocked.
- **Risk gates** remain unchanged.
- **Kill switch** and **approval queue** remain unchanged.
