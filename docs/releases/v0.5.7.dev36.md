# Release v0.5.7.dev36

## Summary

Batch 8.9 adds the **Provider Adapter Interface Contract** — a local, configless artifact that records the future adapter interface definition, provides a disabled adapter harness, and never implements a real provider adapter.

## New Module

- `src/atlas_agent/research/provider_adapter_interface_contract.py`

## New CLI Commands

- `atlas research provider-adapter-interface-contract UNLOCK_STATE_ID`
- `atlas research provider-adapter-interface-contract-list`
- `atlas research provider-adapter-interface-contract-show CONTRACT_ID`
- `atlas research provider-adapter-interface-contract-validate CONTRACT_ID`
- `atlas research provider-adapter-interface-contract-replay CONTRACT_ID`
- `atlas research provider-adapter-interface-contract-summary RUN_ID`
- `atlas research provider-adapter-interface-contract-doctor RUN_ID`
- `atlas research provider-adapter-disabled-smoke CONTRACT_ID`

## Safety Invariants

- Only the disabled adapter is available; no real provider adapter is implemented.
- Provider SDK is not imported.
- HTTP client is not imported.
- Network remains disabled.
- Credentials are not loaded.
- Provider calls remain disabled.
- `send()` on the disabled adapter always raises `ProviderAdapterDisabledError`.
- No provider request is sent.
- No provider response is received.
- No provider response is trusted.
- No approvals, pending orders, or broker calls are created.
- No trading signal is generated.
- No broker is touched.
