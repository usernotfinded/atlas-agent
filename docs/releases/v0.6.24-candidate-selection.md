## Released candidates

- **CAND-018: Hardcoded Release-Literal Sweep** — released on 2026-07-13 as
  part of `v0.6.24`. A checker/test-only candidate that adds
  `scripts/check_hardcoded_release_literals.py` and tests to scan active scripts
  for hardcoded release-identity literals matching `release-metadata.json`. It
  excludes intentionally pinned historical release-specific checkers and fixes
  the same class of drift in
  `scripts/check_autonomous_paper_workflow_demo.py` and
  `scripts/check_paper_provider_isolation.py`. It changes no runtime, safety,
  CLI, broker, provider, credential, version, or release-metadata behavior and
  does not broaden the CAND-014 boundary.

### Why CAND-018 was eligible

- It is a checker/test-only addition; it does not implement, authorize, or
  enable any trading or execution path.
- It closes a concrete release-maintenance gap observed after the `v0.6.23`
  cutover: active checker scripts can embed release identity literals that
  become stale at the next cutover.
- It uses only the Python standard library, performs no network mutation, loads
  no credentials, and does not mutate repository files.
- It preserves the CAND-014 single-module extraction boundary.
- It ships deterministic, offline tests.

## Selection rules

Any future `v0.6.25` candidate must satisfy the repository's candidate selection
rules before acceptance:

1. No live trading enablement — fail-closed defaults unchanged.
2. No broker/provider execution enablement — no direct AI-to-broker path.
3. No secrets or credentials committed.
4. No unsafe marketing claims — no promises of specific returns, eliminated
   risk, live readiness, or autonomous trading readiness.
5. Protected runtime boundaries preserved — `RiskManager`, approval queues, kill
   switch, heartbeat, deadman, audit hash-chain, live-submit opt-in, and
   `can_submit` are not bypassed.
6. Small, reviewable candidate batches that are independently valuable.
7. Deterministic test and documentation coverage.
8. Green verification before acceptance.
9. The CAND-014 provider-artifact extraction boundary is not broadened beyond
   `src/atlas_agent/research/provider_mock_response_final_safety_seal.py`.
