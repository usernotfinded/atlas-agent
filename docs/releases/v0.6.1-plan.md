## Known non-blocking issues

1. **Historical test fixture isolation** — `tests/test_discipline_cli.py::test_discipline_show_default` and `test_discipline_validate_no_file` assumed no local `.atlas/discipline.md` existed. In development environments where a local discipline profile is present (untracked, in `.gitignore`), these tests failed. Fixed in Batch 48 by isolating the tests to a temporary directory.
2. **v0.6 readiness checker post-release awareness** — `scripts/check_v060_readiness.py` failed after `v0.6.0` publication because it expected the tag not to exist. Fixed in this batch by adding a `--post-release` mode that validates the published state while preserving the default pre-release behavior.
3. **Product capability inventory doc link** — `test_inventory_doc_links_to_gap_prioritization` was previously reported as failing but now passes; no action needed.
4. **Release body trailing newline** — The GitHub release body differs from `docs/releases/v0.6.0.md` by exactly one trailing newline. This is benign formatting and does not affect content accuracy.

## Patch criteria

A `v0.6.1` patch release should be considered only when:

1. At least one maintenance candidate is a real fix (not just docs polish).
2. All local checks pass (`dev_check.sh`, `ci_check.sh`, `release_check.sh --quick`).
3. No protected boundaries (`config/`, `brokers/`, `execution/`, `safety/`, `risk/`) are modified unless the fix is directly within those boundaries and fully tested.
4. CI is green on `main` before tagging.
5. No runtime trading behavior changes are introduced.

## Release criteria for v0.6.1

Before cutting `v0.6.1`:

1. Version bumped to `0.6.1` in `pyproject.toml`, `src/atlas_agent/__init__.py`, and all fixtures/mocks.
2. `CHANGELOG.md` updated with a `[0.6.1]` entry.
3. `docs/releases/v0.6.1.md` created with release notes.
4. All checker scripts pass.
5. Full test suite passes.
6. Release assurance generated and `valid: true`.
7. Protected boundary diff is clean (no unexpected changes).
8. Dangerous pattern scan is clean.
9. No PyPI publish unless explicitly approved later.

## Non-goals

- No new runtime features for `v0.6.1`.
- No new broker adapters.
- No new provider integrations.
- No live trading default changes.
- Risk gates must remain unchanged.
- No approval queue or kill switch changes.
- No automatic skill activation or learning execution.

## Safety boundaries

The following must remain unchanged in any `v0.6.1` maintenance release unless the release is specifically scoped to fix a bug within that boundary:

- `src/atlas_agent/config/` — no new live defaults.
- `src/atlas_agent/brokers/` — no new submit paths, no execution enabled by default.
- `src/atlas_agent/execution/` — no direct AI-to-broker paths.
- `src/atlas_agent/safety/` — no kill switch bypasses.
- `src/atlas_agent/risk/` — risk limits must not be weakened.
