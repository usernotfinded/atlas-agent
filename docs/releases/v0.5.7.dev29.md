# Release v0.5.7.dev29 — Provider Credential Boundary Record & Secret-Handling Contract

## Summary

This release adds local provider credential boundary artifacts and their integration into the research chain. No real provider execution is added. No network calls. No API keys. No provider SDKs. No broker changes. No credentials are loaded.

Batch 8.2 introduces a deterministic, auditable boundary artifact that documents future credential-handling requirements before any real provider integration. It consolidates secret storage, input, output, logging, redaction, rotation, revocation, and CI policies into a standalone artifact with hash validation, replay, and timeline lineage. Provider execution remains disabled.

## What Changed

### New Artifact

- `src/atlas_agent/research/provider_credential_boundary.py`
  - `build_provider_credential_boundary_dict()`: deterministic boundary artifact construction from a provider opt-in policy.
  - `create_provider_credential_boundary()`: safe creation with validation, lineage, hash, and denylist scanning.
  - `safe_validate_provider_credential_boundary_data()`: strict validation with invalid sentinel returns.
  - `validate_provider_credential_boundary_artifact()`: detailed per-check validation.
  - `replay_provider_credential_boundary()`: deterministic hash replay.
  - `iter_provider_credential_boundary_artifacts()`: safe listing with invalid sentinels for tampered items.
  - `summarize_provider_credential_boundary_for_run()`: read-only summary without writing artifacts.
  - Secret policy sections: `secret_storage_policy`, `secret_input_policy`, `secret_output_policy`, `secret_logging_policy`, `secret_redaction_policy`, `secret_rotation_policy`, `secret_revocation_policy`, `ci_secret_policy`.
  - 14 boolean safety flags including `credential_value_present`, `credential_lookup_attempted`, `env_read_attempted`, `dotenv_loaded`.
  - Impossible boolean detection: `provider_credential_boundary_impossible_boolean` for tampered flags.
  - denylist-clean manifest: stores only safe metadata; never stores raw forbidden fragments.

### New CLI Commands

- `atlas research provider-credential-boundary PROVIDER_OPT_IN_POLICY_ID`
  - Create a credential boundary artifact from an opt-in policy.
- `atlas research provider-credential-boundary-list`
  - List credential boundary artifacts with safe sentinels.
- `atlas research provider-credential-boundary-show BOUNDARY_ID`
  - Show one credential boundary artifact.
- `atlas research provider-credential-boundary-validate BOUNDARY_ID`
  - Validate a credential boundary artifact with 17+ checks.
- `atlas research provider-credential-boundary-replay BOUNDARY_ID`
  - Replay and compare deterministic hashes.
- `atlas research provider-credential-boundary-summary RUN_ID`
  - Read-only summary without writing artifacts.

All commands remain configless, local-only, and read-only where applicable.

### Integration

- `check-artifacts` now counts and validates provider credential boundary artifacts.
- `timeline` now links credential boundary artifacts under their source opt-in policy.
- `dossier` now includes credential boundary artifacts in the research chain summary.
- Demo workflow extended with credential boundary creation, validation, replay, summary, and timeline lineage checks.
