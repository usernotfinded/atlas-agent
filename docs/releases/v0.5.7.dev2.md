# Atlas Agent v0.5.7.dev2

**Date:** 2026-05-15  
**Tag:** `v0.5.7.dev2`  
**Type:** Development tag

> **Note:** This is a development tag, not a production live-trading guarantee. Live submit remains opt-in and gated. Reconcile remains read-only. No financial outcome is promised. Users should review configuration and run `./scripts/release_check.sh` before relying on the workspace.

---

## Summary

v0.5.7.dev2 polishes the atlas-agent workspace with release automation, safety-contract documentation, output-safety regression tests, a broker-neutral reconcile capability, and Pydantic V2 compatibility cleanup. No runtime trading behavior was changed.

## Highlights

- **Release automation gate**: `scripts/release_check.sh` runs the full validation matrix (pytest, pip check, demo paper workflow, version consistency, forbidden-claims scan).
- **Live-submit safety contract**: Documented the complete live-submit boundary, opt-in gates, state machine, and audit expectations.
- **Output/audit safety tests**: 23 regression tests verify that CLI error envelopes, JSON reports, submit/reconcile reports, and live-submit audit payloads never leak raw exception text, paths, headers, broker bodies, or secrets.
- **Broker-neutral reconcile**: `run_reconcile()` no longer requires an `AlpacaBrokerAdapter` instance. It checks for a `get_order_by_client_order_id` capability on any sync provider.
- **Pydantic V2 cleanup**: `AtlasConfig` now uses `ConfigDict(arbitrary_types_allowed=True)` instead of the deprecated class-based `Config`.

## Safety / Correctness Improvements

- Reconcile remains read-only and never calls `place_order` or `resolve_execution_broker("live")`.
- Reconcile never routes through `OrderRouter`.
- All new failure paths in reconcile use static safe messages.
- `BrokerResolver.can_submit` remains `false` by default; live submit remains strictly opt-in.
- Output safety tests inject unsafe broker body/header strings into actual report paths and assert clean output.

## Developer Workflow Improvements

- `scripts/check_version_consistency.py` verifies `pyproject.toml` and `src/atlas_agent/__init__.py` match.
- `scripts/check_forbidden_claims.py` scans docs for prohibited safety/marketing language.
- Release notes are now published under `docs/releases/`.

## Upgrade Notes

- No configuration changes are required.
- No database or workspace migration is required.
- If you previously relied on `isinstance(sync_provider, AlpacaBrokerAdapter)` in custom reconcile wrappers, switch to a capability check (`callable(getattr(provider, "get_order_by_client_order_id", None))`).

## Known Limitations

- Live trading is disabled by default and remains behind multi-factor opt-in gates.
- Only Alpaca read-only sync is implemented; other broker adapters remain beta/deferred.
- The dashboard is basic and read-only.
- Self-improvement and skill refinement are early-stage.
