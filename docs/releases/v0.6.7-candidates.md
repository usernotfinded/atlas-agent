# v0.6.7 Patch Candidate Selection

> **Not financial advice.** Atlas Agent is a software tool, not a financial advisor. Trading involves significant risk of loss.

> **Status:** completed — all accepted candidates implemented, release prep in progress

`v0.6.6` is the current public GitHub release (tagged and published). `v0.6.7` is
version-prepared in source (`pyproject.toml` and `src/atlas_agent/__init__.py` bumped to
`0.6.7`), but **not** yet tagged or released. This document recorded the safe
public-onboarding maintenance candidates selected for the `v0.6.7` patch release.
Release notes (`docs/releases/v0.6.7.md`) and trust status
(`docs/trust/v0.6.7-status.md`) have been created. Tag creation, GitHub release
creation, and PyPI publishing remain unapproved and must not proceed without explicit
owner authorization.

## Context

Batch 79 (post-v0.6.6 hygiene) completed the standard post-release documentation and
checker metadata updates. `v0.6.6` is now tagged, released, and current. The next
planning line is `v0.6.7`.

This selection document focuses on **public onboarding, demo clarity, and reviewer
usability** — the most impactful low-risk improvements for external users trying to
understand and run Atlas Agent safely.

## Selection criteria

A candidate is eligible for `v0.6.7` only if **all** of the following are true:

- It improves public onboarding, reviewer clarity, or demo discoverability.
- It is a small, backward-compatible fix or docs improvement.
- It does not change runtime trading, broker, provider, risk, approval, or kill-switch behavior.
- It does not add new runtime features, broker adapters, or provider integrations.
- It does not require new secrets, credentials, or network calls.
- It is safe to implement directly on `main` without a feature branch.
- It does not weaken existing checks or safety boundaries.

## Safety boundaries

- Live trading remains disabled by default.
- Live submit remains disabled by default.
- Provider execution remains disabled by default.
- Broker execution remains disabled by default.
- Risk gates remain deterministic and unchanged.
- Approval queue behavior remains unchanged.
- Kill-switch behavior remains unchanged.
- Human approval remains required for live order flow.
- Credentials are not loaded unless explicitly configured.
- No provider APIs, broker APIs, or real webhooks are called from CI or local checks.
