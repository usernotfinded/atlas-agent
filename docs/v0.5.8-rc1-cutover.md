# v0.5.8 RC1 Cutover Guide

> **Not financial advice.** Atlas Agent is a software tool, not a financial advisor. Trading involves significant risk of loss.

## What this document is

This guide explains how to verify the `v0.5.8rc1` release candidate state locally and what steps remain before a stable `v0.5.8` tag can be created.

**This document does not create a tag, publish a package, or enable any runtime features.**

## Tag state verification

The cutover checker supports both pre-tag and post-tag verification:

- **Pre-tag** (tag does not exist yet): checker passes with `tag_state: absent_pre_tag`.
- **Post-tag** (tag exists and points to current HEAD): checker passes with `tag_state: present_matches_head`.
- **Tag mismatch** (tag exists but points to a different commit): checker fails. Force-pushing or moving RC tags is not allowed.
- **Unresolvable tag** (tag exists but cannot be resolved): checker fails.

## How to verify the RC1 candidate locally

Run these commands in order:

```bash
# 1. Version check
grep 'version =' pyproject.toml
grep '__version__' src/atlas_agent/__init__.py

# 2. Cutover verification
python3.11 scripts/check_v058_rc1_cutover.py
python3.11 scripts/check_v058_rc1_cutover.py --json

# 3. Readiness verification
python3.11 scripts/check_v058_rc1_readiness.py
python3.11 scripts/check_v058_rc1_readiness.py --json

# 4. Historical stable tag verification
python3.11 scripts/check_rc1_cutover.py

# 5. Broader release checks
python3.11 scripts/check_version_consistency.py
python3.11 scripts/check_v058_gap_prioritization.py
python3.11 scripts/check_product_capability_inventory.py
python3.11 scripts/check_forbidden_claims.py
python3.11 scripts/check_public_docs_consistency.py
python3.11 scripts/check_public_launch_readiness.py
python3.11 scripts/check_stable_release_decision.py

# 6. Full test suite
python3.11 -m pytest tests -q

# 7. Release quick gate
./scripts/release_check.sh --quick
```

All must pass.

## What must remain disabled

- **Live trading** must remain disabled by default.
- **Provider execution** must remain locked.
- **Broker execution** must remain blocked unless explicit opt-in gates pass.
- **Autonomous real-money trading** must remain out of scope.

## What NOT to do yet

Do not run these commands until explicit maintainer approval:

```bash
# DO NOT RUN until approved
git tag -a v0.5.8rc1 -m "v0.5.8rc1"
git push origin v0.5.8rc1
```

This cutover package is local-only. The tag must be created and pushed manually after review.
